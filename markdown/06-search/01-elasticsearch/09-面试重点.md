# Elasticsearch 面试重点

## 核心原理

- Elasticsearch 基于 Lucene，核心是倒排索引、segment、doc values、近实时 refresh 和分片副本。
- 倒排索引用于全文检索，doc values 面向排序、聚合和脚本访问。
- refresh 让新写入文档可搜索，flush 把 translog 和 Lucene commit 推进到更稳定状态。
- segment 不可变，删除和更新本质是标记删除并写入新文档，后台 merge 合并 segment。

## Mapping

- `text` 会分词，适合全文搜索；`keyword` 不分词，适合精确匹配、排序、聚合。
- mapping 设计错误会导致搜索不准、聚合失败、字段爆炸或存储膨胀。
- nested 解决数组对象扁平化导致的匹配错配，但查询和存储成本更高。

## 查询

- bool query 中 filter 不参与评分，适合缓存和精确过滤；must 参与评分。
- 深分页 from+size 成本高，因为每个分片都要取大量候选再全局排序。
- 深分页优先使用 search_after + PIT，导出场景用 scroll。
- 相关性默认常见 BM25，可用 function_score、boost、同义词和业务排序调整。

## 集群与性能

- 分片数不是越多越好，过多分片会增加集群状态、文件句柄、内存和调度成本。
- 写入优化关注 bulk、refresh interval、replica、routing、mapping 和 merge 压力。
- 查询优化关注过滤条件、字段类型、缓存、索引排序、路由和聚合基数。
- ES 不适合作为强事务数据库，写入成功到搜索可见存在近实时延迟。
