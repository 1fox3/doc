# 性能优化与运维管理

## 性能优化方向

Elasticsearch 性能问题通常来自分片规划、mapping 设计、查询 DSL、写入方式、merge 压力、JVM 堆、磁盘 IO 和集群状态膨胀。优化前应先定位瓶颈，不要直接调参数。

## 写入优化

- 使用 bulk 批量写入。
- 增大 `refresh_interval`，批量导入时可临时关闭 refresh。
- 导入阶段临时降低 `number_of_replicas`，完成后恢复。
- 避免频繁 update，优先追加写或批量合并更新。
- 使用自动 ID 提高纯追加写吞吐。
- 控制 ingest pipeline 复杂度。
- 关注 `indexing_pressure`、thread pool rejected、merge time 和 translog。

## 查询优化

- 精确条件放入 filter context。
- 避免大范围 wildcard、regexp、script query。
- 控制返回字段，必要时关闭 `_source` 大字段或使用 source filtering。
- 避免深分页，使用 `search_after` 和 PIT。
- 对聚合查询设置合理时间范围和桶数量。
- 使用 profile API 分析慢查询。
- 使用 query validation 检查 DSL 重写结果。

## 分片优化

- 避免海量小分片，小分片会放大集群状态、文件句柄、堆内存和调度开销。
- 避免超大分片，超大分片恢复和迁移时间长。
- 使用 rollover 按大小或时间滚动索引。
- 使用 shrink 将只读历史索引收缩为更少分片。
- 使用 force merge 优化只读索引，但不要在高写入热索引上频繁执行。

## 缓存与内存
主要缓存和保护机制：

- query cache：缓存 filter 查询结果，按节点管理。
- request cache：缓存分片级 `size=0` 聚合结果。
- fielddata cache：为 text 字段聚合/排序加载到堆，风险高。
- shard request cache：适合重复仪表盘查询。
- circuit breaker：防止单次请求或字段数据耗尽内存。

常见断路器：

- parent breaker：总内存保护。
- fielddata breaker：字段数据保护。
- request breaker：请求内存保护。
- in-flight request breaker：网络请求保护。
- script compilation breaker：脚本编译频率保护。

## JVM 与系统资源
- JVM heap 通常不超过 32 GB，并保留足够文件系统缓存给 Lucene。
- 使用 SSD 提升随机读写和 merge 性能。
- 禁用 swap 或确保不会发生 swap。
- 调整文件句柄、虚拟内存映射数量和线程数限制。
- 监控 GC、old heap、CPU steal、磁盘 await、IO util 和网络延迟。

## 运维 API
常用检查命令：

- `_cluster/health`：集群健康。
- `_cat/nodes`：节点资源概览。
- `_cat/indices`：索引大小、文档数和健康状态。
- `_cat/shards`：分片分布和未分配原因。
- `_cluster/allocation/explain`：解释分片无法分配原因。
- `_nodes/stats`：节点详细指标。
- `_nodes/hot_threads`：热点线程。
- `_tasks`：长任务和取消任务。
- `_recovery`：分片恢复进度。
- `_snapshot`：快照仓库和备份恢复。

## 慢查询与 Profile
慢日志可按索引配置，用于记录慢 search 和 slow indexing。Profile API 可以展示查询 rewrite、collector 和各子查询耗时。

排查慢查询流程：

- 确认是否只有某些索引、分片或节点慢。
- 用 profile 查看耗时 query、collector 和 fetch。
- 检查是否存在深分页、脚本、大 wildcard、高基数聚合。
- 检查节点 CPU、IO、GC 和 merge 状态。
- 根据结果调整 mapping、DSL、分片或硬件。

## 快照与恢复
Snapshot 是 ES 的标准备份方式，可备份到共享文件系统或对象存储。

实践建议：

- 定期快照关键索引和集群状态。
- 快照仓库要高可用，并限制写入权限。
- 定期演练 restore，不要只验证 snapshot success。
- 大规模恢复前规划目标集群容量、分片并发和限速。

## 索引生命周期
ILM 管理索引从 hot 到 warm、cold、frozen、delete 的生命周期。

常见动作：

- rollover：按大小、文档数或时间滚动。
- shrink：减少只读索引分片数。
- force merge：压缩只读索引 segment。
- allocate/migrate：迁移到不同数据层。
- delete：过期删除。

## 故障处理重点
- red：优先恢复主分片，查看 allocation explain 和磁盘水位。
- yellow：检查副本数、节点数量和分配约束。
- 写入 rejected：降低 bulk 并发、检查写线程池和磁盘 IO。
- 查询超时：检查慢 DSL、热点分片、聚合桶数量和协调节点压力。
- 高 heap：检查 fielddata、聚合、segment 数、mapping 字段数和缓存。
- 高磁盘：清理过期索引、检查副本、force merge 只读索引或扩容。
