## 客户端

### 分区分配策略

#### 配置项

- partition.assignment.strategy
  - 全类名(例如：org.apache.kafka.clients.consumer.RangeAssignor)

#### RangeAssignor分配策略

- 原理
  - 按照消费者的总数和分区总数进行整除运算来获得一个跨度，然后将分区按照跨度进行平均分配

#### RoundRobinAssignor分配策略

- 原理
  - 将消费组内的所有消费者及消费者订阅的所有主题的分区按照字典排序，然后通过轮询的方式将分区依次分配个每个消费者

#### StickyAssignor分配策略

- 原则
  - 分区分配要尽可能的均匀
  - 分区的分配尽可能与上次分配保持相同

#### 自定义分配策略

- PartitionAssignor接口assign方法

- Subscription
  - 消费者的订阅消息

- Assignment
  - 标识分配结果信息

### 消费者协调器(ConsumerCoordinator)和组协调器(GroupCoordinator)

#### 主要负责消费者再均衡的操作

#### 触发再均衡的情况

- 有戏的消费者加入消费组

- 有消费者宕机下线

- 有消费者主动退出消费组

- 消费组对应得GroupCoorinator节点发生了变化

- 消费组内订阅的任一主题或主题的分区数量发生了变化

#### 消费者加入消费者组操作过程

- FIND_COORDINATOR
  - 消费者找到消费组对应的组协调器的broker，并建立网络连接

- JOIN_GROUP
  - 加入消费者组
    - 选举消费组内leader
      - 无leader时，选择第一个加入消费组的消费者
      - 有leader时，使用HashMap中的第一个
    - 选举分区分配策略
      - 收集各个消费者支持的所有分配策略，作为候选集
      - 每个消费者从候选集中找出第一个自身支持的粗略
      - 选票数最多的策略作为消费组的分配策略

- SYNC_GROUP
  - 将选举出来的分配策略来实施分区分配，并通过组协调器将分配方案同步给消费者

- HEARTBEAT
  - 消费者处于正常工作状态，并通过项组协调器发送心跳来位置他们与消费组的从属关系，及对分区的所有权关系
    - heartbeat.interval.ms
    - session.timeout.ms

### 事务

#### 消息保障的3个层级

- at most once至多一次
  - 消息可能会丢失，饭绝对不会重复

- at least once最少一次
  - 消息绝不会丢失，但可能会重复传输

- exactly once恰好一次
  - 每一条消息肯定会被传输一次且仅传输一次

#### 幂等

- 配置
  - enable.idempotence=true

- 实现方式
  - producer id
  - 序列号

- 只能保证单生产者回话中单分区的幂等

#### 事务

- 配置
  - transactional.id

- kafka保证
  - 跨生产者会话的消息幂等发送及跨生产者会话的事务恢复
  - 前者表示具有相同的transactionalId的新生产者创建且工作的时候，旧的拥有相同tanasactionalId的生产者将不在工作
  - 后者指某个生产者宕机后，新的生产者可以保证任何未完成的事务要么被提交，要么被中止，使新的生产者可以从一个正常的状态开始工作

- 控制消息
  - COMMIT
  - ABORT

- 事务协调器(TransactionCoordinator)
  - __transaction_state

- 过程
  - 查找TransactionCoordinator
  - 获取PID
  - 开启事务
  - 事务数据处理
  - 提交或中止事务
