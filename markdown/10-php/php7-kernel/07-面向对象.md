## 面向对象

### 类

#### 不管是内部类还是自定义类，都同zend_class_entry结构表示，类的常量、成员属性、成员方法均保存在这个结构中

#### zend_class_entry

- type
  - 类的类型，内部类、用户自定义类

- name
  - 类名

- zend_class_entry *parent
  - 父类

- ce_flags
  - 类掩码

- default_properties_count
  - 普通属性数

- default_static_members_count
  - 静态属性数

- zval *default_properties_table
  - 普通属性值数组

- zval *default_static_members_table
  - 静态属性值数组

- zval *static_members_table

- HashTable function_table
  - 成员方法符号表

- HashTable properties_info
  - 成员属性基本信息哈希表

- HashTable constants_table
  - 常量符号表

#### 类编译完后注册到一个全局符号表中EG(class_table)

#### 编译过程

- 分配zend_class_entry结构，如果有继承的父类，则编译一条ZEND_FETCH_CLASS指令

- 编译一条类声明得到ZEND_DECLARE_CLASS指令，该指令的作用时将编译后的类注册到EG（class_table）

- 将CG(active_class_entry)指向当前类的zend_class_enrty结构，然后编译常量，成员属性，成员方法，并把编译结果注册到CG（active_class_entry）对应的符号表中

#### 类的自动加载

- 方式
  - __autoload()
    - EG(autoload_func)
    - 只能定义一个加载器
  - spl_autoload_register()
    - 提供了更加灵活的注册方式，可支持任意数量的加载器
    - spl创建了一个队列来保存用户注册的加载器，然后定义了一个spl_autoload_call函数到EG(autoload_func)

- 实际上就是一个钩子函数，实例化类时如果在EG(class_table)中没有找到对应的类，则会调用这个钩子函数，调用完以后再重新查找一次

### 对象

#### zend_object

- handle
  - 一次request期间对象的编号，每个对象都有一个唯一的编号，主要在垃圾回收时使用

- zend_class_entry ce
  - 所属类

- const zend_object_handlers *handlers
  - 对象操作的处理函数，成员属性的读写、成员方法的获取、对象的销毁/克隆等

- properties

- properties_table
  - 非静态成员属性数组

#### ==

- 两个对象的属性和属性值都相等，而且时同一个类的实例

#### ===

- 两个对象一定要指向某个类的同一个实例，即同一个对象

### 继承

#### 信息通过合并的方式，存储到子类中

### 动态属性

#### 动态属性保存在zend_object->properties哈希表中

### 魔术方法

#### 魔术方法与普通方法一样保存在zend_class_entry->function_table中，针对内核常用的成员方法在zend_class_entry中还有一些单独的指针指向具体的成员方法
