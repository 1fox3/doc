# 微服务排障案例

## 调用超时

- 排查：链路追踪定位慢 span，确认是网络、下游服务、数据库、线程池还是序列化。
- 原因：超时配置不合理、下游慢、连接池耗尽、重试放大、线程池隔离不足。
- 修复：设置超时预算、熔断限流、连接池治理、慢 SQL 优化、降级兜底。

## 服务雪崩

- 现象：一个下游慢导致上游线程池打满，故障扩散。
- 排查：调用拓扑、错误率、线程池、重试次数、熔断状态。
- 修复：隔离、熔断、限流、降级，禁止无边界重试。

## 注册发现异常

- 现象：服务不可达、实例列表不一致、流量打到异常实例。
- 排查：注册中心健康、客户端本地缓存、心跳、网络分区、实例 metadata、健康检查。
- 修复：缩短异常摘除时间，增强健康检查，发布时优雅上下线。

## 配置未生效

- 排查：namespace、group、dataId、profile、刷新机制、Bean 是否支持动态刷新、配置优先级。
- 修复：明确配置分层，关键配置灰度发布，保留版本和回滚。

## 灰度事故

- 原因：路由规则不完整、流量标识丢失、缓存共享、数据库变更不兼容。
- 修复：全链路灰度标签，兼容性发布，监控灰度指标，支持快速回滚。
