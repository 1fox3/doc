# 服务调用与网关

## 服务调用方式

微服务之间常见调用方式包括 REST、gRPC、Dubbo、消息队列和事件订阅。同步调用适合需要立即返回结果的查询或短事务，异步调用适合削峰、解耦、最终一致性和非核心链路。

## OpenFeign

OpenFeign 是声明式 HTTP 客户端，通过接口和注解描述远程 API。它通常与服务发现、负载均衡、熔断、重试、日志和编码解码器组合使用。

关键点：

- 明确每个 Feign Client 的连接超时和读取超时，不要使用全局默认值覆盖所有场景。
- 避免在 Feign 层写复杂业务逻辑，Feign 接口应只表达远程契约。
- 对非幂等写接口谨慎开启自动重试，避免重复下单、重复扣款或重复发券。
- 错误解码器应把下游错误转换为明确异常类型，便于上层降级和告警。

## HTTP 契约设计

- API 路径、请求体、响应体和错误码需要版本化管理。
- 对外暴露接口应保持向后兼容，字段新增优于字段语义变更。
- 分页、排序、过滤和幂等键应形成统一规范。
- 错误响应中应包含稳定错误码、可读信息和 TraceId。
- 内部接口也要有契约测试或集成测试，避免服务独立发布时破坏调用方。

## Spring Cloud Gateway

Spring Cloud Gateway 基于 Spring WebFlux 和 Reactor Netty，支持路由匹配、过滤器、限流、熔断、重试、Header 改写和灰度路由。

核心概念：

- Route：路由定义，包含目标 URI、断言和过滤器。
- Predicate：路由断言，根据 Path、Host、Method、Header、Query、Cookie、时间等条件匹配请求。
- Filter：过滤器，用于鉴权、限流、日志、Header 改写、路径重写、熔断和响应处理。
- GlobalFilter：全局过滤器，对所有路由生效，适合统一 TraceId、认证解析和审计日志。

## 网关职责边界

适合放在网关的能力：

- TLS 终止、统一入口和路由转发。
- 认证解析、基础鉴权和 Token 校验。
- 限流、黑白名单、请求大小限制和基础防刷。
- TraceId 注入、访问日志和基础审计。
- 灰度路由、Header 改写和协议适配。

不适合放在网关的能力：

- 大量业务分支和复杂聚合逻辑。
- 需要强事务的业务写流程。
- 依赖多个下游服务的重型编排。
- 个性化程度很高的业务校验。

## Zuul 与 Gateway

Zuul 1.x 基于 Servlet 阻塞模型，属于 Netflix 组件体系。Spring Cloud Gateway 基于响应式模型，和 Spring Cloud 新版本生态集成更好，是当前主流选择。迁移时需要重点检查过滤器执行顺序、Header 处理、异常响应和限流实现差异。

## 灰度路由

灰度路由可以基于 Header、Cookie、用户 ID、租户、版本号、地域或权重实现。请求进入网关后应生成稳定路由标签，下游服务注册元数据中标识版本或灰度分组，指标和日志按版本维度聚合，灰度规则通过配置中心动态调整并保留审计记录。
