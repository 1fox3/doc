# Netty 面试重点

## I/O 模型

- BIO 简单但线程成本高，NIO 使用非阻塞 Channel 和 Selector，适合高并发连接。
- Reactor 将事件分发和业务处理解耦，Netty 常见 BossGroup 接收连接，WorkerGroup 处理读写。
- 一个 Channel 通常绑定一个 EventLoop，同一 Channel 事件串行执行，避免并发加锁。

## Pipeline

- Inbound 事件从头到尾传播，Outbound 事件从尾到头传播。
- 编解码器要处理半包、粘包、协议长度、魔数、版本和序列化格式。
- 耗时业务不能直接在 EventLoop 执行，应切到业务线程池。

## ByteBuf

- readerIndex 和 writerIndex 分离读写位置。
- 直接内存减少一次 JVM 堆到 native 的复制，适合网络 IO，但需要关注释放。
- 引用计数对象必须 release，泄漏会导致直接内存上涨。
- 池化减少分配成本，但增加生命周期管理复杂度。

## 高性能点

- 批量 flush、零拷贝、池化内存、直接内存、减少线程切换、合理水位线和背压。
- TCP 参数如 backlog、keepalive、nodelay、send/receive buffer 会影响连接和延迟。
