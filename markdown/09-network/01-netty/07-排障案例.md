# Netty 排障案例

## EventLoop 阻塞

- 现象：连接延迟、读写超时、单个 EventLoop CPU 高。
- 排查：线程栈、JFR、业务 Handler 是否执行阻塞 IO、慢日志。
- 修复：耗时逻辑切业务线程池，异步化下游调用，控制任务队列。

## 直接内存泄漏

- 现象：堆不高但 RSS 上涨，最终 Direct buffer memory 或容器 OOMKill。
- 排查：Netty leak detector、ByteBuf 引用计数、NMT、allocator 指标。
- 修复：明确 release 责任，使用 try/finally，避免跨线程后忘记释放。

## 粘包半包

- 原因：TCP 是字节流协议，没有消息边界。
- 修复：固定长度、分隔符、长度字段或自定义协议头，不能假设一次 read 是一条完整消息。

## 写缓冲堆积

- 现象：内存上涨、延迟升高、连接被动断开。
- 排查：ChannelOutboundBuffer、水位线、下游网络、客户端消费速度。
- 修复：设置高低水位线，启用背压，限速或断开慢客户端。
