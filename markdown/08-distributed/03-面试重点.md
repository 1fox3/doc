# 分布式面试重点

## 分布式基础

- CAP 不是三选二的口号，而是在网络分区存在时一致性和可用性的取舍。
- BASE 强调基本可用、软状态和最终一致，常用于互联网业务系统。
- 幂等是分布式系统必备能力，常用唯一请求号、业务唯一键、状态机和去重表实现。

## RPC

- RPC 核心链路包括代理、序列化、协议、连接池、负载均衡、超时、重试、熔断和线程模型。
- 超时和重试必须结合幂等性，非幂等接口盲目重试会放大故障。
- 序列化要关注性能、兼容性、跨语言、安全和 schema 演进。

## Dubbo

- 服务暴露：ServiceBean -> Proxy -> Invoker -> Protocol export -> Registry register。
- 服务引用：ReferenceBean -> Registry subscribe -> Directory -> Cluster -> Invoker proxy。
- 集群容错包括 Failover、Failfast、Failsafe、Failback、Forking、Broadcast。
- Filter 是治理扩展点，可做鉴权、日志、监控、限流和链路追踪。

## ZooKeeper

- ZK 适合小数据协调，不适合大数据存储和高频写。
- 临时节点跟随 Session 生命周期，顺序节点适合实现队列和锁。
- Watcher 是一次性通知，需要重新注册，过多 Watcher 会带来服务端压力。
- ZAB 保证主从复制一致性，写请求由 Leader 处理并通过过半确认提交。

## 分布式锁

- 锁要考虑互斥、可重入、超时释放、续期、锁释放安全和 fencing token。
- Redis 锁性能好但一致性有限；ZK 锁一致性更强但延迟和运维成本更高。
- 关键资源更推荐数据库唯一约束、状态机或幂等设计，而不是只依赖锁。
