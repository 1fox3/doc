# Dubbo

Dubbo 是面向服务接口的高性能 RPC 框架，核心能力包括服务暴露、服务引用、注册发现、协议通信、序列化、集群容错、负载均衡、路由、过滤器、异步调用和扩展机制。

## 技术重点

- URL 是 Dubbo 内部传递配置和运行时参数的统一模型。
- Invoker 是可执行调用实体，服务端代表可被调用的服务实现，消费端代表远程调用代理。
- Protocol 负责服务暴露和引用，Cluster 负责把多个 Invoker 聚合成一个具备容错能力的 Invoker。
- Directory 维护可用 Invoker 列表，Router 负责路由过滤，LoadBalance 负责实例选择。
- Filter 链适合做监控、鉴权、日志、限流、上下文传递和灰度标签处理。

## 版本趋势

- Dubbo 3.x 强化应用级服务发现、Triple 协议、云原生和 Mesh 适配。
- 新项目需要重点关注接口粒度、超时重试、线程池隔离、序列化兼容、调用链观测和注册中心容量。
