# 扩展机制与生产治理

## SPI 扩展

Dubbo 使用增强版 SPI 管理扩展点，支持按名称加载、自适应扩展、自动包装和依赖注入。Protocol、Cluster、LoadBalance、Router、Filter、ProxyFactory、Serialization 等都通过扩展点实现。

扩展实现要注意线程安全、初始化成本、异常隔离和默认行为兼容。

## Filter 链

Filter 适合实现横切逻辑，例如日志、指标、Trace、鉴权、限流、参数校验、灰度标签、上下文传递。Filter 中不应执行耗时阻塞逻辑，也不能吞掉异常导致上层误判成功。

## 配置治理

关键配置包括：

- 超时：`timeout`，按接口和方法设置。
- 重试：`retries`，非幂等写接口通常关闭。
- 线程：`threads`、`queues`，结合接口耗时和容量设置。
- 并发：`actives`、`executes`，保护 Consumer 和 Provider。
- 连接：`connections`，控制长连接数量。
- 序列化：`serialization`，确保多版本兼容。
- 路由和标签：用于灰度、机房、版本和泳道。

## 生产排障

- 找不到服务：检查注册中心、接口名、版本、分组、应用级服务发现配置和路由规则。
- 调用超时：检查超时配置、Provider 线程池、网络、序列化、下游依赖和 GC。
- 流量倾斜：检查权重、负载均衡策略、连接复用、一致性哈希和慢实例。
- 重试放大：检查 Failover、retries、上游超时预算和接口幂等性。
- 线程池耗尽：检查慢 SQL、下游阻塞、队列大小、线程数和接口隔离。

## 监控指标

核心指标包括请求量、成功率、错误率、P95/P99 延迟、超时数、重试数、限流数、降级数、Provider 线程池活跃数、队列长度、连接数和注册中心推送延迟。

## 优雅停机

优雅停机应先取消注册或禁用流量，再等待 Consumer 感知变更和存量请求处理完成，最后关闭服务端口和线程池。直接杀进程会造成请求失败和 Consumer 短时间调用失效实例。

## 安全治理

内部 RPC 也需要鉴权、访问控制和审计。泛化调用、运维调用和跨应用调用应有白名单、签名、Token 或 mTLS 保护，避免内部接口被任意调用。
