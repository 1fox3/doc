## 集群容错与负载均衡策略

### 容错

#### 容错策略

- Failover Cluster（默认）
  - 失败重试,需配置重试次数
  - <dubbo:refreence retires="2"/>
  - <dubbo:method name=“sayHello” retires="2"/>
    - 针对具体方法

- Failfast Cluster
  - 快速失败
  - 仅调用一次

- Failsafe Cluster
  - 安全失败
  - 忽略调用时的异常

- Failback Cluster
  - 失败自动恢复
  - 调用失败后，在后台记录失败的请求，并按照一定的策略再进行重试

- Forking Cluster
  - 并行调用
  - 并行调用多个服务提供者，只要其中一个返回成功即可
  - forkes指定并行数

- Broadcast Cluster
  - 广播调用
  - 逐个调用所有的服务提供者，任意一台服务器调用异常则这次调用失败

- 自定义
  - 扩展Cluster接口

#### 容错策略实现

- 集群容错策略是在服务降级策略后进行的

- FailfastClusterInvoker

- FailsafeCluserInvoker

- FailoverClusterInvoker

- FailbackClusterInvoker
  - 创建了一个自定义定时器，并创建了重试任务

- ForkingClusterInvoker
  - 并行数小于等于0或超出提供者个数，则以提供者个数为准

- BroadcastClusterInvoker

#### 实现自定义集群容错策略

- 实现Cluster接口，通过join方法返回对应的ClusterInvoker类

- 继承AbstractCluterInvoker类创建自定义的ClusterInvoker类，并重写doInvoke方法

### 负载均衡

#### 负载均衡策略

- Random LoadBalance
  - 随机策略
  - 按照概率设置权重，可动态调节提供者的权重

- RoundRobin LoadBalance
  - 轮询策略

- LeastActive LoadBalance
  - 最小活跃调用数

- ConsistenHash LoadBalance
  - 一致性Hash策略
  - 保证相同参数的请求发到同一个提供者，如果某一台宕机了，将基于虚拟节点平摊给其他提供者

#### 负载均衡策略实现

- RandomLoadBalance
  - ThreadLocalRandom
  - 会根据容器启动时间和需要预热的时间灵活计算权重

- RoundRobinLoadBalance

- LeastActiveLoadBalance
  - 优先根据最小活跃数
  - 多个存在多个Invoker则随机策略

- ConsistenHashLoadBalance
  - 一致性Hash的特性
    - 单调性
    - 分散性
    - 平衡性
  - 虚拟节点

#### 自定义负载均衡策略

- 实现LoadBalance接口或直接继承AbstractLoadBalance
