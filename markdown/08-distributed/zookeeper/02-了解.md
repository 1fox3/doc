## 脑图内容

## 了解

### 名词

#### 菜谱（recipes）

- 表示原语的实现，包括ZooKeeper操作和维护一个小型的数据节点(znode)

#### znode

- 用类似于文件系统的层级树状结构进行管理

- 使用字节数组存储数据，具体格式由具体应用实现

- 不能读取或修改部分数据

- 每一个znode都有一个版本号，每次数据变化都会自增，调用API时仅当版本号匹配才可成功

### API

#### create/path data

#### delete/path

#### exists/patth

#### setData/path data

#### getData/path

#### getChildren/path

### znode类型

#### 持久节点（persistent）

#### 临时节点（ephemeral）-e

- 创建该znode的客户端的会话因超时或主动关闭而中止时，删除临时节点

- 某个客户端可主动删除临时节点

- 临时节点不能有子节点

#### 有序节点（sequential）-s

- 分配一个单调递增的整数,可与前2种节点组合

### 监控与通知

#### 监视点是一个单次触发操作，需在收到通知后读取节点不安华，并设置新的监视点

#### 监控种类

- znode的数据变化

- znode子节点变化

- znode的创建和删除

### 架构

#### 运行模式

- 独立模式(standalone)
  - 单台机器

- 仲裁模式(quorum)
  - 多台机器，幼教Zookeeper集合（ZooKeeper ensemble）
  - 法定人数

#### 会话

- 会话请求以FIFO顺序执行

- 生命周期
  - NOT_CONNECTED
  - CONNECTING
  - CONNECTED
  - CLOSED

- 发生网络分区时等待CONNECTING
  - 服务器决定会话是否过期
  - 客户端只能主动关闭会话，不能声明会话过期

- 尝试连接服务器
  - 客户端经过会话超时的1/3时间未收到任何消息， 向服务器发送心跳
  - 客户端经过会话超时的2/3时间未收到任何消息， 寻找其他服务器
  - 如何选择连接哪一台服务器
    - 比较客户端和服务器的事务表示id（zkid）

#### 相关命令

- stat /path true
  - 得到一个znode的属性，并设置监视点
