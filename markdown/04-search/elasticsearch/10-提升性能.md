### 查询验证

#### 接口

- _validate
  - 验证查询，并不会真正执行

- 返回结果
  - valid
    - true,表明语句是正确的
  - _shards
    - validate API是随机在某个分片上执行的，因此总分片数总是1
  - explanations
    - 包含底层重写过的查询，如果查询正确，那么真正执行的就是这个语句，否则里面是对为什么查询不正确的详细解释

### 获取详细查询执行报告的查询分析器

#### profile

#### 结果分析

- profile.shard.id
  - 响应中包含的每个分片的唯一id

- profile.shard.searches
  - 包含查询执行详细信息的数组

- profile.shard.rewrite_time
  - 一次完整的查询改写过程所花费的总时间

- profile.shard.collector
  - 这一部分时关于运行查询的Lucene收集器内容

### 热点线程

#### 接口 (4)

- /_nodes/hot_threads

#### 参数

- trheads
  - 需要分析的线程数，默认3个

- interval
  - 为了计算线程在某项操作上花费时间的百分比，ES会对线程做二次检查，可以使用interval参数来定义2次检查的间隔时间，默认500ms

- type
  - 需要检查线程状态的类型，默认是cpu

- snapshots
  - 需要生成的堆栈的快照数量，默认是10个

### 基于时间索引的管理

#### shrink API

- 减少索引数

- 过程
  - 创建一个与源索引有相同定义的目标索引，但主分片数更少
  - 创建硬连接，将源索引中的段连接到目标索引
  - 最后对目标索引执行恢复流程，类似索引关闭再打开

- 要求
  - 目标索引不能存在与集群中
  - 目标索引的分片数必须为源索引分片数的因子
  - 收缩操作节点必须有足够的磁盘空间
