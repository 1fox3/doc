## 底层索引控制

### 相似度模型(评分方式)

#### 可用模型

- TF-IDF
  - ES5.0之前的默认相似度模型

- BM25
  - ES5.0开始默认相似度模型

- DFR-随机偏离模型
  - 是一种基于同名概率模型的相似度模型

- DFI-独立偏离模型
  - 是一种基于同名概率模型的相似度模型

- IB-基于信息的模型
  - 与随机偏离模型类似

- LM Dirichlet模型
  - 该相似度模型结合了狄利克雷与贝叶斯平滑

- LM Jelinek Mercer模型
  - 该相似度模型使用了Jelinek Mercer平滑方法

#### 为每个字段指定相似度模型

- similarity

### 选择合适的目录实现-store模块

#### 存储类型配置key

- index.store.type

#### 存储类型

- simplefs
  - 随机存取文件

- niofs
  - 允许多线程并发操作同一个文件

- mmapfs
  - 读取文件时，会将文件映射到同样大小的虚拟地址空间中去。因为mmap没有加锁操作，因此在多线程读写的时候时可扩展的。

- 默认的混合文件系统fs
  - 32位windows上使用simplefs
  - 在其他32位系统上是niofs
  - 在64位系统上是mmapfs

### 索引更新

#### 强制索引更新

- _refresh

#### 索引更新周期

- refersh_interval
  - 默认1秒

### 事务日志

#### 强制事务日志刷新

- _flush

#### 相关配置

- index.translog.sync_interval
  - 日志多久同步到磁盘上一次，默认5秒

- index.translog.durability
  - 控制在每次索引、删除、更新或批量请求操作后，是否要同步并提交事务日志
    - request
      - 每次请求都会同步并提交
    - async
      - 每隔sync_interval才会在后台做一次同步和提交操作

- index.translog.flush_threshold_size
  - 事务日志的最大容量，当容量超过该参数值时，就强行进行事务日志刷新操作，默认512MB

#### 数据恢复

- elasticsearch-translog truncate -d /path/to/trans/log/

#### 实时读取

- pretty

### 段合并

#### tiered合并策略

- 配置项
  - index.merge.policy.expunge_deletes_allowed
    - 默认10，用于衡量段中已被删除的文档所占的版本比，当执行expungeDeletes时，该参数用于确定索引段是否被合并
  - index.merge.policy.floor_segment
    - 默认2MB，该参数用于阻止频繁刷新微小索引段。小于该数值的索引段有索引合并机制处理，因为索引合并机制用得这和这个参数值大小相同。
  - index.merge.policy.max_merge_at_once
    - 默认10，该参数确定了索引期单次合并设计的索引段数量的上线
  - index.merge.policy.max_merge_at_once_explicit
    - 默认30，该参数确定了索引优化(optimize)操作和expungeDeletes操作能参与的索引段数量上线
  - index.merge.policy.max_merged_segment
    - 默认5GB，确定了索引期段合并中产生的单个索引段的上限
  - index.merge.policy.segments_per_tier
    - 默认10，确定了每层允许出现的索引段数量上线
  - index.reclaim_deletes_weight
    - 默认20，去定了索引合并操作中清除被删除文档这个因素的权重。
  - index.compund_format
    - 默认false，确定了索引是否存储为复合文件格式。为true时，将所有文件存储在一个文件中，

#### ConcurrentMergeScheduler(并发合并调度器)

- 配置项
  - index.merge.scheduler.max_thread_count
    - 最大线程数

#### 强制合并

- 接口
  - /library/_forcemerge

- 参数
  - max_num_segments
    - 合并结束后的段的数量
  - only_expunge_deletes
    - 控制合并操作是否只设计那些里面有删除操作的段，默认为false
  - flush
    - 在强制合并操作结束后，是否需要执行刷新操作，默认为true

### ES缓存

#### 节点查询缓存

- LRU(最近最少使用)淘汰策略

- 配置
  - indices.queries.cache.size
    - 这个参数控制过滤器缓存所使用的内存大小，默认10%
  - indices.queries.cache.enabled
    - 是否启用查询缓存，默认为true

#### 分片查询缓存

- 配置
  - index.requests.cache.enable
    - 默认true
  - index.requests.cache.size
    - 默认1%，缓存占用堆的占比
  - request_cache
    - 针对请求启用或禁用查询缓存

- 默认仅缓存size=0的查询结果

#### 字段数据缓存

- indices.fielddata.cache.size
  - 控制字段数据缓存，默认不设限制

- 推荐使用doc values

#### circuit breaker(断路器)

- 功能
  - 用于限制某些特定的功能使用过多的内存

- 分类
  - 父亲circuit breaker
    - indices.breaker.total.limit
      - 默认JVM堆的70%
  - 字段circuit breaker
    - indices.breaker.fielddata.limit
      - 默认JVM堆的60%
  - request circuit breaker
    - indices.breaker.request.limit
      - 当高于此属性时，拒绝执行查询
  - In-flight request circuit breaker
    - network.breaker.infilight_requests.limit
      - 默认JVM的100%
  - 脚本编译circuit breaker
    - script.max_comlilations_per_minute
      - 默认15，一分中内允许编译多少份不同的脚本
