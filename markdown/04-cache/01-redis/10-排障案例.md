# Redis 排障案例

## Redis 变慢

- 排查：`SLOWLOG`、`LATENCY DOCTOR`、`INFO commandstats`、CPU、网络、客户端连接数。
- 常见原因：big key、复杂命令、fork RDB/AOF rewrite、内存 swap、网络拥塞、热 key。
- 修复：拆分 big key，避免 `KEYS` 等危险命令，分片热点，优化持久化窗口。

## 缓存雪崩

- 现象：DB QPS 突增，Redis 命中率下降。
- 原因：大量 key 同时过期、Redis 集群故障、缓存预热不足。
- 修复：TTL 加随机抖动，多级缓存，限流降级，热点 key 预热和逻辑过期。

## 内存打满

- 排查：`INFO memory`、`MEMORY STATS`、`--bigkeys`、`--memkeys`、过期策略、淘汰策略。
- 原因：big key、过期 key 清理不及时、淘汰策略不匹配、碎片率高。
- 修复：设置 maxmemory 和淘汰策略，拆分数据，治理 TTL，开启主动碎片整理需评估 CPU。

## 主从切换后数据不一致

- 原因：异步复制存在复制延迟，主节点故障前未复制的数据可能丢失。
- 处理：业务侧接受最终一致，关键写入使用 WAIT 或更强外部一致性方案，监控复制延迟。
