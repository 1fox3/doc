## 补充与实践

- 补充：Redis 7 逐步以 listpack 替代 ziplist，底层编码会随元素数量和大小自动转换。
- 实践：选择数据类型时要同时考虑命令复杂度、内存编码、过期粒度和后续扩展。
- 排障：大 key 会造成阻塞、复制延迟和迁移抖动，应通过拆分、压缩或异步清理治理。

## 数据结构与对象

### 简单动态字符串(sds)

#### 用途

- 1、作为缓存key

- 2、作为字符串对象

- 3、缓冲区

#### 定义

- int len
  - buf数组已使用的字节数量

- int free
  - buf数组为使用的字节数量

- char buf[]
  - 字节数组，用于保存字符串

#### 与C字符串的区别

- 1、获取长度复杂度
  - C- O(N)
  - SDS - O(1)

- 2、杜绝缓冲区溢出

- 3、减少修改字符串时带来的内存重分配次数
  - 1、分配空间
    - SDS长度小于1M时
      - 长度翻倍
    - SDS长度大于1M时
      - 增加1M
  - 2、惰性释放
    - free记录剩余字节数量

- 4、二进制安全

- 5、兼容部分C字符串函数

### 字典(dict)

#### 用途 (8)

- 数据库

- 哈希键

#### dictht

- table
  - 哈希表数组(dictEntry)

- size
  - 哈希表大小

- sizemask
  - 哈希表大小掩码，用于计算索引值，总是等于size-1

- used
  - 哈希表已有节点值

#### dictEntry

- key
  - 键

- v (union)
  - 值，可以是指针，unit64_t，int64_t整数

- next
  - 下一个节点指针，主要用于处理键冲突问题

#### dict

- type
  - 类型特性函数，指向一个dictType结构指针，每个dictType结构保存了一簇用于操作特定类型键值对的函数

- privdata
  - 私有数据，保存了需要传递给这些特定函数的可选参数

- ht[2]
  - 哈希表,每个都是一个dictht，默认使用ht[0]

- rehansidx
  - rehash索引，当rehash不在进行时，值为-1

#### dictType

- hashFunction
  - 计算哈希值的函数

- keyDup
  - 复制键的函数

- valDup
  - 复制值的函数

- keyCompare
  - 对比键的函数

- keyDestructor
  - 销毁键的函数

- valDestructor
  - 销毁值的函数

#### rehash过程

- 1、为字典ht[1]哈希表分配空间
  - 扩展操作：第一个大于ht[0].used*2的2的n次幂
  - 收缩操作：第一个大于ht[0].used的2的n次幂

- 2、将ht[0]的所有键值对rehash到ht[1]上，rehashidx设置为0， 并每迁移一个键值对，rehashidx加1

- 3、当ht[0]包含的所有键值对都迁移到ht[1]后释放ht[0],并把ht[1]设置为ht[0],并为ht[1]创建一个空白哈希表

#### rehash过程间

- 1、ht[0]不在新增键值对

- 2、删除、查找、更新等操作会在2个哈希表上进行

#### 扩展

- 1、没有执行BGSAVA或者BGREWRITEAOP命令时，且哈希表的负载因子大于1

- 2、执行BGSAVA或者BGREWRITEAOP命令时，且哈希表的负载因子大于5

#### 收缩

- 负载因子小于0.1时

### 跳跃表(skiplist)

#### 定义 (17)

- 跳跃表是一种有序数据结构，它通过在每个节点中维持多个指向其他节点的指针，从而达到快速访问节点的目的

#### 用途 (18)

- 有序集合键

- 集群节点中作内部数据结构

#### zskiplist(用于保存跳跃表节点的相关信息)

- header
  - 指向跳跃表头节点

- tail
  - 指向跳跃表尾节点

- level
  - 记录跳跃表内，层数最大的那个节点的层数

- level
  - 层

#### zskiplistNode(标识跳跃表节点)

- level(层)
  - forward(前进指针)
  - span(跨度)-记录2个节点之间的距离，主要用于计算排位

- backward(后退指针)

- score(分值)

- obj(成员对象)

### 整数集合(intset)

#### intset

- encoding
  - 编码方式
    - INTSET_ENC_INT16
    - INTSET_ENC_INT32
    - INTSET_ENC_INT64

- length
  - 集合包含的元素数量

- contents[]
  - 保存元素的数组，元素从小到大排序，且不包含重复项

#### 升级

- 过程
  - 1、根据新元素的类型，扩展整数集合底层数组的空间大小，并为新元素分配空间
  - 2、将底层数组现有的所有元素都转换成与新元素相同的类型，并将类型转换后的元素放置到正确的位上，而且在放置元素的过程中，需要位置底层数组的有序性质不变
  - 3、将新元素添加到底层数组里

- 好处
  - 提升灵活性
  - 节约内存

#### 降级

- 维持现状，不作任何操作

  - 整数值

  - content(节点值)

#### 连锁更新

- 在表头插入新节点，导致后续所有节点更新

- REDIS_ENCODING_HT(字典)

- REDIS_ENCODING_LINKEDLIST(双端链表)

- REDIS_ENCODING_INTSET(整数集合)

- REDIS_ENCODING_SKIPLIST(跳跃表和字典)

#### 类型与编码

- REDIS_STRING-字符串
  - REDIS_ENCODING_INT(long类型的整数)
  - REDIS_ENCODING_EMBSTR(embstr编码的简单动态字符串)
  - REDIS_ENCODING_RAW(简单动态字符串)

- REDIS_LIST-列表
  - REDIS_ENCODING_ZIPLIST(压缩列表)
    - 使用条件
      - 1、所有元素长度小于64字节

        - list-max-ziplist-entries
  - REDIS_ENCODING_LINKEDLIST(双端链表)

- REDIS_HASH-哈希
  - REDIS_ENCODING_ZIPLIST(压缩列表)
    - 使用条件
      - 1、所有元素长度小于64字节

      - 2、元素数量小于512个
        - hash-max-ziplist-entries
  - REDIS_ENCODING_HT(字典)

- REDIS_SET-集合
  - REDIS_ENCODING_ZIPLIST(压缩列表)
    - 使用条件
      - 1、集合对象所有元素都是整数值

        - set-max-ziplist-entries
  - REDIS_ENCODING_HT(字典)

- REDIS_ZSET-有序集合
  - REDIS_ENCODING_ZIPLIST(压缩列表)
    - 使用条件
      - 1、元素数量小于128个

  - REDIS_ENCODING_SKIPLIST(跳跃表和字典)
