# Redis

Redis 是以内存为核心的数据结构服务器，常用于缓存、会话、限流、排行榜、分布式锁、消息流和实时统计。学习 Redis 不应只停留在命令层，还要理解对象编码、事件模型、持久化、复制、集群、内存治理和缓存一致性。

## 核心脉络

- 数据模型：五种基础类型和 Stream、Bitmap、HyperLogLog、Geo 等扩展结构。
- 执行模型：单线程命令执行、IO 多路复用、后台线程和 Redis 6 之后的多线程 IO。
- 数据安全：RDB、AOF、混合持久化、复制积压缓冲区和故障恢复。
- 容量治理：过期删除、内存淘汰、碎片整理、大 key 拆分和热 key 隔离。
- 分布式能力：主从复制、Sentinel、Cluster、槽迁移和客户端重定向。
- 工程实践：缓存穿透、击穿、雪崩、一致性、分布式锁、监控和性能排查。

## 实践检查

- 能根据业务访问模式选择数据类型，而不是机械使用 String。
- 能解释 Redis 快的原因，也能说出哪些命令和场景会让 Redis 变慢。
- 能设计缓存更新流程，并说明在失败、并发和延迟场景下如何补偿。
- 能通过 `INFO`、`SLOWLOG`、`MEMORY`、`LATENCY`、`CLIENT LIST`、`redis-cli --bigkeys` 定位问题。
- 能说明 RDB、AOF、复制、Sentinel、Cluster 在可用性和一致性上的取舍。
