#### 读写间空间是的维护操作

- 1、读取一个键后，更新服务前的键空间命中次数或不命中次数

- 2、读取一个键后，跟新键的lru参数，即最后一次使用时间，以便后续计算闲置时间

- 3、读取键时发现已过期，则会删除这个过期键

- 4、如果客户端使用WATCH命令监视了某个键，那么这个键会被标记为脏

- 5、每修改一次键之后，会对脏(dirty)加1

- 6、如果开启了数据库通知功能，那么对键进行修改之后，服务器会按配置发送相应的数据库通知

#### 设置键的生存时间或过期时间(EXPIRE/PEXPIRE)

- PEXPIREAT设置过期时间的毫秒数

#### RDB对过期键处理

- 生成RDB文件
  - 忽略已过期的键

- 载入RDB文件
  - 1、主服务模式运行时，忽略已过期的键
  - 2、从服务模式运行时，载入全部的key，主从同步是会全部删除数据库

#### AOF对过期键处理

- AOF文件写入
  - 已过期的键不会产生任何影响
  - 过期键删除时，会追加一条DEL message命令到AOF文件

- AOF重写
  - 忽略过期键

#### 主从复制时

- 从服务仅依靠主服务的del命令删除过期键

### AOF持久化

#### 定义 (20)

- AOF持久化是通过保存Redis服务器所执行的写命令来记录数据库状态的

#### 实现过程

- 命令追加
  - 记录到aof_buf缓冲区的末尾

- 文件写入
  - serverCron默认每100毫秒执行一次，对正在运行的服务器进行维护

- 文件同步
  - 同步选项(appendfsync)
    - always-总是
    - everysec-超过1秒同步
    - no-由操作系统决定

#### AOF载入和数据还原

- 1、创建一个不带网络连接的为客户端

- 2、从AOF文件中分析并读取一条写命令

- 3、使用伪客户端执行被读的写命令

- 4、一致重复步骤2、3直到AOF文件的所有写命令被处理完毕

#### AOF重写

- 命令
  - BGREWEITEAOF

- 实现
  - 1、通过读取服务器当前的状态来实现
  - 2、追加重写缓冲区的内容到AOF文件的末尾
  - 3、使用新的AOF文件替换掉就得AOF文件

### 事件

#### 文件事件

- 介绍
  - Redis服务器通过套接字与客户端进行连接，而文件事件就是服务器对套接字操作的抽象。服务器与客户端通信会产生相应的文件事件，服务区通过箭筒并处理这类时间来完成一系列网络通信操作

- Reactor模式
  - 套接字
  - IO多路复用
    - select
    - epoll
    - evport
    - kqueue
  - 文件事件分派器
  - 事件处理器

- 事件类型
  - AE_READABLE
  - AE_WRITABLE

- 处理器
  - 1、连接应答处理器
  - 2、命令请求处理器
  - 3、命令回复处理器

#### 时间事件

- 介绍
  - Redis服务中的一些操作需要在给定的时间点执行，而时间事件就是服务器对这类定时操作的抽象

- 实现
  - 所有时间时间都放在一个无序链表中，每当时间时间时间执行器运行时，查找已到达的时间事件，并调用相应的事件处理器

- ServerCron(hz：每秒执行次数，默认10)
  - 1、更新服务器的各类统计信息
  - 2、清理数据库中的过期键值对
  - 3、关闭和清理连接失效的客户端
  - 4、尝试进行AOF或RDB持久化操作
  - 5、如果服务器是主服务器，那么对从服务器进行定期同步
  - 6、如果处于集群模式，对集群进行定期同步和连接测试
