# Framework 面试重点

## Spring

- IoC 核心是 BeanDefinition、BeanFactory、ApplicationContext、BeanPostProcessor 和依赖注入。
- Bean 生命周期包括实例化、属性填充、Aware 回调、初始化前后置处理器、初始化方法、销毁。
- 循环依赖依赖三级缓存解决单例 setter 注入场景，构造器循环依赖无法自动解决。
- AOP 基于代理，JDK 动态代理面向接口，CGLIB 通过子类代理，final 类或 final 方法会影响代理。
- 事务本质是 AOP 拦截方法调用，同类自调用、非 public 方法、异常被吞、事务传播配置错误都会导致事务失效。

## Spring MVC

- 请求链路：Filter -> DispatcherServlet -> HandlerMapping -> HandlerAdapter -> Controller -> 参数解析 -> 返回值处理 -> View/MessageConverter。
- 参数绑定依赖 HandlerMethodArgumentResolver、ConversionService、Formatter、Validator。
- 返回值处理依赖 HandlerMethodReturnValueHandler 和 HttpMessageConverter。
- 全局异常处理常用 `@ControllerAdvice` 和 `@ExceptionHandler`。

## Spring Boot

- 启动核心是 SpringApplication、环境准备、ApplicationContext 创建、自动配置加载和 Bean 实例化。
- 自动配置依赖 `AutoConfiguration.imports` 或旧版 `spring.factories`，通过条件注解控制装配。
- Starter 是依赖聚合和自动配置入口，不应把业务强耦合逻辑写死。
- Actuator 用于健康检查、指标、线程、堆、配置和运行期诊断，生产必须做好安全控制。

## MyBatis

- 执行链路：SqlSession -> Executor -> StatementHandler -> ParameterHandler -> JDBC -> ResultSetHandler。
- 一级缓存默认 SqlSession 级别，二级缓存 Mapper namespace 级别，生产要谨慎使用。
- 动态 SQL 要防止全表更新、SQL 注入和空条件误删。
- 插件基于 Interceptor 拦截 Executor、StatementHandler、ParameterHandler、ResultSetHandler。

## 高频追问

- Spring 为什么默认单例 Bean 不是线程安全保证：容器只保证单例实例唯一，不保证对象内部可变状态并发安全。
- `@Transactional` 为什么会失效：代理没有生效、异常类型不匹配、传播行为不符合预期、数据库引擎不支持事务。
- Spring Boot 自动配置如何排查：看 conditions report、Actuator beans/configprops/env、日志中的 auto configuration report。
