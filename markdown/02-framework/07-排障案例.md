# Framework 排障案例

## 事务未回滚

- 现象：方法抛错后数据库仍提交。
- 排查：确认是否走代理、方法是否 public、异常是否 RuntimeException/Error、是否被 catch、传播行为和数据源事务管理器是否正确。
- 修复：外部通过代理调用，显式指定 rollbackFor，避免吞异常，检查多数据源事务边界。

## Bean 循环依赖

- 现象：启动失败或注入到未完全初始化代理对象。
- 排查：查看依赖链，区分构造器注入、setter 注入、原型 Bean、AOP 代理。
- 修复：重构职责、延迟注入、事件解耦，避免把循环依赖当作正常设计。

## Spring Boot 启动慢

- 现象：应用启动时间变长。
- 排查：Bean 数量、类扫描范围、初始化方法、外部连接、自动配置、JPA/MyBatis mapper 扫描、日志和 DNS。
- 修复：缩小扫描范围，延迟非关键 Bean，异步预热，拆分重初始化逻辑。

## Spring MVC 接口 404 或参数绑定失败

- 排查：路径匹配、ContextPath、HTTP 方法、Controller 是否被扫描、参数名是否保留、JSON converter、Content-Type。
- 修复：统一路径规范，开启 `-parameters`，明确 `@RequestBody`、`@RequestParam`、`@PathVariable`。

## MyBatis 慢查询

- 排查：打印最终 SQL、参数、执行计划、索引、分页方式、N+1 查询、ResultMap 嵌套映射。
- 修复：补索引、改 SQL、批量查询、避免深分页、控制返回字段和集合大小。
