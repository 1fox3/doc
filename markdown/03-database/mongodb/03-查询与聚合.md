# MongoDB 查询与聚合

## 基本查询

MongoDB 查询由过滤条件、投影、排序、分页和执行选项组成。常见操作包括 `find`、`findOne`、`insertOne`、`updateOne`、`updateMany`、`deleteOne`、`deleteMany`。

查询设计应优先保证过滤条件可利用索引，避免在大集合上执行全表扫描、无索引排序和大 offset 分页。

## 查询条件

常用操作符：

- 比较：`$eq`、`$ne`、`$gt`、`$gte`、`$lt`、`$lte`、`$in`、`$nin`。
- 逻辑：`$and`、`$or`、`$nor`、`$not`。
- 数组：`$all`、`$elemMatch`、`$size`。
- 字段：`$exists`、`$type`。
- 正则：`$regex`，只有前缀匹配更可能利用普通索引。

`$or` 查询中每个分支都应有可用索引，否则可能退化为低效扫描。`$nin`、`$ne`、非前缀正则通常选择性较差，要谨慎用于大集合。

## 投影

投影用于限制返回字段，减少网络传输和反序列化成本。若查询字段和返回字段都被索引覆盖，可形成覆盖查询，避免读取完整文档。

注意：默认会返回 `_id`，如果不需要应显式排除：`{ _id: 0 }`。

## 排序与分页

排序字段应尽量包含在索引中。若排序无法使用索引，MongoDB 需要内存排序，大结果集可能触发性能问题。

分页建议：

- 小页数可使用 `skip + limit`。
- 大页数优先使用基于游标的 seek pagination，例如按 `createdAt` 和 `_id` 继续查询。
- 排序字段需要稳定且有索引，避免翻页重复或漏数据。

## 更新语义

常用更新操作符：

- 字段更新：`$set`、`$unset`、`$inc`、`$mul`、`$rename`。
- 数组更新：`$push`、`$addToSet`、`$pull`、`$pop`。
- 插入或更新：`upsert: true`。
- 条件数组更新：`$`、`$[]`、`$[identifier]` 配合 `arrayFilters`。

更新应尽量带上选择性高的过滤条件和索引。高并发下可以把业务版本号、状态机条件放入过滤条件，实现乐观并发控制。

## Aggregation Pipeline

聚合管道由多个 stage 组成，常见 stage：

- `$match`：过滤数据，应尽量靠前并命中索引。
- `$project`：裁剪或计算字段。
- `$group`：分组统计，可能产生大量内存消耗。
- `$sort`：排序，最好使用索引或限制输入规模。
- `$lookup`：集合间关联，适合有限数据量的查询，不应滥用为关系型 join。
- `$unwind`：展开数组，可能显著放大中间结果。
- `$limit`、`$skip`：限制结果数量。

聚合优化重点是减少进入昂贵 stage 的数据量，优先把 `$match`、可下推的 `$sort`、`$limit` 放在前面。

## 执行计划

使用 `explain()` 查看查询计划。重点字段：

- `winningPlan`：最终采用的执行计划。
- `COLLSCAN`：集合扫描，通常需要关注。
- `IXSCAN`：索引扫描。
- `FETCH`：通过索引找到文档后回表读取完整文档。
- `SORT`：内存排序，需确认数据规模。
- `keysExamined`：扫描的索引项数量。
- `docsExamined`：扫描的文档数量。
- `nReturned`：返回文档数量。

理想情况下，`keysExamined` 和 `docsExamined` 应接近 `nReturned`。若二者差距很大，说明索引选择性或索引顺序可能有问题。
