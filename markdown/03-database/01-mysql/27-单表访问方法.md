# 单表访问方法

## 访问方法速查

- `const`：主键或唯一索引等值查询单行。
- `ref`：普通索引等值查询多行。
- `ref_or_null`：普通索引等值加 `IS NULL`。
- `range`：索引范围扫描。
- `index`：全索引扫描。
- `ALL`：全表扫描。

## 索引合并

- `index_merge_intersection` 适合多个条件取交集。
- `index_merge_union` 适合多个条件取并集。
- `sort_union` 需要排序后合并，成本更高。

## 优化判断

- 小表 `ALL` 不一定需要优化，大表热点查询出现 `ALL` 要重点关注。
- 大范围 `range` 加大量回表可能比全表扫描慢。
- 高频查询优先设计复合索引，不要长期依赖索引合并。

```sql
EXPLAIN SELECT * FROM t WHERE a = 1 OR b = 2;
EXPLAIN SELECT * FROM t WHERE a = 1 AND b = 2;
```

## `type` 解读误区

- `type=range` 不一定比 `ref` 差，如果范围很窄且避免排序，性能可能很好。
- `type=index` 不是按条件精准定位，而是扫描整棵索引，数据量大时仍然危险。
- `type=ALL` 在小表或返回大部分数据时可能合理，不应机械地为所有全表扫描建索引。
- 访问方法只是单表维度，最终性能还要看连接顺序、回表、排序和锁等待。

## 改写案例

```sql
-- 不利于索引。
SELECT * FROM t WHERE a + 1 = 10;

-- 改为列和常量直接比较。
SELECT * FROM t WHERE a = 9;
```
