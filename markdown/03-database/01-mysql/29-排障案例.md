# MySQL 排障案例

## 慢 SQL

- 现象：接口慢、数据库 CPU 高、慢日志增长。
- 排查：慢日志 -> EXPLAIN -> optimizer_trace -> 索引和统计信息 -> 锁等待 -> Buffer Pool。
- 常见原因：未命中索引、回表多、排序临时表、深分页、Join 顺序不佳、返回数据过多。
- 修复：建合适联合索引、覆盖索引、改写 SQL、延迟关联、限制返回字段、分页游标化。

## 死锁

- 现象：业务偶发 `Deadlock found when trying to get lock`。
- 排查：`SHOW ENGINE INNODB STATUS` 查看 LATEST DETECTED DEADLOCK，分析两个事务持有哪些锁、等待哪些锁。
- 常见原因：访问顺序不一致、范围更新、缺索引导致扫描范围扩大、唯一索引冲突。
- 修复：统一加锁顺序、缩短事务、补索引、降低批量大小、捕获死锁后重试。

## 主从延迟

- 现象：读从库读不到刚写入数据。
- 排查：复制线程状态、relay log、Seconds_Behind_Master、主库写入峰值、大事务、从库 SQL 线程瓶颈。
- 修复：读写一致性要求高的请求走主库，拆小事务，优化从库 SQL，使用并行复制，治理大批量 DDL/DML。

## Buffer Pool 命中率低

- 现象：磁盘 IO 高、查询抖动。
- 排查：Buffer Pool size、热数据大小、全表扫描、大查询污染、预读、脏页刷新。
- 修复：增大 Buffer Pool，优化索引和 SQL，避免大范围扫描，冷热隔离。

## 连接数打满

- 现象：应用报 Too many connections。
- 排查：连接池配置、慢 SQL、事务未提交、连接泄漏、数据库 max_connections。
- 修复：修复泄漏，设置合理连接池上限和超时，优化慢 SQL，避免盲目增大连接数拖垮数据库。
