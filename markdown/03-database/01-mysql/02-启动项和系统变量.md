# 启动项和系统变量

## 配置来源

- MySQL 配置可来自命令行参数、配置文件、持久化系统变量和编译默认值。
- 常见配置文件路径包括 `/etc/my.cnf`、`/etc/mysql/my.cnf`、实例目录下的 `my.cnf`，实际加载顺序可用 `mysqld --help --verbose` 查看。
- 配置组通常包括 `[mysqld]`、`[client]`、`[mysql]`，服务端只读取 `[mysqld]` 中的服务端参数。

## 系统变量类型

- 全局变量影响新建连接或服务端行为，例如 `max_connections`、`innodb_buffer_pool_size`。
- 会话变量只影响当前连接，例如 `sql_mode`、`transaction_isolation`、`character_set_client`。
- 动态变量可通过 `SET GLOBAL` 或 `SET SESSION` 修改，静态变量必须重启实例。
- MySQL 8.0 支持 `SET PERSIST` 将部分动态变量持久化到 `mysqld-auto.cnf`。

## 常用命令

```sql
SHOW VARIABLES LIKE 'innodb_buffer_pool_size';
SHOW GLOBAL VARIABLES LIKE 'max_connections';
SHOW SESSION VARIABLES LIKE 'sql_mode';
SET SESSION transaction_isolation = 'READ-COMMITTED';
SET PERSIST max_connections = 1000;
```

## 实践要点

- 参数变更前确认作用域、是否动态、默认值、单位和版本差异。
- 不建议只在线修改关键参数，重启后会丢失；需要同步到配置文件或使用 `SET PERSIST`。
- `sql_mode`、字符集、时区、隔离级别属于容易造成应用行为不一致的会话变量，应在连接池初始化时显式设置。

## 变量生效规则

- `SET GLOBAL` 只影响之后新建的会话，不会修改已经存在的连接。
- `SET SESSION` 只影响当前连接，连接关闭后失效。
- `SET PERSIST` 会写入 `mysqld-auto.cnf`，重启后继续生效；如果配置文件和持久化文件同时设置同一变量，需要确认最终覆盖顺序。
- 部分变量只能在启动时指定，例如数据目录、端口之外的某些底层存储参数，修改前必须查询官方文档或 `performance_schema.variables_info`。

## 常见风险参数

- `max_connections` 调大后会增加内存上限，不代表数据库能承受更多并发 SQL。
- `wait_timeout` 设置过长会堆积空闲连接，设置过短可能导致连接池频繁重连。
- `sql_mode` 影响非法日期、除零、字符串截断、分组查询等行为，开发和生产不一致会引发隐蔽 bug。
- `time_zone` 影响 `TIMESTAMP` 展示和转换，跨时区业务应统一连接层时区。
- `lower_case_table_names` 与文件系统大小写相关，初始化后不应随意修改。
