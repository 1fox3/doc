# 相关工具

JVM 工具用于定位进程、线程、内存、GC、类加载、锁竞争和运行时参数问题。生产环境优先使用低侵入方式，必要时再导出堆或开启更重的采样。

## 命令行工具

| 工具 | 用途 | 常用命令 |
| --- | --- | --- |
| `jps` | 查看 Java 进程 | `jps -lv` |
| `jstat` | 查看 GC、类加载、JIT 统计 | `jstat -gcutil <pid> 1s` |
| `jinfo` | 查看或调整 JVM 参数 | `jinfo -flags <pid>` |
| `jcmd` | 综合诊断入口 | `jcmd <pid> VM.flags`、`jcmd <pid> GC.heap_info` |
| `jmap` | 堆信息和堆转储 | `jmap -dump:live,format=b,file=heap.hprof <pid>` |
| `jstack` | 线程栈、死锁、锁等待 | `jstack -l <pid>` |
| `jhsdb` | Serviceability Agent 调试 | `jhsdb jmap --pid <pid>` |
| `jfr` | Java Flight Recorder 文件分析 | `jfr summary app.jfr` |

## 可视化工具

- JDK Mission Control：分析 JFR，适合 CPU、分配、GC、锁、线程和 IO 综合诊断。
- VisualVM：查看堆、线程、采样、堆转储，适合开发和测试环境。
- MAT：分析 hprof，重点使用 Dominator Tree、Leak Suspects、Path To GC Roots 和 OQL。
- JConsole：查看 MBean、线程、内存和类加载，适合基础观测。

## Arthas

- `dashboard`：查看线程、内存、GC、运行时状态。
- `thread -n 5`：查看 CPU 占用最高的线程栈。
- `jad`：反编译已加载类。
- `sc -d`：查看类加载器、类来源和类信息。
- `watch`、`trace`、`monitor`：观察方法参数、调用耗时和调用次数。
- `heapdump`：生成堆转储。

## 典型场景

### CPU 飙高

1. 用 `top -Hp <pid>` 找到高 CPU 线程。
2. 将线程 ID 转为十六进制。
3. 用 `jstack -l <pid>` 查找对应 `nid`。
4. 结合栈顶方法判断死循环、热点计算、锁自旋或频繁 GC。

### 内存泄漏

1. 用 `jstat` 判断是堆、元空间还是 GC 压力。
2. 用 `jcmd <pid> GC.class_histogram` 快速查看对象数量。
3. 用 `jmap` 或 `jcmd GC.heap_dump` 导出 hprof。
4. 用 MAT 分析 Retained Size 和 GC Roots 引用链。

### 线程阻塞

1. 用 `jstack -l` 查看 `BLOCKED`、`WAITING`、`TIMED_WAITING`。
2. 检查死锁、锁 owner、线程池队列和外部 IO。
3. 用 JFR 的 Java Monitor Blocked、Thread Park、Socket Read/Write 事件确认阻塞来源。

### 类加载冲突

1. 用 Arthas `sc -d <class>` 查看类加载器和 CodeSource。
2. 比较同名类是否由不同 ClassLoader 加载。
3. 检查依赖版本、插件隔离、Web 容器加载顺序和 shaded 包。

## 风险

- `jmap -dump` 可能触发 STW，生产环境优先在低峰期执行。
- `jstack` 相对安全，但频繁执行仍可能增加系统负载。
- 动态修改 JVM 参数前必须确认参数是否 manageable，以及是否影响全局行为。
- 线上诊断应保留 GC 日志、JFR、线程栈和堆转储的时间线，避免只看单点快照。
