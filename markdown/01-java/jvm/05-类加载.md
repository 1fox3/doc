# 类加载

类加载把 Class 文件中的二进制字节流转换为 JVM 可使用的运行时数据结构，并在内存中生成 `java.lang.Class` 对象作为访问入口。

## 生命周期

类的生命周期包括加载、验证、准备、解析、初始化、使用和卸载。其中验证、准备、解析合称连接。

## 加载

- 通过类全限定名获取定义该类的二进制字节流，来源可以是文件、JAR、网络、动态生成或加密数据。
- 将字节流代表的静态存储结构转换为方法区运行时数据结构。
- 在堆中生成代表该类的 `Class` 对象。
- 数组类不是由 ClassLoader 直接创建，而是 JVM 在需要时创建；数组类的加载器通常与元素类型加载器一致。

## 验证

- 文件格式验证：检查魔数、版本、常量池、结构完整性。
- 元数据验证：检查继承关系、接口实现、字段和方法约束。
- 字节码验证：通过数据流和控制流分析保证操作数栈、局部变量表和跳转语义安全。
- 符号引用验证：检查类、字段、方法是否存在以及访问权限是否合法。
- `-Xverify:none` 可关闭部分验证，但会降低安全性，生产环境不建议使用。

## 准备

- 为类变量分配内存并设置默认初始值。
- 这里设置的是零值，不是 Java 代码中的显式赋值。
- `static final` 且编译期可确定的常量可能通过 `ConstantValue` 属性在准备阶段赋值。

## 解析

- 将常量池中的符号引用替换为直接引用。
- 解析对象包括类、接口、字段、类方法、接口方法、方法类型、方法句柄和动态调用点限定符。
- 解析可能在初始化前完成，也可能在运行期按需触发。

## 初始化

- 执行类构造器 `<clinit>()`，由静态变量赋值和静态代码块按源码顺序合并生成。
- JVM 保证同一个类的 `<clinit>()` 在多线程环境下只会被一个线程执行。
- 父类会先于子类初始化，接口初始化规则更弱，只有真正用到接口静态字段或默认方法相关场景才触发。

## 主动初始化场景

- 遇到 `new`、`getstatic`、`putstatic`、`invokestatic` 指令。
- 使用反射调用类。
- 初始化子类前父类尚未初始化。
- JVM 启动时初始化主类。
- 使用 `MethodHandle` 或 `VarHandle` 解析静态成员。
- 接口包含默认方法时，实现类初始化前可能需要先初始化接口。

## 类加载器

- Bootstrap ClassLoader：启动类加载器，由 JVM 实现，加载核心类库。
- Platform ClassLoader：平台类加载器，JDK 9 起替代扩展类加载器。
- Application ClassLoader：应用类加载器，加载 classpath 或 module path 上的应用类。
- 自定义 ClassLoader：常用于插件化、热部署、脚本引擎、容器隔离和加密类加载。

## 双亲委派

- `loadClass` 默认先委派父加载器加载，父加载器无法加载时再调用当前加载器的 `findClass`。
- 作用是避免核心类库被重复或恶意替换，保证类型一致性。
- 破坏双亲委派并不必然错误，典型场景包括 JDBC SPI、OSGi、Tomcat WebApp 类隔离、插件系统。
- 自定义加载器通常重写 `findClass`，不直接破坏 `loadClass`，除非确实需要改变委派顺序。

## 类相等规则

- JVM 中两个类是否相同由类全限定名和定义它的 ClassLoader 共同决定。
- 相同全限定名但不同 ClassLoader 加载的类不是同一个类型，可能导致 `ClassCastException`。

## 类卸载

类卸载需要同时满足：该类所有实例已回收、代表该类的 `Class` 对象不可达、加载该类的 ClassLoader 不可达。动态代理、脚本、热部署和插件系统需要重点关注 ClassLoader 泄漏。

## 常见错误

- `ClassNotFoundException`：运行期主动加载类失败。
- `NoClassDefFoundError`：编译期存在但运行期无法找到，或初始化失败后再次使用。
- `NoSuchMethodError`、`NoSuchFieldError`：运行时依赖版本与编译期不一致。
- `IllegalAccessError`：访问权限在运行期不满足。
- `ClassCastException`：类型不兼容，常见于不同 ClassLoader 加载同名类。
