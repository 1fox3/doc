# GC

> 专题：JVM
> 来源：面试内容汇总/JVM.xmind

## 补充与实践

- 补充：G1 是通用低停顿收集器，ZGC/Shenandoah 更适合大堆低延迟场景；CMS 已不再是新版本首选。
- 实践：GC 优化先确认目标是吞吐、停顿还是成本，再结合 GC 日志和对象分配热点调整。
- 排障：频繁 Full GC 常见原因包括老年代晋升压力、元空间膨胀、大对象分配和内存泄漏。

#### CMS

- 过程
  - 初始标记
  - 并发标记
  - 重新标记
  - 并发清除

- 优点
  - 并发收集，低停顿

- 缺点
  - CPU资源敏感
  - 无法处理浮动垃圾
  - 使用标记清除算法，产生大量的空间碎片
