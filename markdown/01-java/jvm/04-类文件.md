# 类文件

Class 文件是 JVM 的二进制接口，描述类、接口、字段、方法、常量池、属性和字节码指令。Java、Kotlin、Scala 等语言都可以编译为符合规范的 Class 文件运行在 JVM 上。

## Class 文件结构

Class 文件采用无分隔符的紧凑二进制格式，基本数据单位是 `u1`、`u2`、`u4` 和表结构。

| 字段 | 说明 |
| --- | --- |
| `magic` | 魔数，固定为 `0xCAFEBABE` |
| `minor_version` | 次版本号 |
| `major_version` | 主版本号，决定可被哪个 JVM 版本加载 |
| `constant_pool_count` | 常量池容量计数，从 1 开始 |
| `constant_pool` | 常量池，保存字面量和符号引用 |
| `access_flags` | 类或接口访问标志 |
| `this_class` | 当前类索引 |
| `super_class` | 父类索引，`java.lang.Object` 为 0 |
| `interfaces_count` | 接口数量 |
| `interfaces` | 接口索引表 |
| `fields_count` | 字段数量 |
| `fields` | 字段表 |
| `methods_count` | 方法数量 |
| `methods` | 方法表 |
| `attributes_count` | 属性数量 |
| `attributes` | 属性表 |

## 常量池

- 字面量包括字符串、整数、浮点数、长整型、双精度浮点数等。
- 符号引用包括类和接口全限定名、字段名和描述符、方法名和描述符、方法句柄、方法类型、动态调用点等。
- 类加载解析阶段会把部分符号引用解析为直接引用。

## 字段表和方法表

- 字段表包含 `access_flags`、`name_index`、`descriptor_index` 和属性集合。
- 方法表结构类似字段表，方法体字节码存放在 `Code` 属性中。
- 构造方法在字节码中名为 `<init>`，类初始化方法名为 `<clinit>`。
- Java 方法签名在 JVM 层面由方法名和描述符共同决定，返回值也在描述符中。

## 常见属性

- `Code`：保存最大操作数栈深度、局部变量表大小、字节码、异常表和子属性。
- `LineNumberTable`：源码行号到字节码偏移的映射，用于异常栈和调试。
- `LocalVariableTable`：局部变量名称、作用域和描述符，用于调试和反射辅助信息。
- `SourceFile`：源码文件名。
- `ConstantValue`：静态常量字段的常量值。
- `RuntimeVisibleAnnotations`：运行时可见注解。
- `BootstrapMethods`：`invokedynamic` 和动态常量使用的引导方法。

## 字节码指令分类

- 加载和存储：`iload`、`aload`、`istore`、`astore`。
- 运算：`iadd`、`isub`、`imul`、`idiv`、`irem`。
- 类型转换：`i2l`、`i2f`、`checkcast`、`instanceof`。
- 对象创建与访问：`new`、`newarray`、`getfield`、`putfield`、`getstatic`、`putstatic`。
- 操作数栈管理：`pop`、`dup`、`swap`。
- 控制转移：`ifeq`、`if_icmpne`、`goto`、`tableswitch`、`lookupswitch`。
- 方法调用：`invokestatic`、`invokespecial`、`invokevirtual`、`invokeinterface`、`invokedynamic`。
- 返回：`ireturn`、`areturn`、`return`。
- 异常处理：`athrow`。
- 同步：`monitorenter`、`monitorexit`。

## 版本兼容

- JVM 只保证向后兼容，低版本 JVM 不能加载高版本 Class 文件。
- `UnsupportedClassVersionError` 通常表示编译目标版本高于运行时 JVM。
- 可用 `javap -verbose Xxx.class` 查看 `major version`、常量池、方法字节码和属性。

## 排查用途

- 反编译行为异常时，用 `javap -c -v` 查看真实字节码。
- 泛型、增强 for、try-with-resources、lambda、switch 等语法糖都可通过字节码确认展开结果。
- 类冲突时，结合 Class 文件版本、CodeSource 和 ClassLoader 判断是否加载了错误版本。
