# 前端编译与优化

前端编译指把 Java 源码编译为 Class 文件的过程，核心工具是 `javac`。它主要完成语法分析、语义分析、注解处理、语法糖消解和字节码生成。

## javac 编译流程

1. 准备过程：初始化编译环境和插入式注解处理器。
2. 解析与填充符号表：词法分析、语法分析生成 AST，并建立符号表。
3. 注解处理：执行符合 JSR 269 的注解处理器，可能生成新的源文件或资源文件。
4. 语义分析：标注检查、类型检查、数据流和控制流分析。
5. 语法糖消解：把高级语法转换为较基础的语法结构。
6. 字节码生成：生成 Class 文件、方法表、字段表、属性和字节码指令。

## 注解处理器

- 标准入口是 `javax.annotation.processing.Processor`，常见基类是 `AbstractProcessor`。
- 核心方法是 `process(Set<? extends TypeElement> annotations, RoundEnvironment roundEnv)`。
- 处理器可读取注解、生成源码、生成配置，但不能直接修改已有源码 AST 标准模型。
- Lombok 通过操作编译器内部 AST 实现源码增强，属于非标准能力。

## 常见语法糖

- 泛型擦除：编译后主要保留原始类型，必要位置插入强制类型转换，泛型签名保存在属性中。
- 自动装箱和拆箱：转换为 `valueOf` 和 `xxxValue` 调用。
- 增强 for：数组转为索引循环，Iterable 转为 Iterator。
- 变长参数：编译为数组参数。
- try-with-resources：编译器生成资源关闭和 suppressed exception 处理逻辑。
- lambda：通常通过 `invokedynamic` 和 `LambdaMetafactory` 生成调用点。
- switch 字符串：转换为 hashCode 判断和 equals 校验。

## 泛型擦除细节

- Java 泛型主要在编译期保证类型安全，运行期大多数泛型类型被擦除为原始类型或上界类型。
- 编译器会在必要位置插入 `checkcast`，所以泛型错误有时表现为运行期 `ClassCastException`。
- 为保持多态，编译器可能生成桥接方法 bridge method，例如子类重写泛型父类方法时。
- 泛型签名可保存在 `Signature` 属性中，反射读取泛型信息依赖这些属性，不代表 JVM 运行期真的保留了完整泛型对象。

## Lambda 与匿名内部类区别

- 匿名内部类会生成独立类文件或内部类结构，`this` 指向匿名内部类实例。
- Lambda 中的 `this` 指向外部类实例，不引入新的 this 语义。
- Lambda 通常通过 `invokedynamic` 链接，运行期生成或复用函数式接口实例。
- 无捕获 lambda 可能复用实例，有捕获 lambda 通常需要保存捕获变量。

## try-with-resources

- 编译器会生成 `close()` 调用和异常抑制逻辑。
- 主异常和关闭资源时抛出的异常会通过 `addSuppressed` 关联。
- 多资源关闭顺序与声明顺序相反。

## 编译期常量

- `static final` 基本类型或字符串常量可能被编译进调用方 Class 文件。
- 修改常量定义但不重新编译调用方，运行时可能仍使用旧值。
- 非编译期常量需要在类初始化阶段赋值。

## 数据流和控制流分析

- 检查局部变量是否赋值后使用。
- 检查返回路径是否完整。
- 检查异常是否被捕获或声明抛出。
- `final` 变量赋值规则也在该阶段处理。

## 排查用途

- 通过 `javac -parameters` 保留方法参数名，便于反射和框架绑定。
- 通过 `javap -v` 查看泛型签名、行号表、局部变量表和注解属性。
- 注解处理器异常时，检查处理轮次、生成文件路径、processor path 和增量编译缓存。
- 面试或排障遇到“源码看不出原因”的问题时，优先反编译或 `javap` 查看语法糖消解后的真实结构。
