## 技术主线

- 类加载器读取字节码，经过加载、验证、准备、解析、初始化后生成运行时元数据。
- 字节码执行引擎基于栈帧、局部变量表、操作数栈、动态连接和返回地址执行方法调用。
- 对象通常分配在堆上，分配路径受 TLAB、逃逸分析、对象头、压缩指针和 GC 策略影响。
- 垃圾回收通过可达性分析识别存活对象，通过分代、Region、写屏障和并发标记降低回收成本。
- JIT 根据热点探测把字节码编译为机器码，并执行内联、逃逸分析、锁消除等优化。
- JMM 约束线程间可见性、有序性和原子性，是理解 `volatile`、锁、CAS 和安全发布的基础。
- 线上 JVM 问题通常要把 GC 日志、JFR、线程栈、堆转储、NMT 和容器指标放在同一条时间线上分析。

## 排障入口

- CPU 飙高：先用 `top` 定位线程，再用 `jstack` 或 Arthas `thread` 查看热点栈。
- 内存上涨：用 `jstat` 判断 GC 压力，用 `jmap`/MAT/OQL 分析堆中大对象和引用链。
- Full GC 频繁：检查老年代晋升、Humongous 对象、元空间、直接内存和对象泄漏。
- 线程阻塞：用 `jstack`、JFR 或 Arthas 定位锁竞争、死锁、等待 IO 和线程池耗尽。
- 类冲突：确认类全名和加载器是否一致，排查多版本依赖、插件隔离和容器类加载模型。
- 容器 OOMKill：同时核算堆、元空间、直接内存、线程栈、代码缓存、JIT、GC native 内存和 page cache。

## 面试复习建议

- 先掌握主线：类加载、运行时内存、对象分配、GC、JMM、JIT、排障工具。
- 每个知识点都要能回答三层：是什么、为什么这样设计、线上出问题怎么定位。
- 高频题要能串起来讲，例如“对象从 new 到回收”的完整路径会涉及类加载、对象布局、TLAB、GC Roots、分代回收和 finalize/Cleaner。
- 调优题不要直接背参数，要先说明观测、判断、验证和回滚路径。
