# 类文件

Class 文件是 JVM 的二进制接口，描述类、接口、字段、方法、常量池、属性和字节码指令。Java、Kotlin、Scala 等语言都可以编译为符合规范的 Class 文件运行在 JVM 上。

## Class 文件结构

Class 文件采用无分隔符的紧凑二进制格式，基本数据单位是 `u1`、`u2`、`u4` 和表结构。

| 字段 | 说明 | 示例值 |
| --- | --- | --- |
| `magic` | 魔数，固定为 `0xCAFEBABE` | `0xCAFEBABE` |
| `minor_version` | 次版本号 | `0` |
| `major_version` | 主版本号，决定可被哪个 JVM 版本加载 | `61`，表示 Java 17 |
| `constant_pool_count` | 常量池容量计数，从 1 开始 | `18`，表示常量池索引可到 `#17` |
| `constant_pool` | 常量池，保存字面量和符号引用 | `#1 = Methodref java/lang/Object."<init>":()V` |
| `access_flags` | 类或接口访问标志 | `ACC_PUBLIC, ACC_SUPER` |
| `this_class` | 当前类索引 | `#6 // SimpleClass` |
| `super_class` | 父类索引，`java.lang.Object` 为 0 | `#2 // java/lang/Object` |
| `interfaces_count` | 接口数量 | `0` |
| `interfaces` | 接口索引表 | 无接口时为空；实现 `Runnable` 时可能是 `#12 // java/lang/Runnable` |
| `fields_count` | 字段数量 | `1` |
| `fields` | 字段表 | `private int value; descriptor: I` |
| `methods_count` | 方法数量 | `2`，包括 `<init>` 和 `add` |
| `methods` | 方法表 | `public int add(int); descriptor: (I)I` |
| `attributes_count` | 属性数量 | `1` |
| `attributes` | 属性表 | `SourceFile: "SimpleClass.java"` |

## 常量池

- 字面量包括字符串、整数、浮点数、长整型、双精度浮点数等。
- 符号引用包括类和接口全限定名、字段名和描述符、方法名和描述符、方法句柄、方法类型、动态调用点等。
- 类加载解析阶段会把部分符号引用解析为直接引用。

### 常量池项

- `CONSTANT_Class_info`：类或接口符号引用。
- `CONSTANT_Fieldref_info`：字段符号引用。
- `CONSTANT_Methodref_info`：类方法符号引用。
- `CONSTANT_InterfaceMethodref_info`：接口方法符号引用。
- `CONSTANT_String_info`：字符串字面量引用。
- `CONSTANT_NameAndType_info`：名称和描述符组合。
- `CONSTANT_MethodHandle_info`、`CONSTANT_MethodType_info`、`CONSTANT_InvokeDynamic_info`：支持方法句柄和动态调用。

## 字段表和方法表

- 字段表包含 `access_flags`、`name_index`、`descriptor_index` 和属性集合。
- 方法表结构类似字段表，方法体字节码存放在 `Code` 属性中。
- 构造方法在字节码中名为 `<init>`，类初始化方法名为 `<clinit>`。
- Java 方法签名在 JVM 层面由方法名和描述符共同决定，返回值也在描述符中。

### 描述符示例

| Java 类型 | 描述符 |
| --- | --- |
| `int` | `I` |
| `long` | `J` |
| `double` | `D` |
| `boolean` | `Z` |
| `void` | `V` |
| `String` | `Ljava/lang/String;` |
| `int[]` | `[I` |
| `String[][]` | `[[Ljava/lang/String;` |
| `void test(int, String)` | `(ILjava/lang/String;)V` |

## 常见属性

- `Code`：保存最大操作数栈深度、局部变量表大小、字节码、异常表和子属性。
- `LineNumberTable`：源码行号到字节码偏移的映射，用于异常栈和调试。
- `LocalVariableTable`：局部变量名称、作用域和描述符，用于调试和反射辅助信息。
- `SourceFile`：源码文件名。
- `ConstantValue`：静态常量字段的常量值。
- `RuntimeVisibleAnnotations`：运行时可见注解。
- `BootstrapMethods`：`invokedynamic` 和动态常量使用的引导方法。
- `StackMapTable`：字节码验证使用的类型状态信息，加速类型检查。
- `Signature`：保存泛型签名，弥补泛型擦除后反射获取泛型信息的需要。
- `Exceptions`：记录方法声明抛出的受检异常。
- `InnerClasses`、`EnclosingMethod`：记录内部类和局部/匿名类关系。
- `NestHost`、`NestMembers`：JDK 11 起支持 nestmate 访问控制。

## 字节码验证

- 验证器通过类型状态分析保证操作数栈和局部变量表在控制流合并点类型一致。
- `StackMapTable` 让验证器不必从头推导所有控制流，提升类加载验证速度。
- 字节码增强、ASM 手写字节码和混淆工具如果生成错误帧信息，可能导致 `VerifyError`。

## 字节码指令分类

- 加载和存储：`iload`、`aload`、`istore`、`astore`。
- 运算：`iadd`、`isub`、`imul`、`idiv`、`irem`。
- 类型转换：`i2l`、`i2f`、`checkcast`、`instanceof`。
- 对象创建与访问：`new`、`newarray`、`getfield`、`putfield`、`getstatic`、`putstatic`。
- 操作数栈管理：`pop`、`dup`、`swap`。
- 控制转移：`ifeq`、`if_icmpne`、`goto`、`tableswitch`、`lookupswitch`。
- 方法调用：`invokestatic`、`invokespecial`、`invokevirtual`、`invokeinterface`、`invokedynamic`。
- 返回：`ireturn`、`areturn`、`return`。
- 异常处理：`athrow`。
- 同步：`monitorenter`、`monitorexit`。

## 简单示例

Java 源码：

```java
public class SimpleClass {
    private int value = 1;

    public int add(int x) {
        return value + x;
    }
}
```

编译后可以用 `javap -c -v SimpleClass.class` 查看 Class 文件结构。重点观察：

```text
public class SimpleClass
  minor version: 0
  major version: 61
  flags: ACC_PUBLIC, ACC_SUPER

Constant pool:
   #1 = Methodref          #2.#3   // java/lang/Object."<init>":()V
   #2 = Class              #4      // java/lang/Object
   #5 = Fieldref           #6.#7   // SimpleClass.value:I
   #6 = Class              #8      // SimpleClass
   #9 = Utf8               value
  #10 = Utf8               I
  #11 = Utf8               add
  #12 = Utf8               (I)I
```

构造方法大致字节码：

```text
public SimpleClass();
  Code:
     0: aload_0
     1: invokespecial #1  // Method java/lang/Object."<init>":()V
     4: aload_0
     5: iconst_1
     6: putfield      #5  // Field value:I
     9: return
```

`add` 方法大致字节码：

```text
public int add(int);
  descriptor: (I)I
  Code:
     0: aload_0
     1: getfield      #5  // Field value:I
     4: iload_1
     5: iadd
     6: ireturn
```

这个例子可以对应到 Class 文件几个核心部分：

- `major version: 61` 表示该 Class 文件目标版本是 Java 17，低版本 JVM 不能加载。
- `Constant pool` 中保存了 `java/lang/Object`、`SimpleClass`、字段 `value:I`、方法 `add:(I)I` 等符号引用和字面量。
- `value` 字段的描述符是 `I`，表示 `int`。
- `add(int)` 的方法描述符是 `(I)I`，表示入参是一个 `int`，返回值也是 `int`。
- 构造方法在字节码中叫 `<init>`，先调用父类 `Object.<init>`，再给实例字段 `value` 赋值。
- `aload_0` 把 `this` 压入操作数栈，`getfield` 读取实例字段，`iload_1` 读取第一个 int 参数，`iadd` 执行整数加法，`ireturn` 返回 int。
- Java 源码中的字段赋值 `private int value = 1` 并不是在准备阶段完成，而是被编译进构造方法 `<init>` 中执行。

## 版本兼容

- JVM 只保证向后兼容，低版本 JVM 不能加载高版本 Class 文件。
- `UnsupportedClassVersionError` 通常表示编译目标版本高于运行时 JVM。
- 可用 `javap -verbose Xxx.class` 查看 `major version`、常量池、方法字节码和属性。

## 排查用途

- 反编译行为异常时，用 `javap -c -v` 查看真实字节码。
- 泛型、增强 for、try-with-resources、lambda、switch 等语法糖都可通过字节码确认展开结果。
- 类冲突时，结合 Class 文件版本、CodeSource 和 ClassLoader 判断是否加载了错误版本。
- `VerifyError`、`NoSuchMethodError`、`IllegalAccessError` 通常要回到 Class 文件版本、方法描述符、访问标志和依赖版本检查。
