## 补充与实践

- 补充：BIO/NIO/AIO 的差异本质是阻塞模型、事件通知机制和线程资源使用方式不同。
- 实践：高并发网络 IO 通常选择 NIO/Reactor 框架，文件传输关注缓冲区、零拷贝和背压。
- 误区：NIO 不必然更快，小规模同步调用中 BIO 可能更简单且足够稳定。

#### InputStreamReader构造函数传入InputStream对象

### InputStream与Reader区别

#### 传输的数据类型不同，InputStream是字节流，Reader是字符流

#### InputStream仅支持ASCII编码，Reader支持更多的字符集

#### InputStream适用于读取非文本数据，例如二进制文件，Reader适用于读取文本数据

### BufferedReader和Reader区别

#### BufferReader是Reader的子类，Reader是抽象类

#### BufferReader使用缓冲区读取字符数据，并提供了读取一行数据的方法readLine，并支持一次读取大量的字符，而Reader每次只能读取一个字符

#### BufferedReader使用了缓冲区，它的读取效率比Reader高

### PrintStream和PringWriter区别

#### PrintWriter可以接收一个Writer类，PrintStream不能

#### PrintStream可以自动flush，而PrintWriter默认不能自动flush

#### PrintWriter不包含用于写入原始字节的方法，因此程序应该使用未编码的字节流

### FileWriter和BufferedWriter区别

#### 缓冲区不同

- BufferedWriter的缓冲区大小默认为8192字节，可自定义

- FIleWriter使用Writer的write方法时缓冲区为1024字节，使用StreamEncoder的write方法时8192字节

#### 效率

- BufferedWriter效率比FileWriter高

- FileWriter每次调用write方法都会调用一次OutputStreamWriter的write方法并进行一次转码

- BufferedWriter只有在缓冲区满了才会调用OutputStreamWriter的write方法，并进行一次转码

### PrintStream

#### 打印流，是一种过滤流，继承FilterOutputStream过滤类

#### 可以自动flush，永远不会抛IOException

#### System.out/error就是PrintStream打印流

### PrintWriter

#### 是PrintStream的扩展，继承了Writer，字节流

#### 默认不能自动flush，永远不会抛IOException

### 序列化

#### 方式

- Serializable

- Externalizable
  - 需要手动序列化，通过writeExternal和readExternal方法进行

#### 静态变量不能被序列化

#### serialVersionUID

- 反序列通过比较该值是否一直来验证版本是否一致，一致则反序列化成功，否则失败，并抛InvalidClassException

- 强烈建议显示声明，并赋值，但不是必须的JVM会自动生成

- 什么情况下需要修改
  - 添加或删除类中的字段
  - 修改字段的类型
  - 修改类的继承关系

#### @Serial

- 用于表示某个成员变量、方法是否为序列化机制的一部分，纯粹是为了更好的进行编译时类型检查，类似于@Override

#### transient

- 修饰的变量不能被序列化

- 只能用来修饰普通成员变量

- 只作用于Serializable接口

#### 注意点

- 序列化对象必须实现序列化接口

- 序列化对象里面的属性是对象的话也要实现序列化接口

- 类的对象序列化后，类的序列化ID不能够修改，不然反序列化失败

- 类的对象序列化后，类的成员变量类型不能够修改，不然反序列化失败

- 类的对象序列化后，类属性的增加或删除不影响序列化，只是值会丢失

- 如果父类序列化了，子类会继承父类的序列化，无序添加序列化接口

- 如果父类没有序列化，子类序列化了，子类的属性能正常序列化，但父类的属性会丢失，不能序列化

- Java序列化的二进制字节数据只能由Java反序列化

### InputStream（字节输入流）

#### 常用方法

- read()
  - 读取单字节，0-255之间，-1表示文件结束

- read(byte b[])
  - 读取一些字节存储到数组b中。b的长度为0，则不读取
  - 返回读取的字节数，-1表示无字节可读

- read(byte b[], int off, int len)
  - 在read（byte b[]）的基础上增加了off（偏移量）和len（要读取的最大字节数）

- skip(long n)
  - 忽略输入的n个字节，返回实际忽略的字节数

- available()
  - 返回可读取的字节数

- close()
  - 关闭输入流释放相关资源

- Java9增加如下方法

- readAllBytes()
  - 读取所有字节，返回字节数组

- readNBytes(byte[] b, int off, int len)
  - 阻塞直到读取len个字节

- transferTo(OutputStream out)
  - 将所有字节从一个输入流传递到一个输出流
