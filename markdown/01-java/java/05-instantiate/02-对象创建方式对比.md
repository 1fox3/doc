# 对象创建方式对比

## 常见方式

| 方式 | 是否调用构造方法 | 典型场景 |
| --- | --- | --- |
| `new` | 是 | 普通业务对象创建 |
| 反射构造 | 是 | 框架、容器、插件化加载 |
| `clone()` | 否 | 复制已有对象，较少推荐 |
| 反序列化 | 否，特殊规则除外 | 从持久化字节恢复对象 |
| `MethodHandle` | 通常是 | 高性能动态调用场景 |
| `Unsafe.allocateInstance` | 否 | 底层框架，极少直接使用 |

## 反射实例化

现代 Java 推荐使用 `Constructor#getDeclaredConstructor`，不推荐 `Class#newInstance`，因为后者只能调用无参构造，异常语义也不清晰。

```java
import java.lang.reflect.Constructor;

public class ReflectionInstantiateDemo {
    static class User {
        private final String name;

        private User(String name) {
            this.name = name;
        }

        String name() {
            return name;
        }
    }

    public static User createUser(String name) throws ReflectiveOperationException {
        Constructor<User> constructor = User.class.getDeclaredConstructor(String.class);
        constructor.setAccessible(true);
        return constructor.newInstance(name);
    }
}
```

## clone 创建对象

`clone()` 会复制已有对象的字段，不会执行构造方法。默认是浅拷贝，引用类型字段仍然指向同一个对象。

```java
public class CloneInstantiateDemo {
    static class User implements Cloneable {
        private String name;

        User(String name) {
            this.name = name;
        }

        @Override
        protected User clone() throws CloneNotSupportedException {
            return (User) super.clone();
        }
    }

    public static User copy(User source) throws CloneNotSupportedException {
        return source.clone();
    }
}
```

## 反序列化创建对象

反序列化会绕过可序列化类本身的构造方法，从字节流恢复字段状态。对于没有实现 `Serializable` 的父类，反序列化子类时会调用该父类的无参构造方法。

```java
import java.io.ByteArrayInputStream;
import java.io.ByteArrayOutputStream;
import java.io.IOException;
import java.io.ObjectInputStream;
import java.io.ObjectOutputStream;
import java.io.Serial;
import java.io.Serializable;

public class DeserializationInstantiateDemo {
    static class User implements Serializable {
        @Serial
        private static final long serialVersionUID = 1L;

        private final String name;

        User(String name) {
            this.name = name;
        }
    }

    public static User copy(User user) throws IOException, ClassNotFoundException {
        ByteArrayOutputStream bytes = new ByteArrayOutputStream();
        try (ObjectOutputStream out = new ObjectOutputStream(bytes)) {
            out.writeObject(user);
        }

        try (ObjectInputStream in = new ObjectInputStream(new ByteArrayInputStream(bytes.toByteArray()))) {
            return (User) in.readObject();
        }
    }
}
```

## MethodHandle 调用构造方法

`MethodHandle` 是比反射更底层、更接近 JVM 调用语义的动态调用机制，适合高频动态调用场景，但代码复杂度更高。

```java
import java.lang.invoke.MethodHandle;
import java.lang.invoke.MethodHandles;
import java.lang.invoke.MethodType;

public class MethodHandleInstantiateDemo {
    static class User {
        private final String name;

        User(String name) {
            this.name = name;
        }
    }

    public static User create(String name) throws Throwable {
        MethodHandles.Lookup lookup = MethodHandles.lookup();
        MethodHandle constructor = lookup.findConstructor(
                User.class,
                MethodType.methodType(void.class, String.class));
        return (User) constructor.invoke(name);
    }
}
```

## 选择建议

- 业务代码优先使用 `new` 或工厂方法，简单、类型安全、可读性好。
- 框架需要根据配置或类名创建对象时使用反射构造。
- 不建议为了“性能”在业务代码中引入 `Unsafe` 或复杂 `MethodHandle`，除非有明确压测和边界控制。
- `clone` 和 JDK 原生序列化都有明显设计成本，生产代码要谨慎使用。
