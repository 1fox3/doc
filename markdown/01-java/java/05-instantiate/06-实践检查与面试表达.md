# 实践检查与面试表达

## 常见误区

| 误区 | 正确理解 |
| --- | --- |
| 实例化等于初始化完成 | 实例化只是创建对象，后续还有属性填充、初始化回调、后置处理等 |
| 有方法覆盖就是 Java 子类重写 | 这里通常指 Spring BeanDefinition 的 `methodOverrides` |
| CGLIB 只用于 AOP | Spring 方法注入也会使用 CGLIB 创建子类 |
| 反射创建对象一定很慢 | 反射有成本，但框架会缓存构造方法，实际瓶颈要看场景和压测 |
| `@Lookup` 是普通方法调用 | Spring 会生成子类覆盖该方法，从容器动态获取 Bean |

## 排查方向

| 现象 | 可能原因 | 排查方式 |
| --- | --- | --- |
| Bean 创建失败 | 构造方法不可访问、参数无法解析、循环依赖 | 查看异常链中的 `BeanCreationException` 和 `UnsatisfiedDependencyException` |
| `@Lookup` 不生效 | 方法为 `final`、`private`，类为 `final`，或 Bean 没被 Spring 管理 | 检查方法签名和 BeanDefinition |
| 原型 Bean 在单例中只创建一次 | 直接注入原型 Bean | 改用 `ObjectProvider`、`@Lookup` 或代理作用域 |
| AOP 失效 | final 方法、内部自调用、对象不是容器 Bean | 检查代理类型和调用路径 |
| 类型判断异常 | CGLIB 子类或 AOP 代理改变运行时类型 | 使用接口编程，必要时检查 `AopUtils.getTargetClass` |

## 面试回答模板

回答 Spring Bean 实例化策略时，可以按以下顺序展开：

1. Spring 创建 Bean 时会先根据 `BeanDefinition` 选择构造方法、工厂方法或普通无参构造。
2. 真正实例化由 `InstantiationStrategy` 完成，默认实现是 `SimpleInstantiationStrategy`。
3. 如果 `RootBeanDefinition` 没有 `methodOverrides`，就直接通过构造方法反射实例化。
4. 如果存在 `lookup-method` 或 `replaced-method`，需要生成 CGLIB 子类覆盖指定方法，因此走 `CglibSubclassingInstantiationStrategy`。
5. 实例化后还会进行依赖注入、初始化和 BeanPostProcessor，AOP 代理通常不是实例化阶段完成的。

## 简洁版回答

`instantiate` 在 Spring 里主要是 Bean 创建早期的实例化步骤。无方法覆盖时，Spring 直接通过构造方法或工厂方法反射创建对象；如果 BeanDefinition 中有 `lookup-method` 或 `replaced-method`，Spring 需要用 CGLIB 生成目标类子类，在子类中覆盖指定方法，所以会走 CGLIB 实例化策略。实例化完成后 Bean 还要经过属性填充、初始化和后置处理，不能等同于完整 Bean 创建完成。

## Demo：单例获取原型对象

推荐方式是显式注入 `ObjectProvider`：

```java
import org.springframework.beans.factory.ObjectProvider;
import org.springframework.context.annotation.Scope;
import org.springframework.stereotype.Component;

@Component
public class JobService {
    private final ObjectProvider<JobContext> contextProvider;

    public JobService(ObjectProvider<JobContext> contextProvider) {
        this.contextProvider = contextProvider;
    }

    public void run(String jobId) {
        JobContext context = contextProvider.getObject();
        context.setJobId(jobId);
        context.execute();
    }
}

@Component
@Scope("prototype")
class JobContext {
    private String jobId;

    void setJobId(String jobId) {
        this.jobId = jobId;
    }

    void execute() {
        System.out.println("run " + jobId + " with " + System.identityHashCode(this));
    }
}
```

如果面试官明确问 `lookup-method`，再补充 CGLIB 方法覆盖；如果是工程实践，优先说明 `ObjectProvider` 更可读、更易测试。
