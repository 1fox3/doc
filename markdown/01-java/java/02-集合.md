- 补充：集合问题应从数据结构、扩容策略、迭代一致性、线程安全和内存占用五个角度回答。

#### 在JDK1.5时被引入，总是优先级最高的元素出队

#### 使用二叉堆的数据结构来实现，底层使用可变长数组来存储数据

#### 非线程安全，不支持存储NULL和non-comparable的对象

#### 默认小顶堆，可接受一个Comparator作为构造参数，从来实现大顶堆

### HashMap和HashTable的区别

#### 线程安全 (66)

- HashMap非线程安全

- HashTable线程安全

#### 效率

- HashMap效率比HashTable高

#### 对NULL的支持

- HashMap支持null为key或值，只是null作为key只能存在一个

- HashTable不支持null，会抛NullPointerException

#### 初始容量和每次扩容大小

- HashMap默认大小为16，每次扩容变为原来的2倍，如果创建时指定了大小，则扩容为不小于指定大小的2的次幂大小

- HashTable默认初始大小为11，每次扩容为原来的2n + 1

#### 底层数据结构 (70)

- HashMap:数组+链表+红黑树

- HashTable：数组+ 链表

### ConcurrentHashMap (30)

#### JDK7和JDK8的区别

- 线程安全实现方式
  - JDK7采用Segment分段锁来保证安全，Segment是继承自ReentrantLock
  - JDK8采用Node+CAS+synchronized保证线程安全，所粒度更细，而且synchronized仅锁定链表或红黑树的首节点

- Hash碰撞解决方法
  - JDK7采用拉链法
  - JDK8采用拉链发结合红黑树

- 并发数
  - JDK7最大并发度为Segment的个数，默认16个
  - JDK8最大并发数为Node数组的大小
