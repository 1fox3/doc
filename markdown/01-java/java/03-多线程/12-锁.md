## 锁

### 分类

#### synchronized

- 用法
  - 普通方法
  - 静态方法
  - 类
  - 同步this实例
  - 同步对象实例

- JDK6优化
  - 锁粗化
  - 锁消除
  - 锁升级

- 也是一种可重入锁

- 锁升级
  - 无锁
    - 偏向锁
      - 轻量级锁
        - 重量级锁
        - 竞争升级
      - 存在多个线程竞争不激烈的执行
    - 一个线程执行同步块，无竞争

#### ReentrantLock

- 可重入锁

#### ReentrantReadWriteLock

- 实现ReadWriteLock接口

- 内部维护一个读锁，一个写锁

- 允许多个线程同时读，或一个线程写

- 读写互斥

#### StampedLock

- JDK8开始提供

- 支持乐观读

- 支持多个读线程和一个写线程

- 注意点
  - 不可重入
  - 不支持Condition
  - 使用时不要中断线程，会导致CPU飙升到100%

### synchronized和ReentrantLock区别

#### synchronized依赖于JVM实现的，ReentrantLock时JDK自身实现的

#### synchronized使用简单，可自动释放锁，ReentrantLock需要手动释放锁

#### ReentrantLock锁粒度比synchronized小

#### ReentrantLock支持公平锁，synchronized仅支持非公平锁

#### ReentrantLock提供了一个Condition类，锁更灵活
