## 详细说明

### 核心区别

- `execute` 提交 Runnable，没有返回 Future，异常通常由线程的异常处理器处理。
- `submit` 返回 Future，任务异常会封装在 Future 中，调用 `get` 时抛出。

### 实践建议

- 如果使用 submit 但从不调用 Future#get，异常可能不明显，需要统一 afterExecute 或日志处理。
