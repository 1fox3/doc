## 线程中断

### 相关方法

#### Thread.interrupt

- 中断目标线程，给目标线程发一个中断信号，线程被打上中断标记

#### Thread.isInterrupted

- 判断目标线程是否被中断，不会清除中断标记

#### Thread.interrupted

- 判断目标线程是否被中断，会清除中断标记

### sleep和wait区别

#### 使用限制

- sleep方法可以让当前线程西面，时间一到当前线程继续之心，任何地方都能使用，需捕获InterruptedException异常

- wait方法必须放在synchronized块里，同样需要捕获InterruptedException，需要配合notify进行唤醒，也可指定休眠实现

#### 使用场景

- sleep一般用于当前线程西面，或者轮询操作暂停

- wait多用于多线程之间的通信

#### 所属类

- sleep属于Thread

- wait属于Object

#### 释放锁

- sleep不会释放锁

- wait可以释放当前线程对lock对象锁的持有

#### 线程切换

- sleep会强制上下文切换

- wait则不一定，wait后可能还有机会重新竞争到锁继续执行

### yield

#### Thread类的方法

#### 当前线程让出CPU的时间片，并变为就绪状态，重新竞争CPU的调度权。可能会获取到，也可能会被其他线程获取到

### yield与sleep区别

#### 相同点

- 暂停当前线程

- 暂停过程中如果已经持有锁，都不会释放锁资源

#### 不同点

- sleep可以制定具体的休眠时间，yield则依赖CPU的时间片划分

- yield不能被中断，而sleep则可以接受中断
