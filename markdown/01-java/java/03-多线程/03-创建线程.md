# 创建线程

## 概述

Java 提供四种主流方式创建和管理线程：继承 `Thread` 类、实现 `Runnable` 接口、使用 `Callable` + `Future` 获取异步返回值、以及通过 `ExecutorService` 线程池统一管理线程生命周期。

前两种方式任务无返回值，适合"执行后不关心结果"的场景；`Callable` 引入了有返回值和受检异常传播能力；线程池则是生产环境的标准做法，解决线程频繁创建销毁的开销问题，同时提供任务队列、拒绝策略和资源上限控制。

## 核心概念

### 继承 Thread 类

直接继承 `Thread` 并重写 `run()` 方法。调用 `start()` 启动新线程，不能调用 `run()`（那只是普通方法调用，不会开新线程）。

缺点：Java 单继承限制，继承了 `Thread` 就无法再继承其他类，耦合性高。

### 实现 Runnable 接口

实现 `Runnable` 的 `run()` 方法，将其传入 `Thread` 构造器。任务逻辑与线程对象解耦，可配合线程池使用，是更推荐的写法。

`Runnable` 是函数式接口（`@FunctionalInterface`），支持 Lambda 表达式。`run()` 不能抛受检异常，异常必须在内部处理或转为运行时异常。

### Runnable 与 Callable 对比

| 维度 | Runnable | Callable\<V\> |
|---|---|---|
| 方法名 | `run()` | `call()` |
| 返回值 | `void` | 泛型 `V` |
| 受检异常 | 不能直接抛出 | 声明 `throws Exception` |
| 线程池提交 | `execute()` / `submit()` | `submit()` |
| 结果获取 | 无 | 通过 `Future.get()` |

`Runnable` 适合执行后不关心结果的任务；`Callable` 适合需要返回结果、传递受检异常或由调用方统一处理任务失败的场景。`FutureTask` 同时实现 `Runnable` 和 `Future`，是把 `Callable` 交给 `Thread` 或线程池执行的桥梁。

### Callable + Future

`Callable<V>` 的 `call()` 方法有返回值，并且可以抛出受检异常。需要配合 `FutureTask` 或 `ExecutorService.submit()` 使用。

`Future<V>` 表示异步计算的结果，核心方法：
- `get()`：阻塞等待结果，可能抛 `ExecutionException`（包装了 `call()` 内的异常）
- `get(timeout, unit)`：带超时的等待，超时抛 `TimeoutException`
- `isDone()`：非阻塞检查是否完成
- `cancel(true)`：尝试中断正在执行的任务

注意：`isDone()` 只表示任务进入终态，不代表任务成功；异常完成、取消完成也会返回 `true`。任务内部异常需要通过 `get()` 抛出的 `ExecutionException.getCause()` 获取。

### 线程池 ExecutorService

`ThreadPoolExecutor` 是线程池的核心实现，七个关键参数：

| 参数 | 含义 |
|---|---|
| `corePoolSize` | 核心线程数，即使空闲也不销毁 |
| `maximumPoolSize` | 最大线程数上限 |
| `keepAliveTime` | 非核心线程空闲存活时间 |
| `unit` | 时间单位 |
| `workQueue` | 任务队列（ArrayBlockingQueue、LinkedBlockingQueue 等） |
| `threadFactory` | 线程工厂，可自定义线程名 |
| `handler` | 拒绝策略（AbortPolicy、CallerRunsPolicy 等） |

任务提交流程：核心线程未满 -> 直接创建线程执行；核心线程已满 -> 放入队列；队列已满且线程数未达上限 -> 创建非核心线程；队列和线程数均已满 -> 触发拒绝策略。

`Executors` 工厂方法（如 `newFixedThreadPool`、`newCachedThreadPool`）在生产环境中不推荐使用，因为其内部使用无界队列或无上限线程数，容易导致 OOM。

## 示例代码

```java
// 方式一：继承 Thread
class MyThread extends Thread {
    @Override
    public void run() {
        System.out.println("Thread name: " + Thread.currentThread().getName());
    }
}

// 方式二：实现 Runnable（Lambda 写法）
Runnable task = () -> System.out.println("Runnable running in: " + Thread.currentThread().getName());

// 启动
new MyThread().start();
new Thread(task, "my-runnable-thread").start();
```

```java
// 方式三：Callable + FutureTask（不借助线程池）
Callable<Integer> callable = () -> {
    Thread.sleep(100);
    return 42;
};

FutureTask<Integer> futureTask = new FutureTask<>(callable);
new Thread(futureTask, "callable-thread").start();

try {
    Integer result = futureTask.get(1, TimeUnit.SECONDS); // 阻塞等待
    System.out.println("Result: " + result);
} catch (TimeoutException e) {
    futureTask.cancel(true);
    System.err.println("Task timed out");
} catch (ExecutionException e) {
    System.err.println("Task threw exception: " + e.getCause());
}
```

```java
// 方式四：线程池（推荐生产写法）
ThreadPoolExecutor executor = new ThreadPoolExecutor(
    4,                                      // corePoolSize
    8,                                      // maximumPoolSize
    60L, TimeUnit.SECONDS,                  // keepAliveTime
    new ArrayBlockingQueue<>(200),          // 有界队列，防止 OOM
    new ThreadFactoryBuilder()              // 自定义线程名（Guava）
        .setNameFormat("biz-pool-%d")
        .build(),
    new ThreadPoolExecutor.CallerRunsPolicy() // 拒绝时由调用线程执行，起限流作用
);

// 提交无返回值任务
executor.execute(() -> System.out.println("fire and forget"));

// 提交有返回值任务
Future<String> future = executor.submit(() -> {
    // 模拟业务处理
    return "processed result";
});

String result = future.get();

// 关闭线程池（优雅关闭）
executor.shutdown();
if (!executor.awaitTermination(30, TimeUnit.SECONDS)) {
    executor.shutdownNow(); // 超时后强制中断
}
```

```java
// CompletableFuture（Java 8+，异步编排更灵活）
CompletableFuture<String> cf = CompletableFuture
    .supplyAsync(() -> fetchData(), executor)         // 异步执行
    .thenApply(data -> transform(data))               // 结果转换
    .exceptionally(ex -> "fallback on error: " + ex.getMessage()); // 异常兜底

String value = cf.get();
```

## 常见问题

**1. 调用 run() 而不是 start()**

`thread.run()` 在当前线程同步执行任务逻辑，不会创建新线程，也不会报错，只会表现为单线程行为。必须调用 `start()` 才能启动新线程。

**2. 线程池使用无界队列导致 OOM**

`Executors.newFixedThreadPool` 内部使用 `LinkedBlockingQueue`（无界），当任务堆积时队列无限增长，最终耗尽堆内存。生产环境应使用有界队列（`ArrayBlockingQueue`）并配置合理的拒绝策略。

**3. Future.get() 阻塞主线程**

`future.get()` 是阻塞调用。如果任务出现死循环或永久阻塞，主线程也会被卡住。应始终使用 `get(timeout, unit)` 设置超时，并在捕获 `TimeoutException` 后调用 `future.cancel(true)` 中断任务。

**4. ExecutionException 包装异常丢失**

`Callable.call()` 内部抛出的异常会被包装在 `ExecutionException` 中，调用 `future.get()` 时才会抛出。需要通过 `e.getCause()` 取到原始异常，否则日志中只能看到 `ExecutionException` 而无法定位根因。

**5. FutureTask 的结果只计算一次**

`FutureTask` 内部有状态机，`call()` 执行完成后结果会被缓存。之后多次调用 `get()` 都直接返回同一个结果，不会重新执行任务。若需要重复执行，必须创建新的 `FutureTask` 实例。
