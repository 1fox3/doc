## 详细说明

### 核心概念

- `@Repeatable` 允许同一个注解在同一元素上重复使用。
- 需要定义一个容器注解来承载多个重复注解。

### 典型场景

- 同一方法配置多条规则、多个权限、多个路由或多个校验条件。

### 复习检查

- 能用自己的话解释 `@Repeatable`，而不是只复述标题。
- 能说出至少一个生产场景、一个常见坑点和一个排查方向。
