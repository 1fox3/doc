## 详细说明

### 核心区别

- 抽象类表达 is-a 关系，适合沉淀公共状态和公共实现。
- 接口表达 can-do 能力，适合定义行为契约和扩展点。
- Java 类只能继承一个父类，但可以实现多个接口。

### 成员能力

- 抽象类可以有成员变量、构造器、普通方法、抽象方法。
- 接口可以有抽象方法、默认方法、静态方法、私有方法，字段默认是 `public static final`。

### 选择建议

- 需要共享状态或模板方法时用抽象类。
- 需要定义能力、插件扩展、跨层契约时用接口。
- 面向框架设计时，接口通常更利于解耦和测试。
