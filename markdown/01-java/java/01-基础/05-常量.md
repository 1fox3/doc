# 常量

## 分类对比

| 类型 | 定义位置 | 修饰方式 | 归属 | 典型场景 |
|------|----------|----------|------|----------|
| 静态常量 | 类中、方法外 | `public/private static final` | 类本身，所有实例共享 | 全局配置、错误码、常量 key |
| 成员常量 | 类中、方法外 | `final`（无 `static`） | 对象实例，初始化后不可修改 | 创建时间、业务编号、初始化配置 |
| 局部常量 | 方法/构造器/代码块内 | `final` | 当前作用域 | 方法内不可重新赋值的临时变量、Lambda 捕获变量 |

## 注意点

- `final` 只保证引用不可变，不保证引用对象的内部状态不可变。
- 基本类型和 `String` 的 `static final` 常量若在编译期可确定，编译器会将其内联到使用方；常量变更后若使用方未重新编译，仍会使用旧值。

## 示例

```java
public class ConstantDemo {
    // 静态常量：属于类本身，所有对象共享
    public static final String SYSTEM_NAME = "order-service";

    // 成员常量：属于对象实例，每个对象初始化后不可变
    private final String orderNo;

    public ConstantDemo(String orderNo) {
        this.orderNo = orderNo;
    }

    public void printOrder() {
        // 局部常量：只在当前方法内有效，赋值后不能再改
        final int maxRetryTimes = 3;
        System.out.println("system=" + SYSTEM_NAME);
        System.out.println("orderNo=" + orderNo);
        System.out.println("maxRetryTimes=" + maxRetryTimes);
    }
}
```
