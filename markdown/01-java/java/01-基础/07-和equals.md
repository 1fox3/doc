# ==和equals

## 核心区别

| | `==` | `equals` |
|--|------|----------|
| 基本类型 | 比较数值 | 不适用 |
| 引用类型 | 比较对象引用地址 | 默认同 `==`，可重写为按字段/业务逻辑判断相等 |

## 典型场景

- `String` 重写了 `equals`，比较字符串内容应使用 `equals`，不用 `==`。
- 包装类型（如 `Integer`）在 `-128~127` 范围内缓存复用对象，超出范围 `==` 会返回 `false`，不可用 `==` 判断数值相等。
- 枚举可安全使用 `==`，因为枚举实例天然单例且类型安全。

## 注意点

- 字符串常量与变量比较建议写 `"OK".equals(value)`，避免 `value` 为 `null` 时 NPE。
- 业务对象重写 `equals` 时必须同时重写 `hashCode`，保持一致。
