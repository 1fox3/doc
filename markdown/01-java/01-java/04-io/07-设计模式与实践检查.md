# 设计模式与实践检查

## IO 中的设计模式

| 模式 | 体现 | 说明 |
| --- | --- | --- |
| 装饰器模式 | `BufferedInputStream` 包装 `FileInputStream` | 动态增强已有流能力 |
| 适配器模式 | `InputStreamReader`、`OutputStreamWriter` | 在字节流和字符流接口之间转换 |
| 工厂方法 | `Files.newInputStream`、`Files.newBufferedReader` | 由工具类统一创建具体流对象 |
| 观察者思想 | `WatchService` 和 `Watchable` | 目录把事件注册到监听服务，消费端获取变化事件 |

## 常见问题

| 问题 | 原因 | 处理方式 |
| --- | --- | --- |
| 中文乱码 | 读写两端字符集不一致，或依赖平台默认编码 | 显式使用 `StandardCharsets.UTF_8` |
| 文件句柄泄漏 | 流未关闭 | 使用 `try-with-resources` |
| 大文件 OOM | 使用 `readAllBytes`、`readString`、`readAllLines` 一次性加载 | 使用 `BufferedReader` 或 Channel 分块处理 |
| 写入后文件为空或不完整 | 缓冲未刷新或程序异常退出 | 正确 `close`，关键位置显式 `flush` |
| `available()` 判断文件长度错误 | 它只表示可无阻塞读取的估计字节数 | 用 `Files.size` 或循环读取返回值 |
| 反序列化失败 | `serialVersionUID` 或类结构不兼容 | 显式维护版本并设计兼容策略 |

## API 选择建议

| 场景 | 推荐 API |
| --- | --- |
| 小文本文件一次性读取 | `Files.readString(path, UTF_8)` |
| 大文本文件逐行处理 | `Files.newBufferedReader(path, UTF_8)` |
| 二进制文件复制 | `Files.copy` 或 `InputStream.transferTo` |
| 频繁小块写文本 | `BufferedWriter` |
| 格式化输出文本 | `PrintWriter` |
| 高并发网络 IO | Netty 或 NIO Selector |
| 对象本地临时持久化 | 谨慎使用 JDK 序列化，优先考虑 JSON、Protobuf 等明确协议 |

## 面试检查清单

- 能解释四个基础抽象类：`InputStream`、`OutputStream`、`Reader`、`Writer`。
- 能说明字节流和字符流的边界：字符流依赖编码，字节流不处理字符集。
- 能说清节点流与过滤流的关系，并举出装饰器例子。
- 能解释 `flush` 和 `close` 的差异。
- 能说明 `PrintStream` 与 `PrintWriter` 的区别和异常处理特点。
- 能说明 `Serializable`、`serialVersionUID`、`transient`、`static` 字段的序列化行为。
- 能描述 NIO 中 `Buffer`、`Channel`、`Selector` 的职责和多路复用流程。
- 能结合场景选择 BIO、NIO 或更高层框架，而不是简单背诵“NIO 更快”。

## 综合 Demo

下面的例子展示一个常见实践：读取 UTF-8 文本文件，过滤空行，写出到另一个文件，并确保父目录存在。

```java
import java.io.BufferedReader;
import java.io.BufferedWriter;
import java.io.IOException;
import java.nio.charset.StandardCharsets;
import java.nio.file.Files;
import java.nio.file.Path;

public class IoPipelineDemo {
    public static void copyNonBlankLines(Path source, Path target) throws IOException {
        Path parent = target.getParent();
        if (parent != null) {
            Files.createDirectories(parent);
        }

        try (BufferedReader reader = Files.newBufferedReader(source, StandardCharsets.UTF_8);
             BufferedWriter writer = Files.newBufferedWriter(target, StandardCharsets.UTF_8)) {
            String line;
            while ((line = reader.readLine()) != null) {
                if (line.isBlank()) {
                    continue;
                }
                writer.write(line);
                writer.newLine();
            }
        }
    }
}
```
