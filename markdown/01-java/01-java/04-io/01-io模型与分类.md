# IO 模型与分类

## 基本概念

Java IO 解决的是数据从一个端点流向另一个端点的问题。端点可以是文件、网络连接、内存数组、进程管道或对象序列化流。

## 按数据流向分类

| 类型 | 含义 | 常见抽象类 |
| --- | --- | --- |
| 输入流 | 从外部数据源读入程序 | `InputStream`、`Reader` |
| 输出流 | 从程序写出到外部目标 | `OutputStream`、`Writer` |

输入与输出是以程序为视角：从文件读入程序是输入，将程序数据写入文件是输出。

## 按传输单位分类

| 类型 | 单位 | 适用场景 | 常见类 |
| --- | --- | --- | --- |
| 字节流 | `byte` | 图片、音频、压缩包、网络报文、任意二进制数据 | `InputStream`、`OutputStream` |
| 字符流 | `char` | 文本数据，需要字符集编解码 | `Reader`、`Writer` |

字符流不是直接替代字节流。磁盘和网络最终传输的仍然是字节，字符流只是通过编码转换把字节解释成字符，或把字符编码成字节。

## 按功能分类

| 类型 | 含义 | 例子 |
| --- | --- | --- |
| 节点流 | 直接连接数据源或目标 | `FileInputStream`、`FileOutputStream`、`FileReader`、`FileWriter` |
| 过滤流 | 包装已有流并增强能力 | `BufferedInputStream`、`DataInputStream`、`ObjectInputStream`、`PrintStream` |

过滤流通常体现装饰器模式，例如用 `BufferedInputStream` 包装 `FileInputStream`，在不改变文件输入能力的情况下增加缓冲能力。

```java
import java.io.BufferedInputStream;
import java.io.FileInputStream;
import java.io.IOException;
import java.io.InputStream;
import java.nio.file.Path;

public class DecoratedStreamDemo {
    public static long countBytes(Path path) throws IOException {
        try (InputStream in = new BufferedInputStream(new FileInputStream(path.toFile()))) {
            long count = 0;
            while (in.read() != -1) {
                count++;
            }
            return count;
        }
    }
}
```

## BIO、NIO、AIO

| 模型 | 关键特点 | 适用场景 |
| --- | --- | --- |
| BIO | 阻塞 IO，一个连接通常绑定一个线程或线程池任务 | 连接数少、逻辑简单、同步调用 |
| NIO | 非阻塞通道加多路复用，一个线程可监听多个连接事件 | 高并发网络服务、Reactor 框架 |
| AIO | 异步 IO，提交操作后通过回调或 `Future` 获取结果 | 异步文件或网络 IO，实际使用频率低于 NIO |

NIO 的优势主要来自事件多路复用和更少的线程阻塞，不是“API 名字更新所以一定更快”。如果连接数少、业务处理同步且简单，BIO 往往更容易维护。
