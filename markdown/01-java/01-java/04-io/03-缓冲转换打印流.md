# 缓冲流、转换流与打印流

## 缓冲流

缓冲流通过内存缓冲区减少底层读写次数，尤其适合频繁小块读写。

| 类型 | 类 | 默认缓冲大小 |
| --- | --- | --- |
| 字节输入缓冲 | `BufferedInputStream` | 8192 字节 |
| 字节输出缓冲 | `BufferedOutputStream` | 8192 字节 |
| 字符输入缓冲 | `BufferedReader` | 8192 字符 |
| 字符输出缓冲 | `BufferedWriter` | 8192 字符 |

`BufferedReader` 提供 `readLine()`，适合按行处理文本。`BufferedWriter` 提供 `newLine()`，会根据系统行分隔符写入换行。

```java
import java.io.BufferedReader;
import java.io.IOException;
import java.nio.charset.StandardCharsets;
import java.nio.file.Files;
import java.nio.file.Path;

public class BufferedReaderDemo {
    public static long countNonBlankLines(Path path) throws IOException {
        long count = 0;
        try (BufferedReader reader = Files.newBufferedReader(path, StandardCharsets.UTF_8)) {
            String line;
            while ((line = reader.readLine()) != null) {
                if (!line.isBlank()) {
                    count++;
                }
            }
        }
        return count;
    }
}
```

## 转换流

转换流用于连接字节世界和字符世界：

| 类 | 方向 | 说明 |
| --- | --- | --- |
| `InputStreamReader` | 字节输入流到字符输入流 | 解码：`byte` -> `char` |
| `OutputStreamWriter` | 字符输出流到字节输出流 | 编码：`char` -> `byte` |

关键原则：文本 IO 应显式指定 `Charset`，避免依赖平台默认编码导致跨机器乱码。

## 打印流

| 类 | 类型 | 常见用途 |
| --- | --- | --- |
| `PrintStream` | 字节打印流，继承 `FilterOutputStream` | `System.out`、`System.err` |
| `PrintWriter` | 字符打印流，继承 `Writer` | 文本文件、HTTP 响应、模板输出 |

共同特点：

- 提供 `print`、`println`、`printf`、`format` 等便捷方法。
- 普通写入异常不会直接抛出 `IOException`，需要通过 `checkError()` 检查错误状态。
- 可在构造时开启 `autoFlush`，但只有调用 `println`、`printf` 或 `format` 等方法时才会自动刷新。

差异：

| 对比点 | `PrintStream` | `PrintWriter` |
| --- | --- | --- |
| 数据模型 | 字节流 | 字符流 |
| 构造参数 | 可接收 `OutputStream`、文件等 | 可接收 `Writer`、`OutputStream`、文件等 |
| 原始字节写入 | 支持 `write(byte[])` | 不适合写原始字节 |
| 文本输出 | 会根据字符集编码 | 直接写字符，再由底层 Writer 或构造参数编码 |

```java
import java.io.IOException;
import java.io.PrintWriter;
import java.nio.charset.StandardCharsets;
import java.nio.file.Files;
import java.nio.file.Path;

public class PrintWriterDemo {
    public static void writeReport(Path path, int total, int success) throws IOException {
        try (PrintWriter writer = new PrintWriter(
                Files.newBufferedWriter(path, StandardCharsets.UTF_8), true)) {
            writer.printf("total=%d, success=%d%n", total, success);
            if (writer.checkError()) {
                throw new IOException("failed to write report: " + path);
            }
        }
    }
}
```

## close 与 flush

| 方法 | 行为 | 后续能否继续写 |
| --- | --- | --- |
| `flush()` | 强制把缓冲区中的数据写到底层目标 | 可以 |
| `close()` | 关闭流并释放资源，通常会先执行刷新 | 不可以 |

使用 `try-with-resources` 管理流最稳妥，它可以确保异常情况下也调用 `close()`。

```java
import java.io.BufferedOutputStream;
import java.io.IOException;
import java.io.OutputStream;
import java.nio.file.Files;
import java.nio.file.Path;

public class FlushCloseDemo {
    public static void writeBytes(Path path, byte[] data) throws IOException {
        try (OutputStream out = new BufferedOutputStream(Files.newOutputStream(path))) {
            out.write(data);
            out.flush();
            // flush 后还可以继续写，close 后不可以。
            out.write('\n');
        }
    }
}
```
