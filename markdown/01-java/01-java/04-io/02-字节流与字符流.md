# 字节流与字符流

## InputStream

`InputStream` 是字节输入流的抽象基类，读取结果是 `byte` 或 `byte[]`。

常用方法：

| 方法 | 说明 |
| --- | --- |
| `read()` | 读取一个字节，返回 `0` 到 `255`，返回 `-1` 表示结束 |
| `read(byte[] b)` | 最多读取 `b.length` 个字节，返回实际读取数量 |
| `read(byte[] b, int off, int len)` | 读取到数组指定范围 |
| `skip(long n)` | 尝试跳过 `n` 个字节，返回实际跳过数量 |
| `available()` | 返回估计可无阻塞读取的字节数，不等于文件总长度 |
| `close()` | 关闭流并释放底层资源 |
| `readAllBytes()` | Java 9 增加，读取所有剩余字节，注意大文件内存风险 |
| `transferTo(OutputStream out)` | Java 9 增加，将输入流内容传输到输出流 |

常见实现：

| 类 | 说明 |
| --- | --- |
| `FileInputStream` | 从文件读取原始字节 |
| `BufferedInputStream` | 增加字节缓冲，默认 8192 字节 |
| `DataInputStream` | 读取 Java 基本类型和 UTF 字符串，通常要配合 `DataOutputStream` |
| `ObjectInputStream` | 读取 Java 序列化对象 |

## OutputStream

`OutputStream` 是字节输出流的抽象基类，负责写出 `byte` 或 `byte[]`。

常用方法：

| 方法 | 说明 |
| --- | --- |
| `write(int b)` | 写出一个字节，只有低 8 位会被写出 |
| `write(byte[] b)` | 写出整个字节数组 |
| `write(byte[] b, int off, int len)` | 写出数组指定范围 |
| `flush()` | 强制刷新缓冲数据 |
| `close()` | 关闭流并释放资源，通常会先刷新 |

常见实现：

| 类 | 说明 |
| --- | --- |
| `FileOutputStream` | 写入原始字节到文件，可自动创建文件但不会自动创建父目录 |
| `BufferedOutputStream` | 增加字节缓冲，默认 8192 字节 |
| `DataOutputStream` | 写出 Java 基本类型和 UTF 字符串 |
| `ObjectOutputStream` | 写出 Java 序列化对象 |
| `PrintStream` | 打印字节流，`System.out` 和 `System.err` 的类型 |

## Reader

`Reader` 是字符输入流的抽象基类，读取结果是 `char` 或 `char[]`。它适合文本，不适合图片、压缩包等二进制文件。

常用方法：

| 方法 | 说明 |
| --- | --- |
| `read()` | 读取一个字符，返回 `-1` 表示结束 |
| `read(char[] cbuf)` | 批量读取字符 |
| `read(char[] cbuf, int off, int len)` | 读取到数组指定范围 |
| `skip(long n)` | 尝试跳过 `n` 个字符 |
| `close()` | 关闭流并释放资源 |

`InputStreamReader` 是字节流到字符流的桥梁，需要明确字符集；`FileReader` 是面向文件的便捷封装，但旧版本构造方法无法显式传入字符集，容易依赖平台默认编码。

## Writer

`Writer` 是字符输出流的抽象基类，负责写出字符或字符串。

常用方法：

| 方法 | 说明 |
| --- | --- |
| `write(int c)` | 写出单个字符 |
| `write(char[] cbuf)` | 写出字符数组 |
| `write(char[] cbuf, int off, int len)` | 写出数组指定范围 |
| `write(String s)` | 写出字符串 |
| `write(String s, int off, int len)` | 写出字符串指定范围 |
| `append(CharSequence csq)` | 追加字符序列 |
| `append(char c)` | 追加单个字符 |
| `flush()` | 刷新缓冲字符 |
| `close()` | 关闭流并释放资源 |

`OutputStreamWriter` 是字符流到字节流的桥梁，需要明确字符集；`FileWriter` 是文件字符输出封装，同样要注意默认编码问题。

## 字节转字符 Demo

```java
import java.io.BufferedReader;
import java.io.IOException;
import java.io.InputStream;
import java.io.InputStreamReader;
import java.nio.charset.StandardCharsets;
import java.nio.file.Files;
import java.nio.file.Path;

public class CharsetReadDemo {
    public static void printLines(Path path) throws IOException {
        try (InputStream input = Files.newInputStream(path);
             InputStreamReader reader = new InputStreamReader(input, StandardCharsets.UTF_8);
             BufferedReader bufferedReader = new BufferedReader(reader)) {
            String line;
            while ((line = bufferedReader.readLine()) != null) {
                System.out.println(line);
            }
        }
    }
}
```

## 字符转字节 Demo

```java
import java.io.BufferedWriter;
import java.io.IOException;
import java.io.OutputStream;
import java.io.OutputStreamWriter;
import java.nio.charset.StandardCharsets;
import java.nio.file.Files;
import java.nio.file.Path;

public class CharsetWriteDemo {
    public static void writeText(Path path, String text) throws IOException {
        try (OutputStream output = Files.newOutputStream(path);
             OutputStreamWriter writer = new OutputStreamWriter(output, StandardCharsets.UTF_8);
             BufferedWriter bufferedWriter = new BufferedWriter(writer)) {
            bufferedWriter.write(text);
            bufferedWriter.newLine();
        }
    }
}
```
