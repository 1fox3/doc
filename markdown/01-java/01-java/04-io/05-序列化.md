# 序列化

## 基本概念

Java 序列化把对象状态转换为字节流，反序列化则从字节流恢复对象。JDK 内置机制主要依赖 `Serializable`、`Externalizable`、`ObjectOutputStream` 和 `ObjectInputStream`。

## Serializable

`Serializable` 是标记接口，不要求实现方法。对象可序列化时，它的非 `transient`、非 `static` 字段会被序列化。

```java
import java.io.Serial;
import java.io.Serializable;

public class UserSession implements Serializable {
    @Serial
    private static final long serialVersionUID = 1L;

    private final long userId;
    private final String username;
    private transient String accessToken;

    public UserSession(long userId, String username, String accessToken) {
        this.userId = userId;
        this.username = username;
        this.accessToken = accessToken;
    }

    public long userId() {
        return userId;
    }

    public String username() {
        return username;
    }

    public String accessToken() {
        return accessToken;
    }
}
```

## serialVersionUID

`serialVersionUID` 用于判断序列化数据与当前类定义是否兼容。建议显式声明，避免编译器根据类结构自动生成后因为无意修改导致反序列化失败。

常见影响：

| 变更 | 兼容性风险 |
| --- | --- |
| 增加字段 | 通常可兼容，新字段为默认值 |
| 删除字段 | 通常可兼容，旧数据字段被忽略 |
| 修改字段类型 | 高风险，可能失败或数据不符合预期 |
| 修改类继承关系 | 高风险 |
| 修改 `serialVersionUID` | 明确表示不兼容，反序列化会抛 `InvalidClassException` |

## transient 与 static

| 修饰 | 是否序列化 | 原因 |
| --- | --- | --- |
| `transient` | 否 | 显式排除普通成员字段 |
| `static` | 否 | 属于类，不属于对象状态 |

`transient` 只能修饰字段，对 `Serializable` 的默认序列化机制生效。

## @Serial

`@Serial` 用于标记序列化机制识别的特殊字段或方法，例如 `serialVersionUID`、`writeObject`、`readObject`。它类似 `@Override`，主要提供编译期检查和可读性。

```java
import java.io.IOException;
import java.io.ObjectInputStream;
import java.io.ObjectOutputStream;
import java.io.Serial;
import java.io.Serializable;

public class TokenBox implements Serializable {
    @Serial
    private static final long serialVersionUID = 1L;

    private String token;

    public TokenBox(String token) {
        this.token = token;
    }

    @Serial
    private void writeObject(ObjectOutputStream out) throws IOException {
        out.defaultWriteObject();
    }

    @Serial
    private void readObject(ObjectInputStream in) throws IOException, ClassNotFoundException {
        in.defaultReadObject();
        if (token == null || token.isBlank()) {
            throw new IOException("invalid token");
        }
    }
}
```

## Externalizable

`Externalizable` 要求实现 `writeExternal` 和 `readExternal`，完全由开发者控制哪些字段如何序列化。它需要公开无参构造方法，使用门槛更高。

```java
import java.io.Externalizable;
import java.io.IOException;
import java.io.ObjectInput;
import java.io.ObjectOutput;

public class Point implements Externalizable {
    private int x;
    private int y;

    public Point() {
    }

    public Point(int x, int y) {
        this.x = x;
        this.y = y;
    }

    @Override
    public void writeExternal(ObjectOutput out) throws IOException {
        out.writeInt(x);
        out.writeInt(y);
    }

    @Override
    public void readExternal(ObjectInput in) throws IOException {
        x = in.readInt();
        y = in.readInt();
    }
}
```

## 序列化与反序列化 Demo

```java
import java.io.IOException;
import java.io.ObjectInputStream;
import java.io.ObjectOutputStream;
import java.nio.file.Files;
import java.nio.file.Path;

public class SerializationDemo {
    public static void write(Path path, UserSession session) throws IOException {
        try (ObjectOutputStream out = new ObjectOutputStream(Files.newOutputStream(path))) {
            out.writeObject(session);
        }
    }

    public static UserSession read(Path path) throws IOException, ClassNotFoundException {
        try (ObjectInputStream in = new ObjectInputStream(Files.newInputStream(path))) {
            return (UserSession) in.readObject();
        }
    }
}
```

## 注意点

- 被序列化对象必须实现 `Serializable` 或 `Externalizable`。
- 对象字段如果也是对象，该字段对应的运行时类型也必须可序列化，否则会抛 `NotSerializableException`。
- 父类实现 `Serializable` 后，子类默认可序列化。
- 父类未实现 `Serializable` 而子类实现时，反序列化会调用父类无参构造方法恢复父类状态。
- JDK 原生序列化数据通常只适合 Java 内部使用，不适合作为跨语言协议。
- 反序列化不可信数据存在安全风险，生产环境应避免直接反序列化外部输入。
