# 注解

注解是附加在代码元素上的元数据，本身不改变程序逻辑，需要编译器、框架或运行期反射解析。

## 解析方式

| 解析时机 | 说明 | 典型示例 |
|----------|------|----------|
| 编译期扫描 | 编译器直接处理，不进入运行期 | `@Override`、`@SuppressWarnings` |
| 运行期反射 | 需要 `@Retention(RUNTIME)`，通过反射读取 | `@Value`、`@Component`、框架注解 |

## 元注解

| 元注解 | 作用 |
|--------|------|
| `@Target` | 限定注解可标注的代码元素（类、方法、字段等） |
| `@Retention` | 控制注解保留阶段：`SOURCE` / `CLASS` / `RUNTIME` |
| `@Documented` | 是否包含在 Javadoc 中 |
| `@Inherited` | 子类是否继承父类的注解 |
| `@Repeatable` | 是否允许在同一元素上重复标注 |

## @Retention 三个级别

- `SOURCE`：仅保留在源码，编译后丢弃，如 `@Override`。
- `CLASS`：进入 class 文件，但运行期不可反射读取（默认值）。
- `RUNTIME`：运行期可通过反射读取，框架注解通常使用此级别。

## 注意点

- 自定义注解通常配合 AOP、拦截器、`BeanPostProcessor` 或编译期注解处理器（APT）使用。
