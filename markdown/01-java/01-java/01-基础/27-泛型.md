# 泛型

泛型把类型参数化，让编译器在编译期做类型检查。Java 泛型通过**类型擦除**实现，运行期大部分泛型信息不可见。

## 分类

- 泛型类：`class Box<T> { ... }`
- 泛型接口：`interface Converter<F, R> { ... }`
- 泛型方法：`public <T> T get(Class<T> clazz) { ... }`

## 标识符约定

| 标识符 | 全称 | 含义 | 典型场景 |
|--------|------|------|----------|
| `T` | Type | 任意类型 | 泛型类、泛型方法的主类型参数 |
| `R` | Result | 返回结果类型 | 函数式接口、转换器的返回值 |
| `K` | Key | 键 | Map 的键类型 |
| `V` | Value | 值 | Map 的值类型 |
| `E` | Element | 集合元素类型 | List、Set 等集合结构 |
| `N` | Number | 数字类型 | 限定为数值子类时 |
| `?` | 通配符 | 未知类型 | 只读场景、参数多态场景 |

## 通配符与 PECS 原则

> **PECS：Producer Extends，Consumer Super**

- `? extends T`：上界通配符，表示 T 或 T 的子类，适合**读取**（不能写入）。
- `? super T`：下界通配符，表示 T 或 T 的父类，适合**写入**。
- `?`：无界通配符，只读，类型完全未知，除 `null` 外不能写入。

```java
// 上界：读取场景（Producer Extends）
public double sum(List<? extends Number> list) {
    return list.stream().mapToDouble(Number::doubleValue).sum();
}

// 下界：写入场景（Consumer Super）
public void addIntegers(List<? super Integer> list) {
    list.add(1);
    list.add(2);
}
```

## 常见限制

- 不能直接 `new T()`。
- 不能创建泛型数组，如 `new List<String>[10]`。
- 运行期不能用 `instanceof List<String>` 判断具体泛型参数（类型已擦除）。
