# 字符串

## String / StringBuilder / StringBuffer 对比

| | `String` | `StringBuilder` | `StringBuffer` |
|-|----------|-----------------|----------------|
| 可变性 | 不可变 | 可变 | 可变 |
| 线程安全 | 是（不可变） | 否 | 是（方法加 `synchronized`） |
| 适用场景 | 少量字符串、常量 | 单线程大量拼接 | 多线程拼接（性能较低） |

## 不可变性原因

- `String` 类本身被 `final` 修饰，不能被继承。
- 内部 `char[]`（Java 9 后改为 `byte[]`）也被 `final` 修饰，且未暴露修改内容的方法。

## 字符串常量池与 intern()

- 字符串字面量自动进入常量池。
- `new String("abc")` 在堆上创建新对象，字面量 `"abc"` 仍在常量池。
- `intern()` 返回常量池中对应字符串的引用，若不存在则加入。

## `+` vs `concat`

| | `+` | `concat` |
|-|-----|----------|
| 支持类型 | 字符串和基本类型 | 仅字符串 |
| null 处理 | 两侧可为 `null` | 调用方为 `null` 抛 NPE |

## isEmpty vs isBlank

- `isEmpty()`：长度为 0 返回 `true`，不忽略空白字符。
- `isBlank()`（Java 11+）：长度为 0 或全为空白字符返回 `true`。

## 长度限制

- 编译期：字符串字面量不能超过 `65535` 字节。
- 运行期：最大长度为 `Integer.MAX_VALUE`。

## 底层实现变更

- Java 9 将 `String` 底层从 `char[]` 改为 `byte[]`，纯 Latin-1 字符串每字符占 1 字节，相比 `char[]` 每字符 2 字节更省内存。
