# 动态代理

## JDK 动态代理 vs CGLIB

| 维度 | JDK 动态代理 | CGLIB |
|------|-------------|-------|
| 实现机制 | 反射 + 接口 | ASM 字节码生成子类 |
| 目标类要求 | 必须实现接口 | 无要求，但不能是 `final` 类 |
| 方法限制 | 只能代理接口声明的方法 | 不能代理 `final` 方法 |
| 核心 API | `Proxy` + `InvocationHandler` | `Enhancer` + `MethodInterceptor` |
| 方法调用性能 | 经过反射，稍慢 | `invokeSuper` 直调父类，稍快 |
| 代理对象创建开销 | 较小 | 较大（字节码生成） |
| Spring 默认策略 | Spring Boot 2.x 前有接口时优先 | Spring Boot 2.x 起默认（`proxyTargetClass=true`） |

## JDK 动态代理示例

```java
// InvocationHandler：在方法前后织入日志
public class LoggingHandler implements InvocationHandler {
    private final Object target;

    public LoggingHandler(Object target) { this.target = target; }

    @Override
    public Object invoke(Object proxy, Method method, Object[] args) throws Throwable {
        System.out.println("[LOG] >> " + method.getName());
        Object result = method.invoke(target, args);
        System.out.println("[LOG] << " + method.getName());
        return result;
    }
}

// 创建代理
OrderService proxy = (OrderService) Proxy.newProxyInstance(
    target.getClass().getClassLoader(),
    new Class<?>[]{OrderService.class},
    new LoggingHandler(target)
);
// 代理类名：com.sun.proxy.$Proxy0
```

## CGLIB 示例

```java
// MethodInterceptor：计时拦截
public class TimingInterceptor implements MethodInterceptor {
    @Override
    public Object intercept(Object obj, Method method, Object[] args, MethodProxy methodProxy)
            throws Throwable {
        long start = System.currentTimeMillis();
        Object result = methodProxy.invokeSuper(obj, args); // 直调父类，无反射开销
        System.out.println(method.getName() + " " + (System.currentTimeMillis() - start) + "ms");
        return result;
    }
}

// 创建代理
Enhancer enhancer = new Enhancer();
enhancer.setSuperclass(PaymentService.class);
enhancer.setCallback(new TimingInterceptor());
PaymentService proxy = (PaymentService) enhancer.create();
// 代理类名：PaymentService$$EnhancerByCGLIB$$xxxxxx
```

## 注意点

- **自调用绕过代理**：`this.方法()` 直接调用目标对象，事务、缓存、异步等注解全部失效。解决方案：注入自身代理 Bean，或将方法抽到另一个 Spring Bean。
- **`final` 方法静默失效**：CGLIB 无法覆写 `final` 方法，`@Transactional` 等注解标注在 `final` 方法上不会报错但完全不生效。
- 典型应用：Spring AOP、声明式事务、RPC 客户端、MyBatis Mapper。
