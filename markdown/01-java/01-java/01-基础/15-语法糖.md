# 语法糖

语法糖是编译器层面的便利语法，不增加 JVM 基础能力，编译期会被转换成已有语法或字节码结构。

## 常见语法糖及底层实现

| 语法糖 | 编译后等价形式 | 注意点 |
|--------|---------------|--------|
| `switch` 支持字符串 | 先 `hashCode` 分组，再 `equals` 精确匹配，映射为整数分支 | hash 冲突时仍用 `equals` 确认，语义正确 |
| 字符串 `+` 拼接 | Java 9 前：`StringBuilder.append`；Java 9 后：`invokedynamic` + `StringConcatFactory` | 循环中大量拼接建议显式用 `StringBuilder` |
| 泛型（类型擦除） | 运行期擦除为原始类型，如 `List<String>` → `List` | 不能 `new T()`、不能创建泛型数组、运行期无法获取具体泛型参数 |
| 自动装箱/拆箱 | `Integer.valueOf(int)` / `Integer.intValue()` | 包装类型为 `null` 时拆箱抛 NPE；包装类型比较用 `equals` 而非 `==` |
| 变长参数 | 编译为数组 | 只能有一个且必须在参数列表最后；重载时可能造成调用歧义 |
| 增强 for 循环 | 数组展开为下标遍历；集合展开为 `Iterator` | 遍历集合时不能直接调用 `remove`，否则触发 `ConcurrentModificationException` |
| 内部类 | 生成独立 `.class` 文件（`OuterClass$InnerClass.class`） | 非静态内部类持有外部类实例引用；静态内部类不持有 |
| 枚举类 | 编译为继承 `java.lang.Enum` 的 `final` 类 | 枚举不能被继承；可定义字段、构造器、方法和实现接口 |
| Lambda | `invokedynamic` + `LambdaMetafactory.metafactory` 运行期动态生成 | 只能捕获 effectively final 的局部变量 |
| `try-with-resources` | 编译器生成 `finally` 关闭逻辑，处理 suppressed exception | 资源类需实现 `AutoCloseable` 或 `Closeable` |
| 断言 `assert` | 条件判断 + 抛出 `AssertionError` | 默认不启用，需 `-ea` 开启；不能替代业务参数校验 |
| `var` 类型推断 | 编译期由初始化表达式确定具体类型 | 只能用于局部变量；类型不明显时滥用降低可读性 |

## 示例

```java
// switch 支持字符串
String type = "A";
switch (type) {
    case "A" -> System.out.println("type A");
    case "B" -> System.out.println("type B");
    default  -> System.out.println("unknown");
}

// 自动装箱与拆箱
Integer count = 1;   // int -> Integer
int value = count;   // Integer -> int

// 增强 for 循环（集合删除元素需用 Iterator）
for (String item : List.of("java", "spring")) {
    System.out.println(item);
}

// try-with-resources
try (BufferedReader reader = new BufferedReader(new FileReader("README.md"))) {
    System.out.println(reader.readLine());
}

// var 类型推断
var message = "hello";
System.out.println(message);
```
