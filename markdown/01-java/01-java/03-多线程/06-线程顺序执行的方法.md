# 线程顺序执行的方法

## 概述

在多线程编程中，控制线程执行顺序是一个常见需求：例如先初始化资源再启动业务逻辑，或者 A 任务完成后才能启动 B 任务。Java 提供了多种机制来实现这一目标，从低级的 `Object.wait()/notify()` 到高级的 `CompletableFuture` 编排，选择哪种取决于场景复杂度和可读性要求。

核心思路是：让后续线程在前置线程完成之前阻塞等待，或者通过单线程化保证提交顺序即执行顺序。

## 核心概念

### 1. Thread.join()

调用 `threadA.join()` 会使当前线程阻塞，直到 `threadA` 执行完毕（即 `threadA` 的 `run()` 方法返回）。底层通过 `Object.wait()` 实现，线程结束时 JVM 会调用 `notifyAll()`。

- `join()` 等待线程永久终止
- `join(long millis)` 带超时，超时后不抛异常，需手动检查线程是否仍存活
- 如果目标线程已经结束，`join()` 立即返回

### 2. Object.wait() / notify()

`wait()` 和 `notify()` 是 `Object` 上的方法，必须在持有该对象的 `synchronized` 锁的情况下调用，否则抛 `IllegalMonitorStateException`。

- `wait()` 释放锁并阻塞当前线程，直到其他线程调用同一对象的 `notify()` 或 `notifyAll()`
- `notify()` 唤醒一个等待线程（随机），`notifyAll()` 唤醒所有等待线程
- 存在虚假唤醒（spurious wakeup），必须用 `while` 循环而非 `if` 检查条件

### 3. Condition.await() / signal()

`ReentrantLock` 配套的 `Condition` 是 `wait/notify` 的替代，语义更清晰，且一个锁可以绑定多个 `Condition`，实现精细的等待队列控制。

- `condition.await()` 释放锁并等待，对应 `Object.wait()`
- `condition.signal()` 唤醒一个等待该 Condition 的线程，对应 `Object.notify()`
- 同样需要在持有对应 Lock 的情况下调用，否则抛 `IllegalMonitorStateException`
- 同样需要 `while` 循环防虚假唤醒

### 4. SingleThreadExecutor（单线程线程池）

`Executors.newSingleThreadExecutor()` 内部只有一个工作线程，所有提交的任务按 FIFO 顺序串行执行，天然保证顺序，无需额外同步。

- 适合：任务之间存在顺序依赖，但不需要并发的场景
- 内部使用无界 `LinkedBlockingQueue`，大量任务提交可能导致 OOM
- 若工作线程异常终止，线程池会自动创建新线程继续处理队列中的任务

### 5. CountDownLatch

`CountDownLatch` 初始化一个计数器，调用 `countDown()` 减 1，调用 `await()` 阻塞直到计数器归零。适合"等待多个前置条件全部满足"的场景。

- 计数器不可重置（一次性），需要重复使用请改用 `CyclicBarrier`
- `await(long timeout, TimeUnit unit)` 支持超时等待，返回 `false` 表示超时

### 6. CompletableFuture.thenRun() / thenApply()

`CompletableFuture` 提供声明式的异步任务编排，`thenRun()` 在前一个任务完成后执行下一个任务，不传递结果；`thenApply()` 传递结果。

- `thenRun(Runnable)` — 上一步结果不传入，纯顺序触发
- `thenApply(Function)` — 上一步结果作为参数传入，返回新结果
- `thenAccept(Consumer)` — 消费上一步结果，不返回新值
- 默认在完成前一步的线程上执行回调；使用 `thenRunAsync()` 可指定线程池

## 示例代码

### Thread.join() 控制 T1 → T2 → T3 顺序

```java
public class JoinDemo {
    public static void main(String[] args) throws InterruptedException {
        Thread t1 = new Thread(() -> System.out.println("T1 执行"), "T1");
        Thread t2 = new Thread(() -> System.out.println("T2 执行"), "T2");
        Thread t3 = new Thread(() -> System.out.println("T3 执行"), "T3");

        t1.start();
        t1.join(); // main 线程阻塞，等待 t1 结束

        t2.start();
        t2.join(); // main 线程阻塞，等待 t2 结束

        t3.start();
        t3.join();
    }
}
// 输出顺序固定：T1 执行 → T2 执行 → T3 执行
```

### wait() / notify() 实现交替打印

```java
public class WaitNotifyDemo {
    private static final Object lock = new Object();
    private static int turn = 1; // 1 表示轮到 T1，2 表示轮到 T2

    public static void main(String[] args) {
        Thread t1 = new Thread(() -> {
            for (int i = 0; i < 3; i++) {
                synchronized (lock) {
                    while (turn != 1) { // 必须用 while，防虚假唤醒
                        try { lock.wait(); } catch (InterruptedException e) { Thread.currentThread().interrupt(); }
                    }
                    System.out.println("T1: " + i);
                    turn = 2;
                    lock.notifyAll();
                }
            }
        });

        Thread t2 = new Thread(() -> {
            for (int i = 0; i < 3; i++) {
                synchronized (lock) {
                    while (turn != 2) {
                        try { lock.wait(); } catch (InterruptedException e) { Thread.currentThread().interrupt(); }
                    }
                    System.out.println("T2: " + i);
                    turn = 1;
                    lock.notifyAll();
                }
            }
        });

        t1.start();
        t2.start();
    }
}
```

### Condition.await() / signal() 精细控制

```java
import java.util.concurrent.locks.Condition;
import java.util.concurrent.locks.ReentrantLock;

public class ConditionDemo {
    private static final ReentrantLock lock = new ReentrantLock();
    private static final Condition condT1 = lock.newCondition();
    private static final Condition condT2 = lock.newCondition();
    private static int turn = 1;

    public static void main(String[] args) {
        Thread t1 = new Thread(() -> {
            lock.lock();
            try {
                while (turn != 1) condT1.await();
                System.out.println("T1 执行");
                turn = 2;
                condT2.signal(); // 精准唤醒 T2，而非 notifyAll
            } catch (InterruptedException e) {
                Thread.currentThread().interrupt();
            } finally {
                lock.unlock();
            }
        });

        Thread t2 = new Thread(() -> {
            lock.lock();
            try {
                while (turn != 2) condT2.await();
                System.out.println("T2 执行");
                turn = 1;
                condT1.signal();
            } catch (InterruptedException e) {
                Thread.currentThread().interrupt();
            } finally {
                lock.unlock();
            }
        });

        t2.start(); // 故意先启动 T2，验证顺序控制效果
        t1.start();
    }
}
```

### SingleThreadExecutor 保证提交顺序

```java
import java.util.concurrent.ExecutorService;
import java.util.concurrent.Executors;

public class SingleThreadDemo {
    public static void main(String[] args) throws InterruptedException {
        ExecutorService executor = Executors.newSingleThreadExecutor();

        executor.submit(() -> System.out.println("任务 1"));
        executor.submit(() -> System.out.println("任务 2"));
        executor.submit(() -> System.out.println("任务 3"));

        executor.shutdown();
        // 输出顺序固定：任务 1 → 任务 2 → 任务 3
        // 无需任何显式同步，顺序由单线程队列保证
    }
}
```

### CountDownLatch 等待多个前置任务完成

```java
import java.util.concurrent.CountDownLatch;

public class CountDownLatchDemo {
    public static void main(String[] args) throws InterruptedException {
        CountDownLatch latch = new CountDownLatch(2); // 等待 2 个前置任务

        Thread t1 = new Thread(() -> {
            System.out.println("前置任务 A 完成");
            latch.countDown(); // 计数器 -1
        });

        Thread t2 = new Thread(() -> {
            System.out.println("前置任务 B 完成");
            latch.countDown(); // 计数器 -1
        });

        t1.start();
        t2.start();

        latch.await(); // 阻塞直到计数器归零
        System.out.println("主任务开始，A 和 B 均已完成");

        // 带超时版本
        // boolean finished = latch.await(5, TimeUnit.SECONDS);
        // if (!finished) System.out.println("超时，部分前置任务未完成");
    }
}
```

### CompletableFuture 链式编排

```java
import java.util.concurrent.CompletableFuture;

public class CompletableFutureDemo {
    public static void main(String[] args) throws Exception {
        CompletableFuture<String> future = CompletableFuture
            .supplyAsync(() -> {
                System.out.println("步骤 1：获取数据");
                return "rawData";
            })
            .thenApply(data -> {
                System.out.println("步骤 2：处理数据 " + data);
                return data.toUpperCase();
            })
            .thenApply(data -> {
                System.out.println("步骤 3：格式化 " + data);
                return "Result: " + data;
            });

        // thenRun 不接收上一步结果，纯粹顺序触发
        future.thenRun(() -> System.out.println("步骤 4：清理资源"));

        System.out.println(future.get()); // 阻塞获取最终结果
    }
}
```

## 常见问题

### 1. wait()/notify() 必须在 synchronized 块内调用

在 `synchronized` 块外调用 `wait()` 或 `notify()` 会抛 `IllegalMonitorStateException`。调用 `wait()` 的对象必须与 `synchronized` 持有的对象是同一个。`Condition.await()/signal()` 同理，必须在持有对应 `ReentrantLock` 的情况下调用。

### 2. 条件检查必须用 while，不能用 if

线程被唤醒后不代表条件一定满足（虚假唤醒是 JVM 规范允许的行为）。永远用 `while (condition not met) { wait(); }` 而非 `if`。

### 3. CountDownLatch 计数器不可重置

`CountDownLatch` 是一次性的，计数归零后无法重置。如果需要可重复使用的屏障，应使用 `CyclicBarrier`。如果需要动态增减计数，应使用 `Phaser`。

### 4. SingleThreadExecutor 使用无界队列，存在 OOM 风险

`Executors.newSingleThreadExecutor()` 内部队列无界，如果任务生产速度远超消费速度，队列会无限增长导致 OOM。生产环境建议手动创建 `ThreadPoolExecutor` 并指定有界队列和拒绝策略：

```java
ExecutorService executor = new ThreadPoolExecutor(
    1, 1, 0L, TimeUnit.MILLISECONDS,
    new LinkedBlockingQueue<>(1000), // 有界队列
    new ThreadPoolExecutor.CallerRunsPolicy() // 满了由调用者线程执行
);
```
