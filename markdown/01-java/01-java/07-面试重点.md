# Java 面试重点

Java 面试要从语言语义、集合、并发、IO、泛型、异常和版本演进串起来。回答时不要只背 API，要解释运行时行为和工程边界。

## 基础语义

- Java 只有值传递；对象引用作为值传递，方法内修改引用指向不会影响调用方变量，修改对象内部状态会影响同一对象。
- `==` 比较基本类型值或引用地址，`equals` 是对象语义相等，重写 `equals` 必须同时重写 `hashCode`。
- `final` 修饰变量表示引用不可变，不代表对象内容不可变；修饰方法禁止重写；修饰类禁止继承。
- 自动装箱会引入对象创建、缓存和空指针风险，`Integer` 常见缓存范围默认是 `-128` 到 `127`。
- 字符串要区分字面量、堆对象、字符串常量池和 `intern()`。

## 集合高频题

| 主题 | 要点 |
| --- | --- |
| ArrayList | 动态数组，随机访问快，插入删除可能移动元素，扩容通常按 1.5 倍 |
| LinkedList | 双向链表，按下标访问慢，节点对象额外内存开销高 |
| HashMap | 数组、链表、红黑树，扰动函数、扩容、负载因子、树化条件 |
| ConcurrentHashMap | JDK 8 以 CAS、synchronized、Node、TreeBin 控制桶级并发 |
| fail-fast | 通过 modCount 尽早发现并发修改，不保证并发安全 |

## 并发高频题

- 创建线程不是越多越好，要根据 CPU、IO、阻塞比例、队列、内存和下游承载能力设计线程池。
- `ThreadPoolExecutor` 重点掌握 corePoolSize、maximumPoolSize、keepAliveTime、workQueue、threadFactory、rejectedExecutionHandler。
- AQS 基于 volatile state、CAS 和同步队列构建锁、信号量、闭锁等同步器。
- `volatile` 解决可见性和有序性，不保证复合操作原子性。
- `synchronized` 解决互斥、可见性和有序性，锁对象必须被多个线程共享。
- `ThreadLocal` 在线程池中必须及时 remove，否则可能导致内存泄漏和上下文串扰。

## IO 高频题

- BIO 一个连接通常对应一个阻塞线程，简单但并发能力受线程数限制。
- NIO 使用 Channel、Buffer、Selector，可用少量线程管理大量连接。
- 零拷贝关注减少用户态和内核态之间的数据复制，Java 中常见 `FileChannel.transferTo`、mmap、sendfile。
- 序列化要关注兼容性、性能、安全和跨语言，Java 原生序列化不适合高性能和不可信输入场景。

## 泛型与反射

- Java 泛型主要通过擦除实现，运行期大多数泛型信息不存在，必要位置插入强制类型转换。
- 泛型签名可保存在 `Signature` 属性中，反射能读取部分声明信息。
- 反射灵活但有访问控制、性能、封装破坏和模块化限制，生产应缓存 Method/Field 并控制权限。

## 面试表达模板

1. 先给结论。
2. 再讲底层结构或语义边界。
3. 补充版本差异和常见坑。
4. 最后落到项目实践或排障方法。
