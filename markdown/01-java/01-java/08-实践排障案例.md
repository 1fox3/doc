# Java 实践排障案例

## 线程池打满

- 现象：接口超时、队列堆积、拒绝策略触发、线程数接近上限。
- 证据：线程池监控、`jstack`、下游耗时、队列长度、拒绝计数。
- 原因：下游慢、队列无界、线程池混用、任务阻塞、重试放大流量。
- 处理：设置超时、熔断、隔离、限流，拆分 CPU/IO 线程池，控制队列长度和拒绝策略。

## HashMap 并发问题

- 现象：并发读写下数据丢失、覆盖、结构异常或结果不一致。
- 原因：HashMap 不是线程安全容器，扩容和链表/树结构修改没有并发保护。
- 处理：使用 ConcurrentHashMap、不可变快照、读写锁或线程封闭，不能靠 `volatile` 包住 Map 解决复合操作。

## ThreadLocal 泄漏

- 现象：堆内存上涨、用户上下文串扰、ClassLoader 无法卸载。
- 证据：heap dump 中 Thread -> ThreadLocalMap -> value 引用链。
- 原因：线程池线程长期存活，业务 value 未 remove。
- 处理：在 `finally` 中 remove，框架拦截器统一清理，避免保存大对象和 ClassLoader 相关对象。

## SimpleDateFormat 线程安全问题

- 现象：日期解析结果错乱、偶发异常。
- 原因：SimpleDateFormat 内部 Calendar 等状态可变，多线程共享会发生竞态。
- 处理：使用 `DateTimeFormatter`，或 ThreadLocal 包装并清理。
