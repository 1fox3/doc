# Java 18-21 新特性

Java 18 到 Java 21 的重点是虚拟线程、模式匹配增强、Record Pattern、Sequenced Collections 和新一代并发编程模型。Java 21 是 LTS。

## Java 18：默认字符集 UTF-8

Java 18 将 JDK 默认字符集统一为 UTF-8。在此之前，默认字符集依赖操作系统和运行环境，例如中文 Windows 可能是 GBK，Linux 服务器通常是 UTF-8。

```java
import java.nio.charset.Charset;

System.out.println(Charset.defaultCharset()); // UTF-8
```

影响：

- 跨平台读写文本文件的行为更一致。
- 减少“本地正常、服务器乱码”这类环境差异问题。
- 老系统如果依赖平台默认编码，需要显式指定原编码，例如 `Charset.forName("GBK")`。

实践建议：

- 新代码读写文本时仍建议显式指定 `StandardCharsets.UTF_8`。
- 升级老项目时要重点检查文件导入导出、批处理、报表、历史数据解析等编码敏感逻辑。

## Java 18：Simple Web Server

Java 18 提供了一个简单静态文件 HTTP 服务器工具 `jwebserver`，适合快速共享目录或本地验证静态页面。

```bash
jwebserver -p 8000
```

也可以通过 API 启动：

```java
import com.sun.net.httpserver.SimpleFileServer;
import java.net.InetSocketAddress;
import java.nio.file.Path;

var server = SimpleFileServer.createFileServer(
        new InetSocketAddress(8000),
        Path.of("."),
        SimpleFileServer.OutputLevel.INFO
);
server.start();
```

注意：它主要用于开发、测试、教学和临时文件预览，不适合当作生产 Web 服务器。

## Java 18：Code Snippets in Java API Documentation

Java 18 增强 Javadoc，支持在 API 文档中嵌入代码片段，使用 `@snippet` 标记。

```java
/**
 * Example:
 * {@snippet :
 * var names = List.of("Tom", "Jerry");
 * names.forEach(System.out::println);
 * }
 */
public void printNames() {
}
```

价值：

- API 示例代码更规范。
- 文档工具可以更好地识别和渲染代码片段。
- 适合 SDK、公共库、框架组件编写文档。

## Java 18：Internet-Address Resolution SPI

Java 18 增加 Internet 地址解析 SPI，允许替换或定制主机名和 IP 地址解析逻辑。

它主要面向框架、运行平台、云原生基础设施和特殊网络环境。普通业务代码很少直接使用，但对服务发现、DNS 策略、测试隔离等底层能力有意义。

## Java 18：Vector API 第三次孵化

Vector API 在 Java 18 继续孵化，用于表达可由 CPU SIMD 指令加速的向量计算。

适合场景：

- 图像处理。
- 加解密。
- 机器学习基础计算。
- 数值计算和批量数组处理。

它仍属于孵化 API，生产使用要关注版本变化和硬件差异。

## Java 19：虚拟线程第一次预览

Java 19 首次预览虚拟线程。它来自 Project Loom，用于降低阻塞 IO 场景下“一任务一线程”的成本。

```java
Thread.startVirtualThread(() -> {
    System.out.println("run in virtual thread");
});
```

Java 19 阶段虚拟线程还不是正式特性，主要用于试用和反馈。Java 21 才正式标准化虚拟线程。

## Java 19：结构化并发第一次孵化

Java 19 引入结构化并发孵化 API，用于把一组并发任务作为一个整体管理。

它试图解决的问题包括：

- 子任务失败后其他任务如何取消。
- 多个并发任务的异常如何传播。
- 父任务和子任务生命周期如何保持清晰。

该能力在后续版本继续预览和调整，使用时必须关注具体 JDK 版本 API。

## Java 19：Record Patterns 第一次预览

Java 19 首次预览 Record Patterns，用于解构 record。

```java
record Point(int x, int y) {}

if (obj instanceof Point(int x, int y)) {
    System.out.println(x + y);
}
```

这个能力在 Java 21 标准化。

## Java 19：Pattern Matching for switch 第三次预览

Java 19 继续预览 switch 模式匹配，让 `switch` 可以基于类型、`null` 和模式进行分支。

```java
String result = switch (obj) {
    case String s -> s.toUpperCase();
    case Integer i -> "int: " + i;
    case null -> "null";
    default -> "unknown";
};
```

这个能力在 Java 21 标准化。

## Java 19：Foreign Function & Memory API 第一次预览

Java 19 将 Foreign Function & Memory API 从孵化推进到预览阶段，用于访问本地函数和堆外内存，目标是提供比 JNI 更安全、更现代的替代方案。

该 API 在 Java 22 标准化。

## Java 20：虚拟线程第二次预览

Java 20 继续预览虚拟线程，改进和稳定 Java 19 中的 API 行为。

面试中可以这样区分：

- Java 19：虚拟线程第一次预览。
- Java 20：虚拟线程第二次预览。
- Java 21：虚拟线程正式特性。

## Java 20：Scoped Values 第一次孵化

Java 20 引入 Scoped Values 孵化 API，用于在线程调用链中传递上下文。

它和 `ThreadLocal` 的目标相似，但语义更受约束：

- 作用域明确。
- 通常只读。
- 更适合虚拟线程。
- 更不容易出现上下文泄漏。

Scoped Values 在 Java 25 标准化。

## Java 20：结构化并发第二次孵化

Java 20 继续孵化结构化并发，配合虚拟线程改善并发任务组织方式。

典型目标是让并发代码结构更像同步代码：父任务启动多个子任务，等待它们完成，并统一处理成功、失败和取消。

## Java 20：Record Patterns 第二次预览

Java 20 继续预览 Record Patterns，进一步完善 record 解构能力。

```java
record User(String name, int age) {}

if (obj instanceof User(String name, int age)) {
    System.out.println(name + ":" + age);
}
```

该能力在 Java 21 标准化。

## Java 20：Pattern Matching for switch 第四次预览

Java 20 继续预览 switch 模式匹配，并继续打磨与 sealed class、record pattern、`null` case 等组合使用的语义。

该能力在 Java 21 标准化。

## Java 21：虚拟线程 Virtual Threads

虚拟线程是 Java 21 最重要的服务端特性之一，来自 Project Loom。

```java
try (var executor = Executors.newVirtualThreadPerTaskExecutor()) {
    IntStream.range(0, 1000).forEach(i -> executor.submit(() -> callRemote(i)));
}
```

### 解决什么问题

传统平台线程与操作系统线程一一对应，数量昂贵。大量阻塞 IO 会占用大量线程资源。

虚拟线程由 JVM 调度，创建和切换成本低，适合大量阻塞 IO 任务。

### 适合场景

- HTTP/RPC 阻塞调用。
- 数据库访问。
- 文件/网络 IO。
- 高并发但大部分时间在等待外部系统的业务接口。

### 不适合场景

- CPU 密集型任务。
- 需要严格限制下游并发的场景，仍然需要连接池、限流和舱壁隔离。
- 长时间持有 synchronized 或 native 阻塞导致 pinning 的场景需要关注。

### 面试重点

虚拟线程提升的是阻塞 IO 场景的并发承载能力，不会让 CPU 计算变快，也不能替代限流、连接池和超时控制。

## Java 21：结构化并发 Structured Concurrency

结构化并发用于把一组相关并发任务作为一个整体管理，便于取消、异常传播和生命周期控制。

它仍处于预览/孵化阶段，需要关注具体 JDK 版本状态。

Java 21 中可以使用 `StructuredTaskScope` 同时发起多个子任务，并在同一个作用域内统一等待和处理结果。

```java
import java.util.concurrent.StructuredTaskScope;

public class StructuredConcurrencyDemo {
    public static void main(String[] args) throws Exception {
        try (var scope = new StructuredTaskScope.ShutdownOnFailure()) {
            var userTask = scope.fork(() -> queryUser());
            var orderTask = scope.fork(() -> queryOrder());

            scope.join();
            scope.throwIfFailed();

            System.out.println(userTask.get() + ", " + orderTask.get());
        }
    }

    private static String queryUser() {
        return "user-001";
    }

    private static String queryOrder() {
        return "order-001";
    }
}
```

编译和运行预览 API：

```bash
javac --enable-preview --release 21 StructuredConcurrencyDemo.java
java --enable-preview StructuredConcurrencyDemo
```

这个示例中，`queryUser()` 和 `queryOrder()` 是同一组业务子任务。如果其中一个失败，`ShutdownOnFailure` 会关闭整个任务作用域，避免其他子任务继续无意义地运行。

## Java 21：Scoped Values

Scoped Values 用于在线程调用链中安全传递上下文，被视为 ThreadLocal 的现代替代方向之一。

优势：

- 不易泄漏。
- 适合虚拟线程。
- 作用域明确。

示例：

```java
import java.lang.ScopedValue;

public class ScopedValueDemo {
    private static final ScopedValue<String> TRACE_ID = ScopedValue.newInstance();

    public static void main(String[] args) {
        ScopedValue.where(TRACE_ID, "trace-001").run(() -> {
            handleRequest();
        });
    }

    private static void handleRequest() {
        log("start handle request");
    }

    private static void log(String message) {
        System.out.println("[" + TRACE_ID.get() + "] " + message);
    }
}
```

编译和运行预览 API：

```bash
javac --enable-preview --release 21 ScopedValueDemo.java
java --enable-preview ScopedValueDemo
```

`TRACE_ID` 只在 `ScopedValue.where(...).run(...)` 的作用域内可见，离开作用域后不能再读取。它适合传递 traceId、租户 ID、登录用户等请求级上下文。

## Java 21：Pattern Matching for switch

Java 21 标准化 switch 模式匹配。

```java
String text = switch (obj) {
    case Integer i -> "int: " + i;
    case String s -> "str: " + s;
    case null -> "null";
    default -> "unknown";
};
```

价值：

- 按类型分支更清晰。
- 与 sealed class 搭配可做穷尽性检查。
- 适合领域模型和状态机分发。

## Java 21：Record Patterns

Java 21 标准化 Record Patterns。

```java
record Point(int x, int y) {}

if (obj instanceof Point(int x, int y)) {
    System.out.println(x + y);
}
```

价值：

- 解构 record。
- 与模式匹配组合，减少样板代码。

## Java 21：Sequenced Collections

Java 21 引入有序集合统一接口。

核心接口：

- `SequencedCollection`
- `SequencedSet`
- `SequencedMap`

常用能力：

- 获取第一个元素。
- 获取最后一个元素。
- 反向视图。

示例：

```java
import java.util.LinkedHashMap;
import java.util.List;
import java.util.SequencedMap;

public class SequencedCollectionsDemo {
    public static void main(String[] args) {
        var names = List.of("Tom", "Jerry", "Spike");

        System.out.println(names.getFirst()); // Tom
        System.out.println(names.getLast());  // Spike
        System.out.println(names.reversed()); // [Spike, Jerry, Tom]

        SequencedMap<String, Integer> scores = new LinkedHashMap<>();
        scores.put("Tom", 90);
        scores.put("Jerry", 95);
        scores.put("Spike", 88);

        System.out.println(scores.firstEntry());     // Tom=90
        System.out.println(scores.lastEntry());      // Spike=88
        System.out.println(scores.reversed());       // {Spike=88, Jerry=95, Tom=90}
    }
}
```

在 Java 21 之前，不同集合获取首尾元素的方式不统一，例如 `List` 用索引，`Deque` 用 `getFirst()` / `getLast()`，有序 `Map` 还要依赖具体实现。Sequenced Collections 提供了统一抽象，让有明确遍历顺序的集合都能用一致的 API 表达首、尾和反向视图。

## Java 21 LTS 意义

Java 21 是继 Java 17 后的新 LTS。

对后端影响：

- 虚拟线程可能改变传统“一请求一线程”的成本模型。
- 现代模式匹配能力增强领域建模表达力。
- Spring Boot 3.2+ 已开始支持虚拟线程相关配置。

## 面试回答模板

Java 18 到 20 主要是 Java 21 重要能力的铺垫：Java 18 统一默认字符集为 UTF-8，Java 19/20 持续预览虚拟线程、Record Patterns 和 switch 模式匹配，并孵化结构化并发、Scoped Values。Java 21 最值得关注的是虚拟线程正式可用，它让阻塞 IO 场景能以更低线程成本承载高并发；但它不是性能银弹，CPU 密集型任务、连接池容量、限流和超时仍然要治理。同时 Java 21 的模式匹配、Record Patterns、Sequenced Collections 提升了代码表达力。
