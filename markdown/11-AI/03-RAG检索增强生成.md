# RAG 检索增强生成

## RAG 是什么

RAG 是 Retrieval-Augmented Generation，检索增强生成。它先从外部知识库检索相关内容，再把检索结果作为上下文交给大模型生成答案。

适合场景：

1. 企业知识库问答。
2. 智能客服。
3. 文档问答。
4. 法务、财务、运维手册查询。
5. 需要引用来源的问答。

## 基本架构

```text
文档 -> 清洗 -> 切分 -> Embedding -> 向量数据库
用户问题 -> Query 改写 -> Embedding -> 检索 -> 重排 -> 拼接上下文 -> LLM -> 答案
```

核心模块：

1. 数据接入：PDF、Word、网页、数据库、Wiki、工单。
2. 文档解析：抽取文本、表格、标题、层级结构。
3. Chunking：把文档切成适合检索的小段。
4. Embedding：把 chunk 转成向量。
5. Vector DB：存储和检索向量。
6. Retriever：召回相关片段。
7. Reranker：对召回结果重新排序。
8. Generator：基于上下文生成答案。

## 文档切分

切分质量直接影响召回效果。

常见策略：

1. 固定长度切分：简单，但可能破坏语义。
2. 按标题层级切分：适合手册、规范、Wiki。
3. 按段落切分：适合普通文档。
4. 滑动窗口：保留上下文连续性。
5. 表格特殊处理：保留表头、单位和行列关系。

经验：

1. Chunk 太小会丢上下文。
2. Chunk 太大会引入噪声，降低召回精度。
3. 通常需要保留文档标题、章节、来源 URL、更新时间等 metadata。

## 检索方式

### 向量检索

基于语义相似度，适合自然语言问题。

优点：能处理同义表达。

缺点：对精确关键词、编号、专有名词不一定稳定。

### 关键词检索

例如 BM25，适合精确词、错误码、产品型号、接口名。

优点：精确匹配强。

缺点：语义泛化弱。

### 混合检索

把向量检索和关键词检索结合，通常是企业 RAG 的更优选择。

## Rerank

Rerank 是对初步召回结果做更精细排序。

流程：

1. Retriever 先召回 top 20 到 top 100。
2. Reranker 重新计算 query 和 chunk 的相关性。
3. 取 top 3 到 top 10 给大模型。

价值：

1. 提高上下文相关性。
2. 降低噪声。
3. 减少大模型上下文成本。

## RAG 常见问题

### 召回不到

可能原因：

1. 文档没入库或入库失败。
2. 文档切分不合理。
3. Embedding 模型不适合领域语言。
4. 用户问题和文档表达差异太大。
5. 只用了向量检索，缺少关键词检索。

解决方法：

1. 增加混合检索。
2. Query rewrite，把用户问题改写成更适合检索的表达。
3. 调整 chunk size 和 overlap。
4. 加强 metadata filter。
5. 引入 reranker。

### 召回到了但答错

可能原因：

1. 上下文太多，噪声干扰。
2. Prompt 没要求只基于材料回答。
3. 多个片段互相冲突。
4. 模型没有引用依据。

解决方法：

1. 限制上下文数量。
2. 使用 rerank。
3. 要求答案引用来源。
4. 对冲突信息按更新时间、权限、来源优先级处理。

### 答案没有来源

解决方法：

1. 每个 chunk 带 source metadata。
2. Prompt 要求每个结论绑定引用。
3. 前端展示来源标题、段落和链接。

## RAG 评估

评估不能只看单次演示效果。

常见指标：

1. Recall@K：正确材料是否被召回。
2. MRR：正确材料排序是否靠前。
3. Answer Correctness：答案是否正确。
4. Faithfulness：答案是否忠于上下文。
5. Citation Accuracy：引用是否准确。
6. Latency：端到端耗时。
7. Cost：单次问答成本。

评估集构建：

1. 从真实用户问题、客服工单、搜索日志中采样。
2. 人工标注标准答案和证据文档。
3. 覆盖简单问答、复杂问答、无答案问题、权限问题。

## 面试常问

### RAG 如何减少幻觉？

RAG 通过外部检索提供可信上下文，让模型基于材料回答，而不是只依赖参数记忆。同时通过引用来源、上下文约束、无答案拒答和答案校验，可以进一步降低幻觉。但 RAG 不能彻底消除幻觉，因为检索结果可能错误，模型也可能没有严格遵守上下文。

### RAG 和微调的区别？

RAG 解决知识注入和可追溯问题，适合知识频繁变化、需要引用来源的场景。微调解决模型行为和领域风格问题，适合固定任务和稳定输出模式。企业知识库一般优先 RAG。

### 如何优化 RAG 效果？

我会从数据、检索、重排、Prompt、评估五个层面优化：清洗文档和 metadata，调整 chunk，使用混合检索和 query rewrite，引入 reranker，控制上下文拼接，并建立离线评估集持续对比效果。
