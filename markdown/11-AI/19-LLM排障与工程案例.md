# LLM 排障与工程案例

## RAG 回答不准

- 排查链路：文档解析、切分、Embedding、召回、重排、上下文拼接、Prompt、模型输出。
- 常见原因：切分过大或过小、召回 topK 不合理、没有 metadata filter、权限过滤缺失、重排缺失、Prompt 没要求引用来源。
- 修复：优化 chunk、混合检索、rerank、查询改写、按权限过滤、构建评估集。

## 幻觉严重

- 原因：上下文缺失、问题超出知识库、Prompt 约束弱、模型温度高、没有拒答策略。
- 修复：要求基于引用回答，低置信拒答，降低 temperature，输出结构校验，引入人工兜底。

## Agent 循环调用工具

- 原因：目标不清晰、工具描述模糊、缺少最大步数、工具返回不可判定。
- 修复：限制 max steps，明确工具输入输出 schema，增加停止条件，记录 trace 分析决策。

## LLM API 延迟高

- 排查：首 token 延迟、总 token 数、上下文长度、模型大小、网络、限流、流式输出。
- 修复：模型分层、上下文压缩、缓存、流式响应、减少不必要工具调用、批处理离线任务。

## 成本过高

- 原因：上下文过长、重复请求、模型过大、没有缓存、评估和调试流量未隔离。
- 修复：Prompt 压缩、Embedding/检索缓存、结果缓存、小模型路由、限流、预算告警。

## Prompt Injection

- 风险：用户或文档内容诱导模型泄露系统提示、绕过权限或调用危险工具。
- 防护：系统指令隔离，工具权限最小化，检索内容不当作指令，输出审计，高风险动作人工确认。

## 项目表达模板

1. 业务目标：解决什么问题，成功指标是什么。
2. 架构：模型、RAG、Vector DB、服务端、权限、观测。
3. 质量：评估集、人工标注、召回率、准确率、引用命中率。
4. 稳定性：限流、降级、缓存、超时、重试、审计。
5. 成本：模型分层、上下文控制、缓存和预算告警。
