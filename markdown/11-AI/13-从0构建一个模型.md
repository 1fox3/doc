# 从 0 构建一个 Model

## 先定义“从 0”是什么意思

面试里“从 0 构建一个 model”可能有两种含义：

1. 教学级从 0：自己定义 tokenizer、dataset、模型结构、loss、训练循环和推理生成。
2. 工业级从 0：从数据采集、清洗、tokenizer 训练、模型架构、分布式训练、评估、对齐到部署完整构建一个基础模型。

两者差别很大。教学级可以单机完成，工业级需要大规模数据、GPU 集群和模型训练平台。

面试可以这样开场：

> 如果是教学级，我可以用 PyTorch 从 0 写一个小型 decoder-only Transformer，用 next token prediction 训练一个语言模型。如果是工业级大模型，从 0 构建要包含数据工程、tokenizer、模型架构、分布式 pretrain、post-train、评估、部署和监控，成本和复杂度远高于普通应用开发。

## 从 0 构建模型的完整步骤

```text
定义任务 -> 准备数据 -> 训练 tokenizer -> 构建 dataset -> 设计模型结构 -> 训练 -> 评估 -> 推理生成 -> 保存加载 -> 部署
```

| 步骤 | 目标 |
| --- | --- |
| 定义任务 | 明确是分类、回归、生成、Embedding 还是多模态 |
| 准备数据 | 收集、清洗、切分训练集和验证集 |
| Tokenizer | 把文本映射成 token id |
| Dataset | 把原始样本变成 tensor |
| Model | 定义网络结构和 forward |
| Loss | 定义优化目标 |
| Training Loop | 前向、loss、反向传播、更新参数 |
| Evaluation | 验证集指标和生成效果 |
| Inference | 加载模型并生成结果 |
| Deployment | 服务化、监控、灰度和回滚 |

## 先从最简单的模型理解

如果是分类模型，例如垃圾邮件分类：

```text
文本 -> tokenizer -> embedding -> encoder -> classifier -> softmax
```

如果是语言模型，例如 GPT：

```text
文本 -> tokenizer -> token embedding -> transformer blocks -> lm head -> next token probability
```

LLM 的核心训练目标是 next token prediction：

```text
输入：我 喜欢 学习
标签：喜欢 学习 AI
```

模型在每个位置预测下一个 token。

## 教学版 Demo：从 0 写一个小型 Transformer LM

下面是一个可以帮助理解 LLM 原理的最小 demo。它不是生产大模型，只是用字符级 tokenizer 和小型 Transformer 学习一小段文本。

### 依赖

```bash
pip install torch
```

### tiny_transformer_lm.py

```python
import math
import torch
import torch.nn as nn
import torch.nn.functional as F


torch.manual_seed(42)


TEXT = """
RAG retrieves relevant documents before generation.
Agent can call tools to finish multi step tasks.
Fine tuning adapts a base model to a specific task.
Transformer uses attention to model token relationships.
"""


class CharTokenizer:
    def __init__(self, text: str):
        chars = sorted(list(set(text)))
        self.stoi = {ch: i for i, ch in enumerate(chars)}
        self.itos = {i: ch for ch, i in self.stoi.items()}

    @property
    def vocab_size(self):
        return len(self.stoi)

    def encode(self, text: str):
        return [self.stoi[ch] for ch in text]

    def decode(self, ids):
        return "".join(self.itos[i] for i in ids)


class CausalSelfAttention(nn.Module):
    def __init__(self, embed_dim: int, num_heads: int, block_size: int, dropout: float):
        super().__init__()
        assert embed_dim % num_heads == 0
        self.num_heads = num_heads
        self.head_dim = embed_dim // num_heads

        self.qkv = nn.Linear(embed_dim, 3 * embed_dim)
        self.proj = nn.Linear(embed_dim, embed_dim)
        self.dropout = nn.Dropout(dropout)

        mask = torch.tril(torch.ones(block_size, block_size))
        self.register_buffer("mask", mask.view(1, 1, block_size, block_size))

    def forward(self, x):
        batch_size, seq_len, embed_dim = x.shape
        qkv = self.qkv(x)
        q, k, v = qkv.chunk(3, dim=-1)

        q = q.view(batch_size, seq_len, self.num_heads, self.head_dim).transpose(1, 2)
        k = k.view(batch_size, seq_len, self.num_heads, self.head_dim).transpose(1, 2)
        v = v.view(batch_size, seq_len, self.num_heads, self.head_dim).transpose(1, 2)

        scores = (q @ k.transpose(-2, -1)) / math.sqrt(self.head_dim)
        scores = scores.masked_fill(self.mask[:, :, :seq_len, :seq_len] == 0, float("-inf"))
        weights = F.softmax(scores, dim=-1)
        weights = self.dropout(weights)

        out = weights @ v
        out = out.transpose(1, 2).contiguous().view(batch_size, seq_len, embed_dim)
        return self.proj(out)


class FeedForward(nn.Module):
    def __init__(self, embed_dim: int, dropout: float):
        super().__init__()
        self.net = nn.Sequential(
            nn.Linear(embed_dim, 4 * embed_dim),
            nn.GELU(),
            nn.Linear(4 * embed_dim, embed_dim),
            nn.Dropout(dropout),
        )

    def forward(self, x):
        return self.net(x)


class TransformerBlock(nn.Module):
    def __init__(self, embed_dim: int, num_heads: int, block_size: int, dropout: float):
        super().__init__()
        self.ln1 = nn.LayerNorm(embed_dim)
        self.attn = CausalSelfAttention(embed_dim, num_heads, block_size, dropout)
        self.ln2 = nn.LayerNorm(embed_dim)
        self.ffn = FeedForward(embed_dim, dropout)

    def forward(self, x):
        x = x + self.attn(self.ln1(x))
        x = x + self.ffn(self.ln2(x))
        return x


class TinyTransformerLM(nn.Module):
    def __init__(self, vocab_size: int, block_size: int, embed_dim=64, num_heads=4, num_layers=2, dropout=0.1):
        super().__init__()
        self.block_size = block_size
        self.token_embedding = nn.Embedding(vocab_size, embed_dim)
        self.position_embedding = nn.Embedding(block_size, embed_dim)
        self.blocks = nn.Sequential(*[
            TransformerBlock(embed_dim, num_heads, block_size, dropout)
            for _ in range(num_layers)
        ])
        self.ln_f = nn.LayerNorm(embed_dim)
        self.lm_head = nn.Linear(embed_dim, vocab_size)

    def forward(self, input_ids, labels=None):
        batch_size, seq_len = input_ids.shape
        positions = torch.arange(seq_len, device=input_ids.device)

        x = self.token_embedding(input_ids) + self.position_embedding(positions)
        x = self.blocks(x)
        x = self.ln_f(x)
        logits = self.lm_head(x)

        loss = None
        if labels is not None:
            loss = F.cross_entropy(logits.view(-1, logits.size(-1)), labels.view(-1))
        return logits, loss

    @torch.no_grad()
    def generate(self, input_ids, max_new_tokens, temperature=1.0):
        for _ in range(max_new_tokens):
            context = input_ids[:, -self.block_size:]
            logits, _ = self(context)
            logits = logits[:, -1, :] / temperature
            probs = F.softmax(logits, dim=-1)
            next_id = torch.multinomial(probs, num_samples=1)
            input_ids = torch.cat([input_ids, next_id], dim=1)
        return input_ids


def get_batch(data, batch_size, block_size):
    ix = torch.randint(0, len(data) - block_size - 1, (batch_size,))
    x = torch.stack([data[i:i + block_size] for i in ix])
    y = torch.stack([data[i + 1:i + block_size + 1] for i in ix])
    return x, y


def main():
    tokenizer = CharTokenizer(TEXT)
    encoded = torch.tensor(tokenizer.encode(TEXT), dtype=torch.long)

    block_size = 32
    batch_size = 16
    model = TinyTransformerLM(
        vocab_size=tokenizer.vocab_size,
        block_size=block_size,
        embed_dim=64,
        num_heads=4,
        num_layers=2,
        dropout=0.1,
    )

    optimizer = torch.optim.AdamW(model.parameters(), lr=3e-4)

    model.train()
    for step in range(1000):
        x, y = get_batch(encoded, batch_size, block_size)
        _, loss = model(x, y)

        optimizer.zero_grad()
        loss.backward()
        optimizer.step()

        if step % 100 == 0:
            print(f"step={step}, loss={loss.item():.4f}")

    model.eval()
    prompt = "RAG"
    input_ids = torch.tensor([tokenizer.encode(prompt)], dtype=torch.long)
    output_ids = model.generate(input_ids, max_new_tokens=100, temperature=0.8)
    print(tokenizer.decode(output_ids[0].tolist()))

    torch.save({
        "model_state_dict": model.state_dict(),
        "stoi": tokenizer.stoi,
        "itos": tokenizer.itos,
        "block_size": block_size,
    }, "tiny_transformer_lm.pt")


if __name__ == "__main__":
    main()
```

运行：

```bash
python tiny_transformer_lm.py
```

这个 demo 做了完整链路：

1. 自己构建字符级 tokenizer。
2. 把文本编码成 token id。
3. 用 sliding window 生成训练样本。
4. 定义 causal self-attention。
5. 定义 Transformer block。
6. 用 cross entropy 做 next token prediction。
7. 训练模型参数。
8. 用自回归方式生成文本。
9. 保存 checkpoint。

## 代码里的关键组件

### Token Embedding

把 token id 转成向量。

```text
token id -> embedding vector
```

### Position Embedding

Transformer 本身不知道顺序，需要位置编码告诉模型 token 在序列中的位置。

### Causal Mask

语言模型生成第 t 个 token 时不能偷看未来 token，所以要用下三角 mask。

```text
第 1 个 token 只能看第 1 个
第 2 个 token 可以看第 1、2 个
第 3 个 token 可以看第 1、2、3 个
```

### Self-Attention

让每个 token 根据上下文选择关注哪些 token。

### Feed Forward

每个位置独立经过 MLP，提升非线性表达能力。

### Residual Connection

残差连接帮助深层网络训练稳定。

### LayerNorm

归一化激活，提升训练稳定性。

### LM Head

把隐藏向量映射到 vocab size，输出每个 token 的概率。

## 训练目标

语言模型训练样本：

```text
x = [R, A, G]
y = [A, G,  ]
```

模型在每个位置预测下一个 token。

loss 是所有位置的 cross entropy 平均值。

```python
loss = F.cross_entropy(logits.view(-1, vocab_size), labels.view(-1))
```

## 如何从教学版走向真实大模型

教学版和真实大模型的差异：

| 模块 | 教学版 | 工业级 |
| --- | --- | --- |
| Tokenizer | 字符级 tokenizer | BPE、SentencePiece、Unigram |
| 数据 | 几百行文本 | TB/PB 级高质量语料 |
| 模型 | 几层 Transformer | 数十到上百层，十亿到万亿参数 |
| 训练 | 单机 CPU/GPU | GPU 集群、分布式训练 |
| Attention | 普通 attention | FlashAttention、GQA、MQA、RoPE |
| 优化 | AdamW | AdamW、ZeRO、FSDP、混合精度 |
| 评估 | 看 loss 和生成效果 | Benchmark、人工评估、安全评估、业务评估 |
| 部署 | 本地脚本 | vLLM、TGI、TensorRT-LLM、模型网关 |

## 工业级从 0 构建大模型流程

### 1. 明确模型目标

需要先确定：

1. 参数规模：1B、7B、14B、70B。
2. 语言：中文、英文、多语言。
3. 能力：通用对话、代码、数学、领域模型。
4. 上下文长度：4K、32K、128K。
5. 部署目标：云端 API、私有化、本地端侧。

### 2. 数据工程

数据通常比模型结构更重要。

```text
采集 -> 清洗 -> 去重 -> 质量打分 -> 安全过滤 -> 配比采样 -> tokenize -> packing
```

重点：

1. 高质量语料比例。
2. 去重，避免 benchmark contamination。
3. 安全过滤和隐私脱敏。
4. 代码、数学、中文、英文等数据配比。
5. 训练集和评估集隔离。

### 3. Tokenizer 训练

工业模型通常使用 BPE 或 SentencePiece。

需要决定：

1. vocab size，例如 32K、50K、100K。
2. 是否支持中文、代码、多语言。
3. 特殊 token，例如 `<bos>`、`<eos>`、`<pad>`、chat template token。
4. Tokenizer 压缩率。

### 4. 模型架构

主流 LLM 是 decoder-only Transformer。

常见设计：

1. RoPE 位置编码。
2. RMSNorm。
3. SwiGLU FFN。
4. GQA 或 MQA 降低 KV cache。
5. FlashAttention 提升训练效率。
6. Tie embedding 降低参数量。

### 5. Pretrain

训练目标是 next token prediction。

工程关键：

1. 混合精度 BF16/FP16。
2. 梯度裁剪。
3. 学习率 warmup 和 cosine decay。
4. checkpoint 保存和恢复。
5. 数据并行、张量并行、流水线并行。
6. loss spike 监控。
7. 训练吞吐 tokens/s 监控。

### 6. Post-train

Base model 不能直接当助手，需要 post-train：

```text
SFT -> DPO / RLHF -> Safety Alignment -> Evaluation
```

### 7. Evaluation

评估维度：

1. Perplexity。
2. 通用 benchmark。
3. 数学、代码、知识、推理。
4. 安全拒答。
5. 幻觉率。
6. 长上下文能力。
7. 业务评估集。

### 8. Deployment

部署要考虑：

1. 推理框架：vLLM、TGI、TensorRT-LLM。
2. 量化：INT8、INT4、AWQ、GPTQ。
3. KV cache 显存。
4. 并发和吞吐。
5. 流式输出。
6. 模型版本管理。
7. 灰度和回滚。

## 从 0 构建模型的面试回答模板

### 简洁版

> 从 0 构建模型要先定义任务和数据，再设计 tokenizer、dataset、模型结构、loss 和训练循环。如果是 LLM，通常是 decoder-only Transformer，用 next token prediction 做 pretrain。训练完成后还要做 SFT、偏好优化和安全对齐，最后通过评估和推理服务部署。工业级从 0 训练还需要大规模数据工程、分布式训练、checkpoint、混合精度和模型评估体系。

### 详细版

> 我会把它分成十步：第一明确模型目标和规模；第二准备和清洗数据；第三训练 tokenizer；第四把文本打包成 token 序列；第五设计 decoder-only Transformer，包括 embedding、position encoding、attention、FFN、LayerNorm 和 LM head；第六用 causal language modeling 训练；第七监控 loss、吞吐和 checkpoint；第八用 SFT 和 DPO/RLHF 做 post-train；第九用 benchmark 和业务评估集验证；第十用 vLLM 等框架部署，并监控延迟、成本和质量。

## 面试追问

### 为什么 LLM 用 decoder-only Transformer？

因为 LLM 的主要任务是自回归生成，下一个 token 只依赖前文。Decoder-only 结构简单、可扩展性强，适合大规模预训练和生成任务。

### 为什么需要 causal mask？

训练时模型预测当前位置的下一个 token，不能看到未来答案。Causal mask 用来屏蔽未来 token，保证训练和推理一致。

### 从 0 训练和 Fine-tuning 区别？

从 0 训练是随机初始化模型，用海量语料学习基础能力。Fine-tuning 是基于已有模型，用较少任务数据适配特定行为或风格。前者成本极高，后者更适合业务团队。

### 为什么 tokenizer 很重要？

Tokenizer 决定文本如何变成 token，影响上下文利用率、训练效率和多语言效果。中文、代码、多语言场景如果 tokenizer 不合适，会导致 token 数变多、成本增加、模型学习困难。

### 如何判断模型训练好了？

不能只看训练 loss。要看验证集 loss、benchmark、业务评估集、生成质量、安全性、幻觉率、长上下文能力和线上反馈。

### 小模型 demo 为什么不能代表真实 LLM？

小模型 demo 只能说明核心原理，例如 tokenizer、attention、loss 和生成。真实 LLM 的难点在大规模数据、模型规模、分布式训练、稳定性、对齐、安全和部署成本。
