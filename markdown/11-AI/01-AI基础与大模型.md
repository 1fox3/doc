# AI 基础与大模型

## 核心概念

### AI、ML、DL、LLM 的关系

AI 是让机器表现出智能行为的总称，机器学习是 AI 的一种方法，深度学习是机器学习中基于神经网络的方法，大语言模型是深度学习在自然语言和多模态任务上的大规模应用。

面试可以这样说：

> LLM 本质上是一个基于大量文本和代码数据训练出来的概率模型，它根据上下文预测下一个 Token。它不是数据库，也不是确定性规则引擎，所以会有幻觉、上下文限制和输出不稳定的问题。工程上通常会结合 RAG、工具调用、输出校验和人工兜底来提高可靠性。

### Token

Token 是模型处理文本的基本单位，可能是一个字、一个词、一个词片段或符号。

需要关注：

1. 上下文窗口按 Token 计算，不按字符计算。
2. 输入和输出 Token 都会影响费用和延迟。
3. 中文通常一个汉字接近一个 Token，但不同 tokenizer 会有差异。

### Embedding

Embedding 是把文本、图片、代码等内容转换成向量，用向量距离表示语义相似度。

常见用途：

1. RAG 文档检索。
2. 相似问题匹配。
3. 推荐、聚类、去重。
4. 文档分类和语义搜索。

### Transformer

Transformer 是现代大模型的核心结构，关键机制是 Self-Attention。

面试不需要推公式，但要能说清楚：

1. Self-Attention 让模型能关注上下文中和当前位置最相关的 Token。
2. Transformer 相比 RNN 更适合并行训练。
3. 大模型通过多层 Attention 和前馈网络学习复杂语言模式。

### Attention

Attention 可以理解为“当前生成内容应该重点参考上下文中的哪些信息”。

例如用户问“这家公司去年营收是多少”，模型需要关注“这家公司”和“去年营收”相关的上下文，而不是所有文本平均处理。

## 大模型训练阶段

### 预训练

预训练使用海量通用语料，让模型学习语言、事实、代码和推理模式。

特点：

1. 成本极高。
2. 得到的是通用基础模型。
3. 目标通常是预测下一个 Token。

### 指令微调

指令微调使用“指令 + 回答”的数据，让模型更像助手，能按照用户要求完成任务。

### RLHF

RLHF 是基于人类反馈的强化学习，用来让模型输出更符合人类偏好，例如更有帮助、更安全、更少有害内容。

### 微调

微调是在已有模型基础上，用特定领域数据继续训练。

适合：

1. 固定风格输出。
2. 特定领域术语。
3. 分类、抽取等稳定任务。

不一定适合：

1. 高频变化的知识库问答。
2. 需要可追溯引用的企业文档问答。
3. 权限敏感的实时数据查询。

这些场景通常优先考虑 RAG 或工具调用。

## 推理参数

### Temperature

Temperature 控制输出随机性。

1. 低 temperature：输出更稳定，适合问答、抽取、代码生成。
2. 高 temperature：输出更多样，适合创意写作、头脑风暴。

### Top-p

Top-p 控制模型从累计概率最高的一批候选 Token 中采样。

实际应用中通常不要同时大幅调整 temperature 和 top-p，避免输出不可控。

### Max Tokens

限制最大输出长度。设置过小会截断答案，设置过大会增加成本和延迟。

### Stop Sequence

用于指定模型遇到某些字符串时停止生成，常用于结构化输出、代码生成和多轮协议控制。

## 幻觉问题

幻觉是指模型生成了看似合理但实际错误或无依据的内容。

常见原因：

1. 模型本质是概率生成，不是事实数据库。
2. 上下文缺少必要信息。
3. Prompt 要求模型必须回答，导致模型编造。
4. 检索结果错误或上下文混入噪声。

缓解方法：

1. 使用 RAG 提供可信上下文。
2. Prompt 中要求“不知道就回答不知道”。
3. 要求答案引用来源。
4. 对关键字段做程序校验。
5. 高风险场景加入人工审核。

## 面试常问

### LLM 和搜索引擎有什么区别？

LLM 是生成模型，擅长理解、总结、改写和推理，但可能产生幻觉。搜索引擎是检索系统，擅长从已有网页或文档中查找信息，但不会天然完成复杂推理。企业应用里常把两者结合，用检索系统找到可靠材料，再由 LLM 生成答案。

### 为什么大模型会胡说？

因为它的训练目标是根据上下文预测最可能的 Token，而不是验证事实真伪。当上下文不足、问题超出知识范围或 Prompt 强迫回答时，模型可能生成概率上合理但事实错误的内容。

### 微调和 RAG 怎么选？

如果问题是补充知识、需要实时更新、需要引用来源，优先 RAG。如果问题是固定任务模式、固定输出风格、领域语言适配，可以考虑微调。实际项目中经常是 RAG 解决知识，Prompt 和少量微调解决行为风格。
