# AI 项目实战案例

## 案例一：企业知识库问答

### 业务目标

员工可以用自然语言查询公司制度、技术文档、操作手册，减少人工咨询和搜索成本。

### 架构

```text
文档源 -> 文档解析 -> 切分 -> Embedding -> 向量库
用户问题 -> 权限过滤 -> 混合检索 -> Rerank -> LLM -> 带引用答案
```

### 难点

1. 文档格式复杂，PDF、Word、表格解析质量不稳定。
2. Chunk 切分影响召回。
3. 不同部门文档有权限隔离。
4. 多个文档版本可能冲突。
5. 用户问题表达和文档标题不一致。

### 优化点

1. 按标题层级切分，并保留 metadata。
2. 使用向量检索 + BM25 混合检索。
3. 引入 reranker。
4. Prompt 要求只基于上下文回答并引用来源。
5. 建立问答评估集，持续评估 Recall@K 和答案准确率。

### 面试讲法

> 我做企业知识库时没有直接把文档塞给模型，而是做了 RAG。文档侧做解析、清洗、切分和向量化，查询侧做权限过滤、混合检索和重排，然后让模型基于 topK 上下文回答并给出引用。核心优化点是 chunk 策略、混合检索、rerank 和评估集。

## 案例二：智能客服

### 业务目标

自动回答用户常见问题，降低人工客服压力，提高响应速度。

### 架构

```text
用户问题 -> 意图识别 -> FAQ/RAG 检索 -> LLM 生成 -> 风险判断 -> 自动回复或转人工
```

### 难点

1. 用户表达口语化。
2. 同一问题有多个业务条件。
3. 涉及订单、退款、投诉等高风险操作。
4. 错误回答会影响用户体验。

### 优化点

1. 对高频问题建立 FAQ 精确命中。
2. 对复杂问题使用 RAG。
3. 订单类问题通过工具查询实时状态。
4. 退款、赔付、投诉转人工或二次确认。
5. 记录用户反馈用于评估。

### 面试讲法

> 智能客服不能只靠大模型自由回答。我会把问题分层：高频 FAQ 走确定性匹配，知识类问题走 RAG，订单状态走工具调用，高风险问题转人工。这样能兼顾效率、准确率和风险控制。

## 案例三：文档总结与信息抽取

### 业务目标

从合同、报告、会议纪要中抽取关键信息，生成摘要和结构化字段。

### 架构

```text
文件上传 -> 文档解析 -> 分段处理 -> LLM 抽取 -> JSON 校验 -> 合并结果 -> 人工确认
```

### 难点

1. 长文档超过上下文窗口。
2. 表格和扫描件解析不稳定。
3. 模型输出 JSON 可能不合法。
4. 字段抽取需要高准确率。

### 优化点

1. 分段摘要再合并。
2. OCR 和版面解析单独处理。
3. 使用 JSON schema 校验。
4. 对金额、日期、主体名称做规则校验。
5. 关键字段人工复核。

### 面试讲法

> 长文档处理不能一次性丢给模型。我会先解析文档结构，再分段抽取，最后合并结果。输出必须走 schema 校验，对金额、日期这类关键字段再做规则校验，必要时进入人工复核。

## 案例四：代码助手

### 业务目标

帮助研发理解代码、生成单测、解释报错、做代码审查。

### 架构

```text
代码索引 -> 语义检索 -> 相关文件上下文 -> LLM -> 建议或补丁
```

### 难点

1. 代码上下文分散在多个文件。
2. 模型可能编造不存在的 API。
3. 生成代码需要符合项目风格。
4. 需要避免修改无关代码。

### 优化点

1. 建立代码索引和符号检索。
2. 提供相关文件、调用关系和测试信息。
3. 生成后运行测试和静态检查。
4. 限制修改范围。
5. 通过代码审查流程合入。

### 面试讲法

> 代码助手的关键不是让模型凭空写代码，而是给它准确的代码上下文，包括相关文件、调用关系和测试。生成后必须经过编译、测试和人工 review。

## 案例五：自动化工作流 Agent

### 业务目标

用户用自然语言触发多步骤任务，例如“帮我查本周异常订单并生成报表发给运营”。

### 架构

```text
用户目标 -> Agent 计划 -> 工具调用 -> 中间结果 -> 审核 -> 执行最终动作
```

### 难点

1. 工具调用链路长。
2. 参数可能错误。
3. 高风险动作需要确认。
4. Agent 可能循环或跑偏。

### 优化点

1. 限制最大步骤数。
2. 工具参数 schema 校验。
3. 高风险动作人工确认。
4. 全链路审计日志。
5. 失败可恢复和可重试。

### 面试讲法

> Agent 适合目标明确但路径不完全固定的任务。生产中我不会让 Agent 完全自由执行，而是限制工具权限、步骤数和超时，高风险动作需要人工确认，所有工具调用都记录审计日志。
