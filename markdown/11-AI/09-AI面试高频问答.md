# AI 面试高频问答

## 你做过哪些 AI 应用？

回答模板：

> 我主要做过 AI 应用层的落地，比如企业知识库问答、文档总结抽取、智能客服和内部效率工具。技术上不是简单调用大模型 API，而是结合 RAG、Prompt 编排、工具调用、结构化输出校验、权限控制和监控评估，解决准确率、成本、延迟和安全问题。

## 如何设计一个 RAG 系统？

回答模板：

> 我会分为离线知识处理和在线问答两部分。离线侧做文档解析、清洗、切分、Embedding 和向量入库，并保留来源、权限、时间等 metadata。在线侧对用户问题做改写和权限过滤，使用向量检索加关键词检索召回，再通过 reranker 排序，最后把 topK 上下文交给大模型生成带引用的答案。

## RAG 如何解决幻觉？

回答模板：

> RAG 不能彻底消除幻觉，但能显著降低。它通过检索外部可信知识给模型提供依据，再通过 Prompt 限制只基于上下文回答，要求引用来源，遇到资料不足时拒答。工程上还要做检索评估、答案校验和人工兜底。

## RAG 效果不好怎么排查？

回答模板：

> 我会先拆分问题是召回不好还是生成不好。如果正确文档没召回，检查文档入库、chunk、embedding、query rewrite、混合检索和 metadata filter。如果召回到了但答错，检查 rerank、上下文噪声、Prompt 约束、引用和模型能力。最后用评估集量化 Recall@K、答案准确率和引用准确率。

## 微调和 RAG 怎么选？

回答模板：

> 如果是知识更新、企业文档问答、需要引用来源，优先 RAG。如果是固定任务、固定格式、特定语气或领域表达，可以考虑微调。很多项目会组合使用：RAG 解决知识，Prompt 或微调解决输出风格和任务模式。

## Agent 和 Chatbot 有什么区别？

回答模板：

> Chatbot 通常是单轮或多轮问答，主要生成文本。Agent 是目标驱动的，可以规划步骤、调用工具、观察结果并继续执行。Agent 能处理更复杂任务，但也带来更高成本、延迟和不可控风险，所以生产环境要限制权限、步骤数和高风险动作。

## 如何控制 LLM 成本？

回答模板：

> 我会从模型、Token、缓存和配额四个方面控制。简单任务用小模型，复杂任务才用强模型；压缩 Prompt 和上下文，控制 RAG topK；对重复问题和文档摘要做缓存；按用户或租户设置 Token 配额，并监控单次调用成本和异常用量。

## 如何保证输出稳定性？

回答模板：

> 先降低 temperature，明确 Prompt 结构和输出格式，必要时给 few-shot 示例。结构化输出使用 JSON mode 或 function calling，并在服务端做 schema 校验和失败重试。关键业务结果还要做规则校验或人工确认。

## 如何处理大模型接口超时？

回答模板：

> 我会设置合理超时和取消机制，长任务异步化，聊天类使用流式输出改善体感延迟。失败时根据错误类型做重试或降级，例如切换备用模型、返回检索结果或转人工。同时监控首 Token 延迟、总延迟和超时率。

## 如何防止 Prompt Injection？

回答模板：

> 不能只靠 Prompt 防注入。需要把用户输入和检索文档都当作不可信数据，明确区分指令和数据。真正的安全控制要放在后端，包括工具权限校验、检索权限过滤、敏感输出检测、审计日志和高风险动作人工确认。

## 如何评估一个 AI 应用？

回答模板：

> 要同时评估效果、体验、成本和安全。效果上看准确率、召回率、幻觉率、引用准确率；体验上看延迟和用户反馈；成本上看 Token 和模型费用；安全上看越权、敏感信息和高风险输出。最好建立离线评估集加线上反馈闭环。

## 如何选择模型？

回答模板：

> 先看任务类型和约束，再用业务评估集对比。简单抽取和分类可以用小模型，复杂推理和代码任务用强模型。数据敏感或高调用量场景考虑私有化部署，快速验证和能力要求高的场景可以先用闭源 API。

## AI 应用中如何做权限控制？

回答模板：

> 权限控制必须在业务服务、检索层和工具层完成，不能依赖模型判断。RAG 检索时按用户身份过滤文档，工具调用时走后端鉴权，输出来源也要检查权限，同时记录审计日志。

## 如果模型回答错了怎么办？

回答模板：

> 先定位错误来自检索、Prompt、模型能力还是业务数据。如果是检索问题，优化 chunk、召回和 rerank；如果是生成问题，优化 Prompt、结构化约束和模型选择；如果是数据问题，修复知识源和同步流程。高风险场景需要人工兜底和反馈闭环。

## 怎么介绍自己的 AI 项目亮点？

回答模板：

> 我会突出三个点：第一，业务价值，例如减少人工处理时间或提升问答效率；第二，工程设计，例如 RAG、工具调用、权限控制、流式输出和监控；第三，效果优化，例如评估集、召回率提升、成本下降和稳定性提升。
