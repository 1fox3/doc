# 锁

> 专题：MySQL
> 来源：面试内容汇总/Mysql.xmind

## 补充与实践

- 补充：InnoDB 通过 undo log 和 ReadView 实现 MVCC，通过 redo log 保证崩溃恢复，通过 binlog 支持复制和归档。
- 实践：长事务会拖住 undo 清理并影响版本链，线上应限制事务时长和交互式事务。
- 误区：可重复读不等于完全不会幻读，InnoDB 通过 next-key lock 在当前读场景解决幻读。
