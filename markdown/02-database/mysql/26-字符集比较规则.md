# 字符集比较规则

## 推荐配置

- 新系统统一使用 `utf8mb4`，MySQL 8.0 可优先考虑 `utf8mb4_0900_ai_ci`。
- 如果业务要求大小写敏感，可选择 `_cs` 或 `_bin` 规则。
- 连接池初始化应固定字符集、时区和 `sql_mode`，避免不同连接行为不一致。

## 技术影响

- 字符集影响存储字节数，进而影响索引长度、页容量和排序临时空间。
- 比较规则影响等值判断、唯一索引判重、排序结果和 `LIKE` 匹配。
- 混合字符集比较可能触发隐式转换，导致索引无法按预期使用。

## 常用 SQL

```sql
SHOW VARIABLES LIKE 'character_set%';
SHOW VARIABLES LIKE 'collation%';
SHOW CREATE DATABASE db_name;
SHOW CREATE TABLE t_name;
```

## 迁移注意

- 修改库默认字符集不会自动改已有表和列。
- 表级转换前要评估唯一键冲突、索引长度和业务排序规则变化。
- 对大表执行字符集转换通常是重建表，需要评估锁和在线变更方案。

## 层级优先级

- 列定义优先于表默认值。
- 表默认值优先于数据库默认值。
- 数据库默认值优先于服务器默认值。
- 连接字符集影响客户端字符串字面量如何解释，以及结果如何返回给客户端。

## 典型故障

- 表中历史数据已经乱码，单纯修改连接编码无法恢复原始字符。
- 应用连接使用 `latin1` 写入 UTF-8 字节，数据库按 latin1 存储，读取时可能出现双重编码问题。
- 迁移到大小写不敏感 collation 后，原本大小写不同的唯一值可能冲突。

## 校验 SQL

```sql
SELECT table_schema, table_name, table_collation
FROM information_schema.tables
WHERE table_schema = 'db_name';

SELECT table_schema, table_name, column_name, character_set_name, collation_name
FROM information_schema.columns
WHERE table_schema = 'db_name'
  AND character_set_name IS NOT NULL;
```
