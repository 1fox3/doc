## 补充与实践

- 补充：MySQL 8.0 优化器会结合统计信息、直方图和成本模型选择执行计划，索引存在不代表一定会用。
- 实践：设计复合索引时按等值、范围、排序、覆盖的优先级综合考虑，同时关注字段选择性和写入成本。
- 排障：慢 SQL 分析先看 `EXPLAIN ANALYZE`、扫描行数、回表次数、临时表和文件排序。
