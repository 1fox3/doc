# InnoDB数据页结构

## 页的角色

- InnoDB 以页为磁盘和 Buffer Pool 的基本管理单位，默认页大小为 16KB。
- 数据页既是聚簇索引和二级索引的节点，也是范围扫描、页分裂、页合并和预读的基础。

## 数据页组成

- `File Header` 保存页号、校验和、LSN、页类型和链表指针。
- `Page Header` 保存页内记录数量、空闲空间、页目录等管理信息。
- `Infimum` 和 `Supremum` 是页内最小和最大伪记录，用于维护记录链表边界。
- `User Records` 保存真实行记录，`Free Space` 保存可用空间。
- `Page Directory` 保存槽位，支持在页内快速二分定位记录。
- `File Trailer` 用于页完整性校验。

## 页内查找

- B+Tree 先通过根页和非叶子页定位到目标叶子页。
- 叶子页内通过页目录粗定位槽位，再沿记录链表精确查找。
- 页内记录按索引键有序排列，范围扫描可沿页内记录和页间双向链表顺序读取。

## 实践要点

- 随机主键会导致叶子页频繁分裂，增加碎片和写放大。
- 批量删除后页空间不一定立即归还给操作系统，可能只形成页内空洞或可复用页。
- 页损坏、校验失败和 LSN 异常通常需要结合错误日志、备份和 InnoDB 恢复参数处理。

## 页分裂和页合并

- 插入记录时，如果目标页空间不足，InnoDB 会分裂页并调整 B+Tree 指针。
- 顺序递增主键通常追加到右侧页，分裂较少；随机主键会在任意页插入，更容易产生分裂和碎片。
- 删除记录后空间先在页内复用，页利用率过低时可能触发页合并。
- 页分裂会增加 redo 写入、Buffer Pool 脏页和磁盘随机写。

## 读路径细节

- 根据索引条件从根页开始比较键值，逐层定位到叶子页。
- 叶子页不在 Buffer Pool 时触发一次 16KB 页读取。
- 叶子页内通过 Page Directory 缩小范围，再沿记录链表比较具体记录。
- 范围扫描会沿叶子页链表继续向前或向后读取，扫描量由索引区间决定。

## 观察角度

- 随机 I/O 高时，检查热点查询是否大量回表或范围扫描过宽。
- Buffer Pool 命中率低时，页读取会直接反映为磁盘读延迟。
- 表碎片明显时，可以比较 `data_length`、`index_length`、实际行数和平均行长。
