## 脑图内容

## InnoDB数据页结构

### 页结构

#### File Header(文件头部)

- 页的一些通用信息

#### Page Header(页面头部)

- 数据页专有的一些信息

#### Infimum

- 页面中的最小记录(虚拟记录)

#### Supermum

- 页面中的最大记录(虚拟记录)

#### User Records(用户记录)

- 用户存储的记录内容
  - 记录通过next_record方式，组成链表的形式

#### Free Space(空闲空间)

- 页中尚未使用的空间

#### Page Directory(页目录)

- 页中某些记录的相对位置
  - 记录查找方式
    - 采用二分法查找记录所在分组对应的槽，然后找到该槽所在分组中主键最小的那条记录
    - 通过记录的next_record属性遍历该槽所在组中的各个记录
  - 分组规则
    - Infimum单独为一个分组
    - Supermum所在的分组可以由1-8条记录
    - 普通分组可有4~8条记录
  - 每个分组在Page Directory存在一个槽，槽指向个分组中主键最大的记录

#### File Trailer(文件尾部)

- 校验页是否完整
