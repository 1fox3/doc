## 脑图内容

## InnoDB记录存储结构

### InnoDB页

#### innodb_page_size

- InnoDB页大小配置。默认16KB，内存最小加载单位

#### 在MYSQL数据目录初始化时(mysqld --initialize)就已确定，不能被修改

### InnoDB行格式

#### 指定行格式

- ROW FORMAT=

#### 分类

- COMPACT
  - 额外信息
    - 变长字段长度列表
      - 逆序存放
    - NULL值列表
      - 二进制位逆序存放
      - 1是null，0不是null
    - 记录头信息(5字节)
      - 预留位1
      - 预留位2
      - deleted_flag
        - 该记录是否被删除
      - min_rec_flag
        - B+树中每层非叶子节点中的最小的目录项记录都会添加该标记
      - n_owned
        - 一个页面中的记录会被分成若干个组，每个组中有一个记录时带头大哥，其他的都是小弟，带头大哥记录的n_owned值代表该组中所有的记录条数，小弟记录的n_owned值为0
      - heap_no
        - 当前记录在页面堆中的相对位置
      - record_type
        - 当前记录的类型，0-普通记录，1-B+树非叶子节点的目录项记录，2-Infimum记录，3-Supermum记录
      - next_record
        - 下一条记录的相对位置
  - 真实数据
    - 默认添加列
      - row_id
        - 行id，唯一标识，不一定有
      - trx_id
        - 事务id
      - roll_pointer
        - 回滚指针
    - 各列的值

- REDUNDANT
  - 额外信息
    - 字段长度偏移列表
      - 逆序存放
      - 有专门的NULL比特位来标识该字段是否为NULL
    - 记录头信息(5字节)
      - 预留位1
      - 预留位2
      - deleted_flag
        - 该记录是否被删除
      - min_rec_flag
        - B+树中每层非叶子节点中的最小的目录项记录都会添加该标记
      - n_owned
        - 一个页面中的记录会被分成若干个组，每个组中有一个记录时带头大哥，其他的都是小弟，带头大哥记录的n_owned值代表该组中所有的记录条数，小弟记录的n_owned值为0
      - heap_no
        - 当前记录在页面堆中的相对位置
      - n_field
        - 记录中列的数量
      - 1byte_offs_flag
        - 字段长度偏移列表中每个列对应的偏移量时使用的1字节还是2字节表示的
      - next_record
        - 下一条记录的相对位置
  - 真实数据
    - 默认添加列
      - row_id
        - 行id，唯一标识，不一定有
      - trx_id
        - 事务id
      - roll_pointer
        - 回滚指针
    - 各列的值

- DYNAMIC
  - 与COMPACT格式很像
  - 处理溢出页时，仅记录20字节大小的指向溢出页的地址
  - Mysql5.7默认格式

- COMPRESSED
  - 对页面进行了压缩

#### 溢出列

- 处理方式
  - 仅记录前678字节，以及一个指向其他页面的指针
  - 剩下的数据用过链表的方式存储

- 产生临界点
  - 2条记录大于页容量
