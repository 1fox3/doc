#### 数据结构

#### 索引

- 不可变

- 倒排索引
  - 分词在文档中出现的位置和数量
  - 基于FST数据结构实现

#### _uid与_id

- _uid是内部进行存储和索引使用的，_id通常是从_uid中提取出来的

#### 文档修改方式

- 删除再新建

#### 评分

- TF_IDF(向量空间模型)
  - TF
    - 一个分词在index的某个文档中出现的次数越多，则权重越高
  - IDF
    - 一个分词在index的不同文档中出现的次数越多，者权重越低

- BM25(概率模型) 5.0+默认方式

#### 主节点选举

- 节点将自己直到的后选节点按照nodeId字段排序，选择第一个节点投票

- 票数超过一半当选

#### 节点类型

- 数据节点
  - 负责保存数据，段合并，执行查询
  - node.data:true

- 主节点
  - 负责管理整个集群
  - node.master:true

- Ingest节点
  - 数据处理管道有一到多个ingest节点组成，有ingest节点负责每个环节的处理
  - node.ingest:true

- 部落节点
  - 是一种特殊的协调节点，可以连接多个集群，并在所有集群中执行查询或其他操作

- 协调节点
  - 查询的执行过程分为分散阶段和集中阶段，这2个阶段都由接受查询请求的协调节点来管理，它们也是集群中的负载均衡器

#### 数据转换与联盟搜索

- ingest管道
  - 结构定义
    - description
      - 描述
    - processors
      - 处理器列表
  - 接口
    - _ingest
  - 处理管道中的错误
    - on_failure，出现错误时，处理过程
    - ignore_failure=true，忽略错误
  - 内置处理器，23种
    - Append处理器
    - Convert处理器
    - Grok处理器

- 联盟搜索
  - 定义
    - 同是使用多个集群
  - 创建部落节点
    - 以tribe为前缀
  - 部落节点可以查询数据，但是不能通过部落节点创建索引
  - 处理索引冲突
    - tribe.on_conflict，默认any
      - any
        - 随机选择一个索引
      - drop
        - 忽略同名索引
      - prefer_TRIBE_NAME
        - 允许选择使用哪个集群的索引
  - 屏蔽写操作
    - tribe.blocks.write
      - 所有写请求
    - tribe.blocks.metadata
      - 禁止元数据修改
    - tribe.blocks.indices.write
      - 禁止索引修改

#### 索引更新

- 强制索引更新
  - _refresh

- 索引更新周期
  - refersh_interval
    - 默认1秒

#### 事务日志

- 强制事务日志刷新
  - _flush

- 相关配置
  - index.translog.sync_interval
    - 日志多久同步到磁盘上一次，默认5秒
  - index.translog.durability
    - 控制在每次索引、删除、更新或批量请求操作后，是否要同步并提交事务日志
      - request
        - 每次请求都会同步并提交
      - async
        - 每隔sync_interval才会在后台做一次同步和提交操作
  - index.translog.flush_threshold_size
    - 事务日志的最大容量，当容量超过该参数值时，就强行进行事务日志刷新操作，默认512MB

- 数据恢复
  - elasticsearch-translog truncate -d /path/to/trans/log/

- 实时读取
  - pretty
